## 기능

### 1) 목적지까지 “안전하게 도착”에 필요한 핵심 기능

#### 1-1. 음성 기반 경로 안내 (턴바이턴 + 세밀 단계)

- “200m 직진” 같은 큰 덩어리 말고, **10~30m 단위로 짧고 확실하게**
- 안내 문장 구조는 일정하게:  
    `[현재상태] → [행동] → [근거/랜드마크] → [다음 체크포인트]`  
    예) “점자블록 위입니다. 20m 직진하세요. 오른쪽에 펜스가 이어집니다. 20m 후 횡단보도입니다.”

- **프론트**
    - 내비 화면(최소 UI) + 음성(TTS) 출력, 안내 큐 관리(중복/연타 방지)
    - 진동 패턴 출력(좌/우/정지/주의 등)
    - GPS/자이로/나침반 센서 수집 및 샘플링(배터리 고려)
    - “안내 빈도/문장 길이/진동 강도” 설정 UI 및 즉시 반영
- **백엔드**
    - 경로 요청/응답 표준화(waypoint, step, maneuver, distance 등)
    - 안내 세션(시작/종료/강제중단) 상태 관리
    - 최근 경로/즐겨찾기/목적지 저장
    - 로그(오류, 이탈, 재탐색 횟수) 수집
- **AI**
    - “안내 문장 생성” 최적화(한 문장=한 행동, 랜드마크 삽입, 과잉 안내 억제)
    - 상황에 따른 안내 강도 조절(복잡 교차로/직선 보도 등)

#### 1-2. 기준선(점자블록/연석/벽/펜스) 추종 안내

- 전맹 보행은 “직선 + 기준선”이 안정적이니까, 앱이 이를 적극 지원해야 함
- 가능하면:
    - **점자블록 감지/추종**
    - **보도 경계(연석) / 펜스 / 벽 등 ‘따라갈 수 있는 선’ 안내**
    - 기준선이 끊길 때 “끊김”을 명확히 알림 (“점자블록이 끊깁니다. 3m 직진 후 오른쪽 기준선을 찾으세요.”)

- **프론트**
    - 카메라 스트림 입력(필요 시), 온디바이스 전처리(해상도/프레임 조절)
    - 기준선 감지 결과를 즉시 피드백(“점자블록 유지”, “기준선 끊김”)으로 출력
    - 사용자가 “카메라 모드 ON/OFF” 쉽게 전환
- **백엔드
    - (선택) 기준선 관련 지도 데이터 레이어 저장/서빙(점자블록 구간, 보도 경계 등)
    - 사용자 피드백 기반 지도 품질 개선용 데이터 적재(익명)
- **AI**
    - 점자블록/보도경계 세그멘테이션
    - 기준선 “유지 가능성” 판단(끊김/가려짐/우회 필요)
    - 기준선 기반 경로 안정화(걷는 방향을 직선으로 유지하도록 교정 신호 생성)

#### 1-3. 횡단보도 & 신호등 상태 안내 (보행 안전 최우선)

- “지금 건너도 되는가”는 지도앱에서 제일 위험한 구간
- 필요한 안내:
    - 횡단보도 접근 알림 (미리: 10~20m 전)
    - **정확한 대기 위치(신호등/기둥/점자블록)** 유도
    - 신호 상태(보행 가능/대기)와 남은 시간(가능하면)
    - 대각선/다중 횡단보도에서 “어느 방향으로 건너는지” 확실히

- **프론트**
    - 횡단보도 접근 시 강한 알림(음성+진동) UX
    - 카메라로 신호등/횡단보도 인식 모드 자동 전환(사용자 설정 가능)
    - “대기 위치” 안내를 위해 방향/거리 피드백 루프 제공
- **백엔드**
    - 횡단보도/교차로 POI 데이터(좌표, 방향성, 횡단보도 ID) 제공
    - (가능 시) 공공데이터/신호 API 연동 및 캐시
    - 교차로 위험도/사고 다발 지점 데이터 관리
- **AI**
    - 신호등 상태 인식(보행자 신호: 가능/대기/점멸 등)
    - 횡단보도 위치 인식 및 “건널 방향” 추론(대각선, 다중 횡단 등)
    - 위험 판단(차량 접근/불법 우회전 등은 가능하면)

#### 1-4. 장애물 탐지 및 위험 알림 (실시간)

- 킥보드/콘/바리케이드/공사구간/차도 침범 등
- 단순 “장애물 있음”이 아니라:
    - **방향(12시/3시/9시)** + 거리(대략) + 회피 행동(좌/우/정지)
    - 위험도에 따라 알림 강도 다르게(진동 패턴)

- **프론트**
    - 카메라 기반 탐지 UI(화면 최소, 음성/진동 중심)
    - 위험도에 따른 진동 패턴/음성 톤(짧고 강하게)
    - “정지” 같은 즉시 행동을 최우선으로 끼워넣는 인터럽트 처리
- **백엔드**
    - 위험 구간 제보/검증/공유(크라우드) 기능
    - 위험 이벤트 로그 저장(어디서 어떤 위험이 자주 발생하는지)
    - 사용자별 알림 민감도 저장
- **AI**
    - 객체 탐지(킥보드, 콘, 바리케이드, 공사 펜스, 노면 단차 등)
    - 거리/방향 추정(12시 방향, 2~3m 등)
    - 위험도 스코어링(충돌 가능성/회피 가능성) + 행동 추천(좌회피/우회피/정지)

### 2) Orientation(내 위치/방향) 확보 기능

#### 2-1. 방향 안정화(“내가 어디를 보고 있는지”)

- GPS만으로는 방향이 흔들리면 안내가 무의미해짐
- 필요한 기능:
    - “정면이 북쪽입니다/목적지 방향은 2시입니다” 같은 **시계방향 안내**
    - 회전할 때 **진동으로 방향 맞추기(목적지 방향에 가까워질수록 진동 약해짐/패턴 변화)**

- **프론트**
    - 나침반/자이로 기반 “방향 맞추기” 인터랙션(진동으로 정렬)
    - 안내 음성에서 “2시 방향” 같은 표현을 일관되게 사용
- **백엔드**
    - 방향 보정 관련 파라미터(기기별 센서 편차) 저장(선택)
    - 세션별 센서 품질 지표 기록(디버깅/개선)
- **AI**
    - 센서 융합(자이로+가속도+나침반)으로 헤딩 안정화(필터링)
    - “사용자가 실제로 걷는 진행 방향” 추정(흔들림 제거)

#### 2-2. 내 위치를 “말로 이해”시키는 주변 설명

- 전맹은 화면 지도가 없으니 주변을 언어로 구성해야 함
- 예: “현재 OO역 3번 출구 앞, 오른쪽 5m에 편의점, 앞쪽은 큰 횡단보도”
- 즉 **랜드마크 기반 위치 설명**이 있어야 방향정위가 됨

#### 2-3. 길 이탈 감지 & 복귀 안내

- “경로에서 벗어났습니다”만 말하면 공포감 큼
- 필요한 안내:
    - 얼마나 벗어났는지(대략)
    - 복귀 행동(“3m 뒤로” / “왼쪽으로 30도 회전” / “점자블록 찾기”)

- **프론트**
    - 이탈 감지 시 즉시 알림 + “복귀 모드” UX(짧은 지시 반복)
    - “뒤로 3m” 같은 행동을 안전하게 수행하도록 안내 속도 조절
- **백엔드**
    - 재탐색(리라우팅) API 호출 및 결과 저장
    - 이탈/복귀 이벤트 로그 수집(어느 구간에서 많이 이탈하는지)
- **AI**
    - 이탈 판단 로직(정확히 얼마나 벗어났는지, 계속 벗어나는지)
    - 복귀 전략 선택(되돌리기 vs 우회 경로로 재설계)
    - “기준선 다시 잡기” 같은 전맹 친화 복귀 지시 생성

### 3) 목적지 도착 이후까지: “마지막 30m” 문제 해결

- **프론트**
    - 도착 후에도 안내 유지(입구 찾기 모드)
    - 주변 탐색(카메라/센서) 기반 “입구 탐색 루프” 제공
- **백엔드**
    - 장소 상세: 출입구 좌표/정문·후문/접근 난이도/랜드마크 메타데이터 저장
    - 사용자 메모(“이 건물 입구는 오른쪽으로 돌아가야 함”) 저장
- **AI**
    - 입구/간판/문 형태 인식(가능 범위 내)
    - 도착점 주변의 “가장 설득력 있는 접근 경로” 추론(장애물/기준선 고려)
    - 사용자 메모를 요약해 도착 안내 문장에 반영

#### 3-1. 목적지 입구/출입문/정확한 도착점 안내

- 전맹은 “도착했습니다” 이후가 진짜 시작인 경우 많음
- 필요한 기능:
    - 건물 **정문/후문/출입구** 구분
    - “입구는 오른쪽으로 8m 이동, 자동문 소리 방향” 같은 접근 안내

#### 3-2. 층/엘리베이터/계단 정보 (가능 범위 내)

- “건물 안”까지는 실외 지도앱 범위를 넘을 수 있지만,
    - 최소한 “엘리베이터 위치”, “계단 우회” 같은 정보가 있으면 체감이 큼

### 4) 검색/장소 정보는 “전맹에게 의미 있는 것만”

- **프론트**
    - 음성 입력 기반 검색(필수급), 결과 리스트를 TTS로 읽기
    - 상세 정보는 “접근성 핵심 필드”만 읽게 UI 구성(과잉 정보 차단)
    - 즐겨찾기/최근검색/내 메모 접근 UX
- **백엔드**
    - 장소 검색(외부 지도 API 연동: 카카오 등) + 결과 정규화
    - 장소 상세 조회 + 접근성 필드 구조화
    - 최근검색/즐겨찾기 CRUD
    - ~~사용자 메모/태그 저장~~
- **AI**
    - 검색 결과 랭킹 보정(“접근 쉬운 곳” 우선)
    - 리뷰/텍스트에서 접근성 단서 추출(계단, 경사로, 출입구 등)
    - 상세정보를 전맹용 “요약 템플릿”으로 재구성(행동 중심)

#### 4-1. 장소 상세 정보: 핵심만, 접근성 중심

전맹 기준으로 “좋은 장소 정보”는 리뷰가 아니라 **도착/출입 가능성**임.

- 꼭 필요한 필드 예시:
    - **정확한 출입구 위치(어느 방향/어느 도로쪽)**
    - **주요 랜드마크(출구 번호, 큰 간판, 주변 구조물)**
    - **접근 경로 특징(점자블록 유무, 계단/경사로, 공사 잦음 등)**
    - **실내 안내 필요 여부(복잡/단순)**

#### 4-2. 즐겨찾기/최근검색은 “경로 재사용”이 핵심

- “집→회사” 같은 반복 경로를 **한 번 설정하면 다시 쓰기**
- 즐겨찾기엔 주소보다 “내가 이해하는 이름(‘회사 정문’ ‘병원 접수처’)”이 중요

### 5) 알림 설계: 말이 많으면 오히려 위험해짐

- **프론트**
    - 설정 UI(슬라이더/프리셋: 조용/표준/촘촘)
    - 상황별 자동 전환 토글(교차로에서 강화 등)
    - TTS 속도/톤, 진동 강도 제어
- **백엔드**
    - 설정 조회/변경 API
    - 사용자별 프리셋 저장 및 버전 관리(앱 업데이트에도 유지)
- **AI**
    - 상황 인식 기반 알림 스케줄링(지금 말해야 하는지, 침묵해야 하는지)
    - 같은 메시지 반복 억제 + 핵심만 남기는 문장 압축

#### 5-1. 안내 빈도/문장 길이/진동 강도 개인화

- 사용자가 “지금은 조용히/지금은 촘촘히”를 선택 가능해야 함
- 상황 기반 자동 전환도 유용:
    - 횡단보도/복잡 교차로/공사구간에서는 안내 강화
    - 직선 보도에서는 안내 최소화

#### 5-2. 한 문장에 하나의 행동

- 전맹에게 길 안내는 “행동 지시”가 가장 중요
- 정보를 한꺼번에 쏟으면 사고 위험 증가

### 6) 있으면 체감 큰 “플러스 기능”

- **정류장 안내(버스/지하철 출입구 방향)**
- **실시간 위험 제보 공유(공사/킥보드 적치 구간)**
- **야간/우천 시 모드(미끄럼/물튀김 위험 강조)**
- **긴급 버튼(현재 위치를 보호자/도움 요청으로 공유)**

## 논의 목록

1. 근처(최대 3km 반경) 특정 장소(카테고리로 구분) 목록
	1. 카테고리를 다중으로 받는 것을 허용할 것인지 혹은 단일로 유지할 것인지
2. 검색 결과 목록 보여줄 때
	1. pagination 제거 후 무한 스크롤로 변경하는 건 어떤지
3. Kakao Local REST API에서는 장소 상세 정보 확인이 안 됨 & Kakao Map에 경우 JS, 프론트단에서 진행됨
	1. 상세 정보라고 해도 시각장애인이 상세 정보를 다 볼 수 있을까? 전맹 시각장애인일 경우 안 보여서 정보를 들어야 할 텐데 그러면 많은 정보가 필요할까?
	2. 상세 정보로 보여줄 데이터가 무엇이 있을지
		1. 장소명(도로명 주소) + 유형(카테고리)
		2. 현 위치로부터의 거리(미터) + 방향(방위각/시계방향)
		3. 정확한 출입구 위치(어느 방향/어느 도로쪽)
		4. 주요 랜드마크
