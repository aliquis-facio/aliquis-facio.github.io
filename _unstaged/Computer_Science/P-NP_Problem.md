# P-NP Problem

P-NP 문제는 컴퓨터 과학에서 가장 유명한 미해결 문제 중 하나이다.

핵심 질문은 다음과 같다.

> 정답이 맞는지 빠르게 확인할 수 있는 문제는, 정답을 찾는 것도 빠르게 할 수 있는가?

Clay Mathematics Institute도 P vs NP 문제를 “해답을 쉽게 확인할 수 있다면, 그 해답을 쉽게 찾을 수도 있는가?”라는 질문으로 설명한다. 또한 이 문제는 밀레니엄 문제 중 하나로, 아직 해결되지 않았다. 

---

# P란?

P는 **Polynomial Time**의 약자이다.

P에 속하는 문제는 컴퓨터가 **다항 시간 안에 풀 수 있는 문제**를 의미한다.

여기서 다항 시간이란 입력 크기 `n`에 대해 실행 시간이 다음과 같은 형태로 증가하는 경우를 말한다.

```text
n
n^2
n^3
n^5
````

예를 들어 데이터 개수가 `n`개일 때, 최대 `n^2`번 정도 계산해서 풀 수 있다면 다항 시간 알고리즘이라고 볼 수 있다.

```text id="p-class"
P = 빠르게 풀 수 있는 문제
```

여기서 “빠르게”는 사람이 느끼기에 빠르다는 뜻이 아니라, 이론적으로 **입력 크기에 대해 다항 시간 안에 해결 가능하다**는 뜻이다.

---

# P 문제 예시

대표적인 P 문제는 다음과 같다.

* 정렬된 배열에서 특정 값 찾기
* 최단 경로 찾기
* 두 수의 최대공약수 구하기
* 그래프에서 두 정점이 연결되어 있는지 확인하기
* 어떤 숫자가 소수인지 판별하기

예를 들어 배열에서 최댓값을 찾는 문제를 생각해보자.

```text id="find-max"
입력: [3, 8, 2, 10, 5]
문제: 가장 큰 값은 무엇인가?
정답: 10
```

이 문제는 배열을 한 번만 훑으면 된다.

```text id="find-max-time"
데이터 개수: n
필요한 비교 횟수: 대략 n번
시간 복잡도: O(n)
```

따라서 이 문제는 P에 속한다고 볼 수 있다.

---

# NP란?

NP는 **Nondeterministic Polynomial Time**의 약자이다.

NP에 속하는 문제는 **정답 후보가 주어졌을 때, 그 정답이 맞는지 다항 시간 안에 검증할 수 있는 문제**를 의미한다. MIT의 설명에서도 NP는 “해답을 다항 시간 안에 검증할 수 있는 문제들의 집합”으로 설명된다. ([MIT 뉴스][1])

```text id="np-class"
NP = 정답이 주어졌을 때 빠르게 확인할 수 있는 문제
```

여기서 중요한 점은 다음이다.

```text id="np-important"
NP는 반드시 빠르게 풀 수 있다는 뜻이 아니다.
NP는 정답 후보를 빠르게 검증할 수 있다는 뜻이다.
```

즉, 답을 찾는 것은 어려울 수 있지만, 누군가 답을 알려주면 그 답이 맞는지 확인하는 것은 쉬운 문제들이 NP에 속한다.

---

# NP 문제 예시

예를 들어 스도쿠 문제를 생각해보자.

스도쿠 퍼즐을 처음부터 푸는 것은 어렵다.

하지만 누군가 완성된 답안을 보여주면, 그 답이 맞는지 확인하는 것은 비교적 쉽다.

```text id="sudoku"
문제: 이 스도쿠를 완성할 수 있는가?
답 찾기: 어려움
답 검증: 쉬움
```

검증할 때는 각 행, 열, 박스에 숫자가 규칙대로 들어갔는지만 확인하면 된다.

따라서 일반화된 스도쿠 문제는 NP 문제의 예시로 자주 설명된다. 

---

# 쉽게 풀기와 쉽게 확인하기

P-NP 문제를 이해할 때 가장 중요한 차이는 다음이다.

```text id="solve-vs-verify"
풀기 = 답을 직접 찾는 것
검증 = 주어진 답이 맞는지 확인하는 것
```

예를 들어 비밀번호 찾기를 생각해보자.

```text id="password-example"
문제: 비밀번호가 무엇인가?
```

비밀번호를 모르면 가능한 값을 하나씩 시도해야 한다.

```text id="password-solving"
0000
0001
0002
...
9999
```

하지만 누군가 비밀번호가 `1234`라고 알려주면, 그 값이 맞는지 확인하는 것은 매우 쉽다.

```text id="password-verification"
입력한 비밀번호 == 실제 비밀번호?
```

즉, **찾는 것**과 **확인하는 것**은 난이도가 다를 수 있다.

P-NP 문제는 바로 이 차이를 묻는다.

---

# P와 NP의 관계

모든 P 문제는 NP에 포함된다.

왜냐하면 빠르게 풀 수 있는 문제라면, 정답이 주어졌을 때 그 정답이 맞는지 확인하는 것도 빠르게 할 수 있기 때문이다.

```text id="p-subset-np"
P ⊆ NP
```

예를 들어 어떤 문제를 빠르게 풀 수 있다면, 주어진 답이 맞는지 확인할 때도 직접 문제를 풀어서 답과 비교하면 된다.

따라서 다음은 확실하다.

```text id="certain-relation"
P는 NP의 부분집합이다.
```

하지만 아직 모르는 것은 이것이다.

```text id="unknown-relation"
P = NP 인가?
P ≠ NP 인가?
```

---

# P = NP 라는 뜻

`P = NP`라는 것은 다음을 의미한다.

```text id="p-equals-np"
정답을 빠르게 검증할 수 있는 모든 문제는
정답을 빠르게 찾을 수도 있다.
```

즉, NP에 속하는 모든 문제가 사실은 P에도 속한다는 뜻이다.

만약 `P = NP`라면 다음과 같은 의미가 된다.

```text id="p-np-impact"
어떤 문제의 답이 맞는지 빠르게 확인할 수 있다면,
그 답을 찾는 알고리즘도 빠르게 만들 수 있다.
```

이 경우 많은 어려운 문제들이 빠르게 풀릴 가능성이 생긴다.

예를 들어 다음과 같은 분야에 큰 영향을 줄 수 있다.

* 암호학
* 최적화 문제
* 인공지능
* 자동 증명
* 스케줄링
* 물류 경로 최적화
* 보안 시스템

특히 현대 암호 시스템 중 일부는 특정 계산 문제가 풀기는 어렵지만 확인은 쉽다는 성질에 의존한다.
따라서 `P = NP`가 증명되고 실제로 효율적인 알고리즘까지 발견된다면 암호학에 큰 충격을 줄 수 있다.

---

# P ≠ NP 라는 뜻

`P ≠ NP`라는 것은 다음을 의미한다.

```text id="p-not-np"
정답을 빠르게 확인할 수는 있지만,
정답을 빠르게 찾을 수는 없는 문제가 존재한다.
```

즉, NP 안에는 P에 속하지 않는 문제가 있다는 뜻이다.

많은 컴퓨터 과학자들은 `P ≠ NP`일 가능성이 높다고 생각한다. 다만 아직 수학적으로 증명되지는 않았다. 

직관적으로 말하면 다음과 같다.

```text id="p-not-np-intuition"
답을 보는 것과 답을 찾는 것은 다르다.
검산은 쉬워도 문제 풀이는 어려울 수 있다.
```

예를 들어 어려운 수학 문제의 풀이를 검토하는 것은 가능하지만, 그 풀이를 처음부터 직접 떠올리는 것은 훨씬 어렵다.

---

# NP-Complete

P-NP 문제를 이해할 때 자주 등장하는 개념이 **NP-Complete**이다.

NP-Complete는 NP 문제 중에서도 특히 중요한 문제 집합이다.

간단히 말하면 다음과 같다.

```text id="np-complete"
NP-Complete = NP 안에서 가장 어려운 축에 속하는 문제
```

조금 더 정확히 말하면, NP-Complete 문제는 다음 조건을 만족한다.

```text id="np-complete-condition"
1. 그 문제 자체가 NP에 속한다.
2. NP에 속하는 모든 문제를 그 문제로 다항 시간 안에 변환할 수 있다.
```

즉, NP-Complete 문제 하나를 빠르게 풀 수 있다면, NP에 속한 다른 문제들도 빠르게 풀 수 있게 된다.

```text id="np-complete-importance"
어떤 NP-Complete 문제가 P에 속한다면,
P = NP가 된다.
```

대표적인 NP-Complete 문제로는 SAT 문제가 있다. Cook-Levin 정리에 따르면 Boolean satisfiability problem, 즉 SAT는 NP-Complete 문제이다. 

---

# SAT 문제

SAT는 Boolean Satisfiability Problem의 약자이다.

논리식이 주어졌을 때, 그 논리식을 참으로 만들 수 있는 변수 값의 조합이 존재하는지 묻는 문제이다.

예를 들어 다음과 같은 논리식이 있다고 하자.

```text id="sat-example"
(A OR B) AND (NOT A OR C)
```

문제는 다음과 같다.

```text id="sat-question"
A, B, C에 true 또는 false를 넣어서
전체 식을 true로 만들 수 있는가?
```

만약 누군가 다음 값을 알려줬다고 하자.

```text id="sat-answer"
A = true
B = false
C = true
```

그러면 이 값들을 논리식에 넣어 전체 결과가 참인지 빠르게 확인할 수 있다.

따라서 SAT는 “답이 주어지면 검증이 쉬운 문제”이다.

그리고 SAT는 NP-Complete 문제이므로 P-NP 문제에서 매우 중요한 위치를 가진다.

---

# NP-Hard

NP-Complete와 함께 자주 나오는 개념이 **NP-Hard**이다.

NP-Hard는 NP 문제들만큼 어렵거나 그보다 더 어려운 문제를 의미한다.

```text id="np-hard"
NP-Hard = NP의 모든 문제보다 적어도 어렵다고 볼 수 있는 문제
```

NP-Hard는 반드시 NP에 속할 필요는 없다.

즉, 다음 차이가 있다.

| 구분          | 의미                                     |
| ----------- | -------------------------------------- |
| NP          | 답을 빠르게 검증할 수 있는 문제                     |
| NP-Complete | NP에 속하면서, NP 문제 중 가장 어려운 축에 있는 문제      |
| NP-Hard     | NP 문제만큼 어렵거나 더 어려운 문제. 반드시 NP에 속하지는 않음 |

정리하면 다음과 같다.

```text id="np-hard-complete"
NP-Complete는 NP이면서 NP-Hard인 문제이다.
```

# 예시로 이해하기

## 문제 1. 미로 찾기

```text id="maze"
입구에서 출구까지 갈 수 있는가?
```

미로 문제는 일반적으로 그래프 탐색으로 빠르게 풀 수 있다.

```text id="maze-class"
미로 찾기 = P 문제로 볼 수 있음
```

직접 풀기도 쉽고, 주어진 경로가 맞는지 확인하는 것도 쉽다.

---

## 문제 2. 스도쿠

```text id="sudoku2"
이 스도쿠를 완성할 수 있는가?
```

일반화된 스도쿠는 답이 주어지면 확인은 쉽지만, 답을 찾는 것은 어려울 수 있다.

```text id="sudoku-class"
스도쿠 = NP 문제의 예시
```

---

## 문제 3. 여행 경로 문제

여러 도시가 있을 때, 특정 거리 이하로 모든 도시를 방문하는 경로가 있는지 묻는 문제를 생각해보자.

```text id="tsp"
도시 A, B, C, D를 모두 방문하면서
총 이동 거리가 100 이하인 경로가 있는가?
```

누군가 경로를 알려주면 총 거리를 계산해서 100 이하인지 확인할 수 있다.

```text id="tsp-verify"
A → C → B → D
총 거리 계산
100 이하인지 확인
```

검증은 쉽다.
하지만 가능한 경로를 모두 찾는 것은 도시 수가 늘어나면 매우 어려워진다.

# P-NP 문제가 중요한 이유

P-NP 문제가 중요한 이유는 단순히 이론적인 문제가 아니기 때문이다.

만약 `P = NP`라면, 지금까지 어렵다고 생각했던 많은 문제를 효율적으로 풀 수 있을 가능성이 생긴다.

예를 들어 다음과 같은 문제들이 영향을 받을 수 있다.

```text id="importance-list"
1. 암호 해독
2. 최적 경로 탐색
3. 자동 프로그램 분석
4. 인공지능 추론
5. 수학 정리 자동 증명
6. 물류와 스케줄링 최적화
7. 단백질 구조 예측 같은 조합 최적화 문제
```

반대로 `P ≠ NP`라면, 어떤 문제들은 검증은 쉬워도 효율적으로 푸는 것은 근본적으로 어렵다는 것이 증명된다.

---

# 자주 헷갈리는 부분

## 1. NP는 “다항 시간이 아니다”라는 뜻이 아니다

NP를 “Not Polynomial”로 오해하면 안 된다.

```text id="not-np"
틀린 이해:
NP = 다항 시간이 아닌 문제
```

정확한 의미는 다음이다.

```text id="correct-np"
NP = Nondeterministic Polynomial Time
```

즉, 답이 주어졌을 때 다항 시간 안에 검증할 수 있는 문제이다.

---

## 2. P 문제도 NP에 속한다

P와 NP가 완전히 분리된 집합이라고 생각하면 안 된다.

```text id="p-in-np"
P ⊆ NP
```

P 문제는 빠르게 풀 수 있으므로 당연히 빠르게 검증할 수도 있다.

---

## 3. NP 문제라고 해서 모두 어려운 것은 아니다

NP에는 P 문제도 포함된다.

즉, NP 안에는 쉬운 문제도 있고 어려운 문제도 있다.

```text id="np-has-p"
NP 안에는 P 문제도 포함된다.
```

정말 어려운 축에 있는 문제들을 다룰 때 NP-Complete나 NP-Hard 개념이 나온다.

---

## 4. 아직 답은 모른다

현재까지 P-NP 문제는 해결되지 않았다.

즉, 다음 중 무엇이 맞는지는 아직 증명되지 않았다.

```text id="unknown"
P = NP
P ≠ NP
```

다만 많은 연구자들은 `P ≠ NP`일 가능성이 높다고 보고 있다. 

---

# 한 줄 요약

```text id="one-line"
P-NP 문제는 “정답을 빠르게 확인할 수 있는 모든 문제를 정답도 빠르게 찾을 수 있는가?”를 묻는 컴퓨터 과학의 대표적인 미해결 문제이다.
```

# 핵심 정리

P는 다항 시간 안에 풀 수 있는 문제들의 집합이다.

```text id="summary-p"
P = 빠르게 풀 수 있는 문제
```

NP는 답이 주어졌을 때 다항 시간 안에 검증할 수 있는 문제들의 집합이다.

```text id="summary-np"
NP = 빠르게 검증할 수 있는 문제
```

모든 P 문제는 NP에 속한다.

```text id="summary-subset"
P ⊆ NP
```

하지만 NP의 모든 문제가 P에 속하는지는 아직 모른다.

```text id="summary-question"
P = NP 인가?
P ≠ NP 인가?
```

NP-Complete 문제는 NP 문제 중 가장 어려운 축에 속하는 문제이다.

```text id="summary-npc"
NP-Complete 문제 하나라도 빠르게 풀 수 있으면 P = NP가 된다.
```

P-NP 문제는 암호학, 최적화, 인공지능, 알고리즘 이론 등 여러 분야에 큰 영향을 주는 중요한 미해결 문제이다.
