## 3. 버그헌팅 기본 절차

버그헌팅은 보통 다음 흐름으로 진행된다.

```text
1. 대상 범위 확인
2. 타겟 정보 수집
3. 공격 표면 식별
4. 취약점 탐색
5. 취약점 검증
6. 영향도 분석
7. 보고서 작성
8. 패치 확인
```

### 3.1. 대상 범위 확인

버그헌팅을 시작하기 전에 가장 먼저 확인해야 하는 것은 **Scope**이다.

Scope는 테스트가 허용된 대상을 의미한다.

예시:

```text
*.target-system.com
api.target-system.com
Android App v1.2.3
iOS App v2.0.1
```

Out of Scope는 테스트하면 안 되는 대상이다.

예시:

```text
사내망
직원 계정
제3자 서비스
물리적 공격
사회공학 공격
DDoS
무차별 대입 공격
```

### 3.2. 취약점 보고서 구성

좋은 버그헌팅 보고서는 다음 요소를 포함해야 한다.

| 항목    | 설명                   |
| ----- | -------------------- |
| 제목    | 취약점 핵심을 간단히 표현       |
| 요약    | 어떤 문제가 있는지 설명        |
| 영향도   | 공격자가 무엇을 할 수 있는지 설명  |
| 재현 절차 | 단계별 재현 방법            |
| 증거 자료 | 요청/응답, 화면 캡처, 로그 등   |
| 권장 조치 | 수정 방향 제시             |
| 범위 확인 | 테스트 대상이 Scope 내인지 명시 |
