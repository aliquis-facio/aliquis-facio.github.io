## 1. 버그헌팅이란

**버그헌팅(Bug Hunting)** 은 소프트웨어, 웹 서비스, 모바일 앱, API, 클라우드 환경 등에서 보안 취약점을 찾아내고, 이를 서비스 운영자에게 제보하는 활동이다.

버그헌팅의 핵심은 단순히 취약점을 찾는 것이 아니라, **허가된 범위 안에서 취약점을 발견하고 책임감 있게 보고하는 것**이다.

### 버그헌팅의 목적

* 서비스의 보안 수준 향상
* 공격자가 악용하기 전에 취약점 발견
* 기업의 보안 리스크 감소
* 사용자 개인정보 및 시스템 보호
* 보안 연구자에게 보상 또는 평판 제공

### 버그헌팅 대상 예시

* 웹 애플리케이션
* 모바일 앱
* REST API / GraphQL API
* 클라우드 스토리지
* 인증·인가 시스템
* 관리자 페이지
* 서브도메인
* 공개 GitHub 저장소
* 배포 서버 및 설정 파일

### 버그헌팅에서 중요한 요소

| 요소                     | 설명              |
| ---------------------- | --------------- |
| Scope                  | 테스트가 허용된 대상 범위  |
| Out of Scope           | 테스트하면 안 되는 대상   |
| Rules of Engagement    | 테스트 시 지켜야 하는 규칙 |
| Impact                 | 취약점이 실제로 미치는 영향 |
| Reproducibility        | 취약점 재현 가능성      |
| Responsible Disclosure | 책임 있는 제보        |
