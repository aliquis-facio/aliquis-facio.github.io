## 2. 윤리적 해커란

**윤리적 해커(Ethical Hacker)**는 시스템 소유자의 허가를 받고 보안 취약점을 찾는 사람이다.

공격자와 같은 기술을 사용하더라도, 목적과 절차가 다르다.

### 윤리적 해커와 악의적 해커 비교

| 구분     | 윤리적 해커        | 악의적 해커            |
| ------ | ------------- | ----------------- |
| 허가 여부  | 허가된 범위에서 테스트  | 허가 없이 침입          |
| 목적     | 보안 강화         | 정보 탈취, 금전적 이익, 파괴 |
| 보고 여부  | 취약점 보고        | 은닉 또는 악용          |
| 법적 책임  | 규칙 준수 시 보호 가능 | 불법 행위             |
| 데이터 처리 | 최소 접근, 비공개 유지 | 유출 또는 판매          |

### 윤리적 해커가 지켜야 할 원칙

* 허가된 대상만 테스트한다.
* 서비스 장애를 유발하지 않는다.
* 실제 사용자 데이터를 열람하거나 저장하지 않는다.
* 취약점 증명은 최소한으로 수행한다.
* 발견 내용을 제3자에게 공개하지 않는다.
* 취약점 보고서에는 재현 절차와 영향도를 명확히 작성한다.
* 보상보다 서비스 보호를 우선한다.
