* 도서 대출
* 도서 반납

즉, **상태 기반 로직과 예외 처리**를 본다.

## 문제 1. 모든 저자 목록 가져오기

대상 파일:

`author/models.py`

원본 코드:

```python
@classmethod
def get_all_authors(cls) -> QuerySet['Author']:
    """
    MIDTERM EXAM 1 : 아래 코드가 정상 작동하도록 고치세요.

    저장된 모든 작가의 목록을 가져오는 함수
    :return: 저장된 모든 작가의 목록
    """
    return None
```

이 함수는 저자 목록 페이지에서 호출된다.

`author/views.py`

```python
def author_list(request):
    context = {
        'authors': Author.get_all_authors()
    }
    return render(request, 'author/list.html', context)
```

즉, 여기서는 **모든 Author 객체를 반환**하면 된다.

수정 코드:

```python
@classmethod
def get_all_authors(cls) -> QuerySet['Author']:
    return cls.objects.all()
```

### 왜 이렇게 고치는가

* `cls`는 `Author` 클래스 자체다.
* `cls.objects.all()`은 DB에 저장된 모든 저자 레코드를 반환한다.
* 이 결과가 템플릿으로 넘어가서 목록으로 렌더링된다.

## 문제 2. 특정 id의 저자 한 명 가져오기

대상 파일:

`author/models.py`

원본 코드:

```python
@classmethod
def get_single_author_by_id(cls, author_id: int) -> 'Author':
    """
    MIDTERM EXAM 2 : 아래 코드가 정상 작동하도록 고치세요.

    id가 author_id인 작가 객체 하나를 반환한다.
    :raise django.http.Http404: 해당 id를 가지는 Author가 존재하지 않는 경우
    :return: id가 author_id인 Author 객체
    """
    return None
```

이 함수는 저자 상세 페이지에서 호출된다.

`author/views.py`

```python
def author_detail(request, author_id: int):
    context = {}

    author = Author.get_single_author_by_id(author_id)
    context['author'] = author

    published_books = author.get_published_books()
    context['published_books'] = published_books

    return render(request, 'author/detail.html', context)
```

### 핵심

문제 설명에 “없으면 `Http404` 발생”이라고 되어 있으므로,
가장 자연스러운 해법은 `get_object_or_404`다.

수정 코드:

```python
from django.shortcuts import get_object_or_404
```

```python
@classmethod
def get_single_author_by_id(cls, author_id: int) -> 'Author':
    return get_object_or_404(cls, id=author_id)
```

### 왜 이렇게 고치는가

* 단순히 `cls.objects.get(id=author_id)`를 써도 조회는 가능하다.
* 하지만 존재하지 않으면 `DoesNotExist`가 발생한다.
* 문제는 `Http404`를 요구하므로 `get_object_or_404`가 더 적합하다.

## 문제 3. 특정 저자의 출간 도서 목록 가져오기

대상 파일:

`author/models.py`

원본 코드:

```python
def get_published_books(self) -> QuerySet[Book]:
    """
    MIDTERM EXAM 3 : 아래 코드가 정상 작동하도록 고치세요.

    해당 저자가 쓴 모든 도서의 목록을 반환한다.
    :return: 해당 저자가 쓴 모든 도서의 목록
    """
    return None
```

`Book` 모델을 보면:

```python
author = models.ForeignKey('author.Author', ...)
```

즉, 책은 저자를 외래키로 참조한다.
따라서 현재 저자 `self`에 연결된 책들을 조회하면 된다.

수정 코드:

```python
def get_published_books(self) -> QuerySet[Book]:
    return Book.objects.filter(author=self)
```

### 왜 이렇게 고치는가

* 현재 메서드는 `Author` 인스턴스 메서드다.
* 따라서 `self`는 “지금 상세 페이지에서 보고 있는 저자”다.
* 그 저자를 `author`로 가지는 `Book`들을 필터링하면 된다.

# 4. `author/models.py` 최종 코드

```python
from django.db import models
from django.db.models import QuerySet
from django.shortcuts import get_object_or_404

from book.models import Book


class Author(models.Model):
    """
    책의 저자 모델
    """
    name = models.CharField(max_length=20, verbose_name='이름')
    description = models.TextField(verbose_name='저자 소개')

    class Meta:
        db_table = 'authors'
        verbose_name = '저자'
        verbose_name_plural = '저자 목록'

    @classmethod
    def get_all_authors(cls) -> QuerySet['Author']:
        return cls.objects.all()

    @classmethod
    def get_single_author_by_id(cls, author_id: int) -> 'Author':
        return get_object_or_404(cls, id=author_id)

    def get_published_books(self) -> QuerySet[Book]:
        return Book.objects.filter(author=self)
```

# 5. 문제 4. 모든 도서를 출간일 내림차순으로 가져오기

대상 파일:

`book/models.py`

원본 코드:

```python
@classmethod
def get_all_books_order_by_published_at_desc(cls) -> QuerySet['Book']:
    """
    MIDTERM EXAM 4 : 아래 코드가 정상 작동하도록 고치세요.

    저장된 모든 도서의 목록을 출간일 내림차순으로 가져오는 함수
    :return: 저장된 모든 도서의 목록 (출간일 내림차순 정렬 적용)
    """
    return None
```

이 함수는 `book/views.py`에서 정렬 옵션과 연결된다.

```python
def book_list(request):
    context = {}

    sort = request.GET.get('sort')
    if sort == 'published_at_desc':
        books = Book.get_all_books_order_by_published_at_desc()
    else:
        books = Book.get_all_books()

    context['books'] = books

    return render(request, 'book/list.html', context)
```

즉, `?sort=published_at_desc`가 들어오면
출간일 기준 최신순으로 도서를 보여줘야 한다.

수정 코드:

```python
@classmethod
def get_all_books_order_by_published_at_desc(cls) -> QuerySet['Book']:
    return cls.objects.all().order_by('-published_at')
```

### 왜 이렇게 고치는가

* `order_by('-published_at')`에서 `-`는 내림차순을 뜻한다.
* 따라서 가장 최근 출간 도서가 먼저 나온다.

# 6. 문제 5. 주석 해제 후 발생하는 `NoReverseMatch` 해결

이 문제는 단순 문법보다 **디버깅 문제**에 가깝다.

먼저 `templates/book/detail.html`을 보면:

```django
<!-- MIDTERM EXAM 5~7 : 5번부터 7번 문제를 풀기 전에, 아래 주석을 해제해 주세요. -->
<!-- MIDTERM EXAM 5 : 아래 주석을 해제했을 때 발생하는 에러를 해결하세요. -->
{# {% include 'book/borrow-histories.html' %} #}
```

이 주석을 해제하면 다음처럼 된다.

```django
{% include 'book/borrow-histories.html' %}
```

그런데 그러면 `borrow-histories.html` 내부 URL 태그가 실행된다.

원본 코드:

```django
{% if book.is_checked_out %}
    <button class="ui button disabled">대출</button>
    <a href="{% url 'book_borrow:give-back' book.id %}">
        <button class="ui button">반납</button>
    </a>
{% else %}
    <a href="{% url 'book_borrow:borrow' book.id %}">
        <button class="ui button" data-tooltip="오늘로부터 7일">대출</button>
    </a>
    <button class="ui button disabled">반납</button>
{% endif %}
```

템플릿은 `book_borrow:borrow`, `book_borrow:give-back`라는
**namespace가 있는 URL 이름**을 기대한다.

그런데 `book_borrow/urls.py`를 보면:

```python
from django.urls import path

from . import views


urlpatterns = [
    path('borrow', views.book_borrow, name='borrow'),
    path('give-back', views.book_give_back, name='give-back'),
]
```

여기엔 `app_name`이 없다.

즉, namespace가 등록되지 않아서
템플릿의 `{% url 'book_borrow:borrow' book.id %}`가 실패한다.

수정 코드:

```python
from django.urls import path

from . import views

app_name = 'book_borrow'

urlpatterns = [
    path('borrow', views.book_borrow, name='borrow'),
    path('give-back', views.book_give_back, name='give-back'),
]
```

### 왜 이렇게 고치는가

* `app_name = 'book_borrow'`가 있어야
* URL namespace `book_borrow:`가 활성화된다.
* 그러면 템플릿의 URL reverse가 정상 동작한다.

# 7. 문제 6. 도서 대출 기능 구현

대상 파일:

`book/models.py`

원본 코드:

```python
def borrow(self, user_id: int) -> 'BorrowHistory':
    """
    MIDTERM EXAM 6 : 아래 명세에 맞게 코드를 완성하세요.

    해당 도서를 대출 처리한다.
    :param user_id: 대출을 요청한 사용자의 id
    :raises AlreadyCheckedOutBookException: 이미 대출 중인 도서에 대출을 요청한 경우
    :return: 생성된 대출 이력
    """
    return None
```

이 함수는 `book_borrow/views.py`에서 호출된다.

```python
def book_borrow(request, book_id: int):
    book = Book.get_single_book_by_id(book_id)

    if request.method == 'POST':
        borrow_user_id = int(request.POST['borrow-user'])
        borrowed_history = book.borrow(borrow_user_id)
        return redirect('book:detail', book.id)
```

즉, 실제 POST 요청이 들어오면
`Book.borrow()`가 비즈니스 로직을 담당한다.

### 먼저 확인할 수 있는 보조 메서드

원본 코드에는 이미 이런 메서드들이 있다.

```python
def is_checked_out(self) -> bool:
    borrow_histories = self.get_borrow_histories_order_by_borrowed_at_desc()
    return borrow_histories.filter(Q(returned_at=None)).exists()
```

현재 대출 중인지 확인해준다.

또 `BorrowHistory` 모델에는:

```python
@staticmethod
def get_seven_days_later(given_dt: datetime) -> datetime:
    return given_dt + timedelta(days=7)
```

반납 기한을 계산해준다.

즉, 구현 흐름은 다음이다.

1. 이미 대출 중인지 확인
2. 대출 중이면 예외 발생
3. 아니면 사용자 조회
4. BorrowHistory 생성
5. 반납기한 = 현재 시각 + 7일
6. 생성된 이력 반환

필요한 import:

```python
from datetime import datetime
from django.shortcuts import get_object_or_404
from django.contrib.auth.models import User

from book_borrow.models import BorrowHistory
from book_borrow.exceptions import AlreadyCheckedOutBookException
```

수정 코드:

```python
def borrow(self, user_id: int) -> 'BorrowHistory':
    if self.is_checked_out():
        raise AlreadyCheckedOutBookException()

    user = get_object_or_404(User, id=user_id)

    borrow_history = BorrowHistory.objects.create(
        book=self,
        user=user,
        return_deadline=BorrowHistory.get_seven_days_later(datetime.now()),
    )
    return borrow_history
```

### 왜 이렇게 고치는가

* `self.is_checked_out()`으로 현재 상태를 먼저 검사해야 한다.
* 이미 대출 중인데 또 대출하면 안 되므로 예외를 발생시킨다.
* 사용자 id는 POST로 넘어오기 때문에 DB에서 실제 `User` 객체를 조회해야 한다.
* 새 대출 기록을 `BorrowHistory.objects.create()`로 만든다.
* `book=self`이므로 현재 책에 대한 대출 기록이 된다.
* 반납 기한은 오늘 기준 7일 뒤다.

# 8. 문제 7. 도서 반납 기능 구현

대상 파일:

`book/models.py`

원본 코드:

```python
def give_back(self) -> 'BorrowHistory':
    """
    MIDTERM EXAM 7 : 아래 명세에 맞게 코드를 완성하세요.

    해당 도서를 반납 처리한다.
    :raises book_borrow.exceptions.BookHasNoBorrowHistory: 대출 이력이 없는 도서에 반납을 요청한 경우
    :raises book_borrow.exceptions.BookIsNotCheckedOutException: 이미 반납된 도서에 반납을 요청한 경우
    :return: 반납 처리된 대출 이력
    """
    return None
```

이 함수는 `book_borrow/views.py`에서 호출된다.

```python
def book_give_back(request, book_id: int):
    book = Book.get_single_book_by_id(book_id)
    returned_history = book.give_back()
    return redirect('book:detail', book.id)
```

### 이미 준비된 보조 메서드

`Book` 모델에는:

```python
def get_latest_borrow_history(self) -> Optional['BorrowHistory']:
    borrow_histories = self.get_borrow_histories_order_by_borrowed_at_desc()
    return borrow_histories.first()
```

가 있다.

또 `BorrowHistory`에는:

```python
def is_returned(self):
    return self.returned_at is not None

def give_back(self) -> None:
    self.returned_at = datetime.now()
    self.save()
```

가 있다.

즉, 구현 흐름은 다음이다.

1. 가장 최근 대출 이력을 가져온다.
2. 이력이 없으면 `BookHasNoBorrowHistory`
3. 이력은 있지만 이미 반납 상태면 `BookIsNotCheckedOutException`
4. 아니면 그 이력에 대해 `give_back()` 호출
5. 반납 처리된 이력 반환

필요한 import:

```python
from book_borrow.exceptions import (
    BookHasNoBorrowHistory,
    BookIsNotCheckedOutException,
)
```

수정 코드:

```python
def give_back(self) -> 'BorrowHistory':
    latest_borrow_history = self.get_latest_borrow_history()

    if latest_borrow_history is None:
        raise BookHasNoBorrowHistory()

    if latest_borrow_history.is_returned():
        raise BookIsNotCheckedOutException()

    latest_borrow_history.give_back()
    return latest_borrow_history
```

### 왜 이렇게 고치는가

* 대출 이력이 아예 없으면 반납할 대상 자체가 없다.
* 최근 대출 이력이 이미 반납된 상태면 현재는 대출 중이 아니다.
* 이 두 경우를 구분해서 예외를 던져야 한다.
* 정상 상태일 때만 실제 반납 메서드를 호출한다.

# 9. `book/models.py` 최종 코드

```python
from datetime import datetime
from typing import Optional

from django.contrib.auth.models import User
from django.db import models
from django.db.models import QuerySet, Q
from django.shortcuts import get_object_or_404

from book_borrow.models import BorrowHistory
from book_borrow.exceptions import (
    AlreadyCheckedOutBookException,
    BookHasNoBorrowHistory,
    BookIsNotCheckedOutException,
)


class Book(models.Model):
    """
    책 모델
    """
    author = models.ForeignKey('author.Author', null=False, blank=False, on_delete=models.CASCADE, verbose_name='저자')
    title = models.CharField(max_length=50, verbose_name='제목')
    published_at = models.DateField(verbose_name='출간일')

    class Meta:
        db_table = 'books'
        verbose_name = '도서'
        verbose_name_plural = '도서 목록'

    @classmethod
    def get_all_books(cls) -> QuerySet['Book']:
        return cls.objects.all()

    @classmethod
    def get_all_books_order_by_published_at_desc(cls) -> QuerySet['Book']:
        return cls.objects.all().order_by('-published_at')

    @classmethod
    def get_single_book_by_id(cls, book_id: int) -> 'Book':
        return get_object_or_404(cls, id=book_id)

    def get_borrow_histories_order_by_borrowed_at_desc(self) -> QuerySet[BorrowHistory]:
        return BorrowHistory.objects.filter(book=self).order_by('-borrowed_at')

    def get_latest_borrow_history(self) -> Optional['BorrowHistory']:
        borrow_histories = self.get_borrow_histories_order_by_borrowed_at_desc()
        return borrow_histories.first()

    def is_checked_out(self) -> bool:
        borrow_histories = self.get_borrow_histories_order_by_borrowed_at_desc()
        return borrow_histories.filter(Q(returned_at=None)).exists()

    def borrow(self, user_id: int) -> 'BorrowHistory':
        if self.is_checked_out():
            raise AlreadyCheckedOutBookException()

        user = get_object_or_404(User, id=user_id)

        borrow_history = BorrowHistory.objects.create(
            book=self,
            user=user,
            return_deadline=BorrowHistory.get_seven_days_later(datetime.now()),
        )
        return borrow_history

    def give_back(self) -> 'BorrowHistory':
        latest_borrow_history = self.get_latest_borrow_history()

        if latest_borrow_history is None:
            raise BookHasNoBorrowHistory()

        if latest_borrow_history.is_returned():
            raise BookIsNotCheckedOutException()

        latest_borrow_history.give_back()
        return latest_borrow_history
```

# 10. 이 중간고사가 왜 이런 구조로 출제되었는가

이 시험은 그냥 “Django 문법을 아는지”만 보지 않는다.
실제로는 아래 능력을 한 번에 확인한다.

## ORM 조회 능력

* `all()`
* `filter()`
* `order_by()`

## 객체 단건 조회와 404 처리

* `get_object_or_404`

## 모델 간 관계 이해

* Author → Book
* Book → BorrowHistory
* BorrowHistory → User

## URL reverse와 namespace

* `{% url 'book_borrow:borrow' book.id %}`
* `app_name = 'book_borrow'`

## 상태 기반 비즈니스 로직

* 이미 대출 중인가
* 대출 이력이 없는가
* 이미 반납된 상태인가

즉, 이 시험은 **조회 → 디버깅 → 상태 로직** 순서로 난이도를 올리는 구조다.
