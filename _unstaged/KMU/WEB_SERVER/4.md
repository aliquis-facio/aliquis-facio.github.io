## 2. 화면 예쁘게 꾸미기

### 2-1. 스타일시트 적용

Django에서 CSS를 적용하려면 먼저 정적 파일(static)을 연결해야 한다.

* `settings.py`에 static 디렉터리 위치를 추가
* 프로젝트 루트에 `static` 디렉터리 생성
* `style.css` 작성
* 템플릿에서 `{% load static %}`와 `{% static 'style.css' %}`로 연결 

예시로는 답변 등록 화면에서:

* `textarea`를 100% 너비로 확장
* 버튼 위에 `margin-top` 추가

같은 스타일 적용이 소개된다. 

---

### 2-2. CSS 개요

CSS는 웹 문서의 **디자인 요소**를 담당한다.

* HTML: 문서의 구조와 내용
* CSS: 색상, 크기, 위치, 배치 등 표현 담당 

CSS를 사용하는 이유는 다음과 같다.

* 내용과 디자인을 분리할 수 있음
* 디자인은 유지한 채 내용만 변경 가능
* 반응형 디자인 구현 가능
* 같은 구조에 서로 다른 테마 적용 가능 

#### 기본 문법

CSS는 다음 두 요소로 구성된다.

* **선택자(selector)**: 스타일을 적용할 대상
* **선언부(declaration)**: 속성과 값

예:

```css
h1 {
  color: red;
  font-size: 20px;
}
```

---

### 2-3. 박스 모델

페이지의 모든 요소는 기본적으로 박스 형태로 취급된다. 

* **content**: 실제 내용
* **padding**: 내용과 테두리 사이 안쪽 여백
* **border**: 테두리
* **margin**: 바깥쪽 여백

#### 공백 주기

공백 개념은 크게 두 가지로 정리된다.

* `margin`: 요소 바깥 여백
* `padding`: 요소 안쪽 여백 

Bootstrap 표기 규칙도 함께 소개된다.

* `m`: margin
* `p`: padding
* `t`: top
* `b`: bottom
* `l`: left
* `r`: right
* `x`: left/right
* `y`: top/bottom 

예:

* `m-3`
* `mx-3`
* `py-3`

---

### 2-4. 레이아웃과 포지션

레이아웃은 화면 요소를 배치하는 방식이다. 자료에서는 특히 다음 속성이 중요하다고 설명한다. 

* `position`
* `float`
* `display`

#### Grid System

Bootstrap의 Grid System은 화면을 최대 12열로 분할한다. 예를 들면:

* `col-12`: 한 줄 전체
* `col-6 col-6`: 2등분
* `col-4 col-4 col-4`: 3등분 

#### Position

대표적인 position 속성은 다음과 같다.

* `static`: 기본 배치
* `relative`: 원래 위치 기준 이동
* `absolute`: 기준 요소를 따라 절대 위치 지정
* `fixed`: 화면에 고정
* `z-index`: 요소의 겹침 순서 결정 

---

## 3. Bootstrap으로 화면 꾸미기

### 3-1. Bootstrap 시작

Bootstrap은 반응형 웹 UI를 빠르게 만들 수 있는 프레임워크다.

적용 시 필요한 요소:

* `<meta name="viewport">`
* Bootstrap CSS 링크
* Bootstrap JS 번들 스크립트 

자료에서는 CDN을 통해 Bootstrap을 연결하는 예시를 보여주며, CSS는 `<head>`에, JS는 `</body>` 직전에 넣는 구조를 설명한다. 

---

### 3-2. Bootstrap 주요 구성 요소

#### Container

* `.container`: 반응형 고정 폭 컨테이너
* `.container-fluid`: 전체 폭 컨테이너 

#### Grid System

* Flexbox 기반
* 최대 12열
* 다양한 화면 크기별 클래스 제공 (`col-`, `col-sm-`, `col-md-` 등) 

#### Jumbotron

강조할 내용을 크게 보여주는 박스 형태 컴포넌트다. 

#### Button Styles

다양한 버튼 스타일이 제공된다.

* `btn-primary`
* `btn-secondary`
* `btn-success`
* `btn-warning`
* `btn-danger` 등 

#### Button Group

여러 버튼을 한 줄로 묶어 표현할 수 있다. 

#### Badges

추가 정보를 작은 라벨 형태로 표시하는 컴포넌트다. 

#### Pagination

페이지 이동 UI를 만들 수 있다.

* `.pagination`
* `.page-item`
* `.page-link` 

#### Card

내용을 카드 형태로 표현하는 컴포넌트다. header, body, footer, overlay 등을 조합할 수 있다.

## 5. 표준 HTML과 템플릿 상속

### 표준 HTML 구조

표준 HTML 문서는 다음 요소를 포함해야 한다.

* `html`
* `head`
* `body`

그리고 CSS 파일은 `head` 안에 위치해야 한다. `head`에는 `meta`, `title`도 포함된다. 

### Django 템플릿 상속

Django는 템플릿 상속 기능을 제공한다.

* 부모 템플릿: `base.html`
* 자식 템플릿: `question_list.html`, `question_detail.html` 등 

부모 템플릿에는 공통 틀을 두고:

```django
{% block content %}
{% endblock %}
```

자식 템플릿에서 해당 영역만 채운다. 

#### 장점

* 공통 HTML 중복 제거
* 유지보수 용이
* 전체 레이아웃 통일 가능

#### 추가 포인트

기존 `style.css`는 삭제하지 않고 **내용만 비운다**. 이후 Bootstrap으로 처리하기 어려운 부분을 위해 남겨둔다.

## 6. 질문 등록 기능 만들기

### 6-1. 질문 등록 버튼

질문 목록 화면에 질문 등록 버튼을 추가한다.

* URL: `{% url 'pybo:question_create' %}`
* Bootstrap 클래스: `btn btn-primary` 

그리고 `pybo/urls.py`에 URL 매핑을 추가한다. 

---

### 6-2. QuestionForm 작성

질문 등록에는 Django Form을 사용한다.

* `forms.ModelForm` 상속
* 내부 `Meta` 클래스 필수
* 사용할 모델과 필드 지정 

예시 개념:

* model = `Question`
* fields = `['subject', 'content']`

---

### 6-3. question_create 뷰 처리

`question_create(request)`는 GET과 POST를 나눠 처리한다.

#### GET

* 빈 폼 생성
* 질문 등록 화면 렌더링

#### POST

* 입력값 바인딩
* `form.is_valid()` 검증
* 저장 후 질문 목록으로 리다이렉트 

#### `commit=False`를 사용하는 이유

자료에서는 아래 코드의 의미를 설명한다.

```python
question = form.save(commit=False)
```

이유는 `Question` 모델의 `create_date`가 폼 입력에 없기 때문이다.
바로 저장하면 오류가 날 수 있으므로:

1. 임시 객체 생성
2. `create_date` 등 추가 값 설정
3. 최종 저장

순서로 처리한다. 

---

### 6-4. question_form.html

처음에는 다음처럼 자동 렌더링을 사용한다.

```django
{{ form.as_p }}
```

의미:

* 폼 필드를 `<p>` 태그 기준으로 자동 생성 

자료에서는 다음 렌더링 옵션도 함께 설명한다.

* `form`
* `form.as_p`
* `form.as_table`
* `form.as_ul` 

즉, 폼의 각 필드를 어떤 HTML 구조로 감쌀지 선택할 수 있다.

---

### 6-5. 자동 폼의 한계와 수작업 폼

`{{ form.as_p }}`는 편리하지만 Bootstrap 세부 적용이 어렵다. 그래서 이후에는 제목과 내용 입력 HTML을 직접 작성하는 방식으로 바꾼다. 

#### 수작업 폼의 장점

* Bootstrap 클래스 세밀 적용 가능
* 오류 메시지 위치를 자유롭게 제어 가능
* 입력값 유지 처리 가능

#### 오류 처리

예를 들어 필수 입력을 빠뜨리면 오류 메시지를 표시하고, 이미 입력한 값은 유지되도록 처리한다.

## 9. Django 1만 명 동시 접속 웹서버 아키텍처

### 9-1. runserver의 한계

자료에서는 Django 개발 서버 `runserver`가 한 번에 1개 요청만 처리한다고 설명한다. 따라서 동시 접속자가 많아지면 대기열이 길어질 수 있다. 

---

### 9-2. 6계층 + α 구조

#### 1계층: CDN

정적 파일을 사용자 가까운 서버에서 제공한다.

* CSS, JS, 이미지 전달
* 서버 부하 감소
* 대표 예: Cloudflare, CloudFront 

#### 2계층: Nginx

리버스 프록시 역할을 한다.

* 로드 밸런싱
* 정적 파일 서빙
* SSL 종료
* Rate Limiting
* 버퍼링 처리 

`/static/`은 Nginx가 직접 처리하고, `/api/`, `/admin/`, `/health/`는 Django로 넘기는 흐름도 설명한다. 

#### 3계층: Gunicorn + Uvicorn

* Gunicorn: 프로세스 매니저, 워커 관리
* Uvicorn: 비동기 워커, I/O 대기 중 다른 요청 처리 가능 

워커 수 공식 예시:

* CPU 코어 × 2 + 1 

#### 4계층: Django

* URL 매핑
* 비즈니스 로직
* DB 조회 수행 

#### 5계층: Redis

* 캐시 서버
* DB보다 빠른 응답
* 자주 조회되는 데이터를 캐싱 

자료에서는 DB 조회가 10~50ms, Redis 조회가 0.1~1ms 수준이라고 설명한다. 

#### 6계층: PostgreSQL

* 실서비스용 관계형 DB
* 읽기/쓰기 동시 처리 가능
* SQLite보다 동시성에 유리 

#### +α 계층: Celery

* 이메일 발송
* 이미지 리사이즈
* PDF 생성 등
* 무거운 작업을 비동기로 처리 

---

### 9-3. RPS와 부하 테스트

RPS는 초당 처리 요청 수다.

자료에서 제시한 예시:

* 100명: 평균 응답 약 29ms, 실패율 1%, RPS 48.6
* 500명: 평균 응답 약 58ms, 실패율 38%, RPS 234.3 

현업 기준 예시도 제시된다.

* 소규모 블로그: 10~50
* 중소 쇼핑몰: 100~500
* 대형 서비스: 1,000~10,000
* 대규모 글로벌 서비스: 100,000+ 

---

### 9-4. Locust

Locust는 Python 기반 부하 테스트 도구다.

설정 항목:

* Number of users
* Spawn rate
* Host

확인 지표:

* Statistics: RPS, 응답 시간, 실패율
* Charts: 시간에 따른 변화
* Failures: 오류 원인 

---

### 9-5. Nginx 로드 밸런싱 방식

* **Round Robin**: 순서대로 분배
* **Least Connections**: 연결 수가 가장 적은 서버로 분배
* **IP Hash**: 같은 IP는 항상 같은 서버로 보냄
