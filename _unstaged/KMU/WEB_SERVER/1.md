# 1주차 강의자료 정리

## 1. 장고란?

장고는 **웹 프로그램을 쉽고 빠르게 만들 수 있게 도와주는 웹 프레임워크**다.

* **튼튼한 웹 프레임워크**: 기본적인 보안 공격을 막아주는 기능이 포함됨
* **여러 기능 제공**: 웹 프로그램 개발에 필요한 도구와 기능이 대부분 준비되어 있음
* **개발 생산성 높음**: 스타터 키트처럼 빠르게 웹 서비스를 만들 수 있음 

---

## 2. 파이썬 설치

### 윈도우 기준

파이썬 설치 시 핵심은 다음 두 가지다.

1. **최신 파이썬 설치 파일 다운로드**
2. 설치 과정에서 **파이썬 경로(PATH) 추가 옵션 체크**

설치 후에는 명령 프롬프트에서 버전을 확인한다.
자료에서는 만약 **“찾을 수 없는 명령”** 오류가 발생하면 삭제 후 다시 설치하라고 안내한다. 

---

## 3. 가상환경(virtual environment)

가상환경은 **프로젝트마다 독립된 Python 실행 환경을 만드는 것**이다.
즉, Python과 라이브러리를 프로젝트별로 분리해서 관리하는 공간이다. 

### 왜 필요한가?

한 컴퓨터에서 여러 프로젝트를 개발하면 프로젝트마다 필요한 패키지 버전이 다를 수 있다.
예를 들어 자료에서는 **django-restframework가 Django 4.2 이상을 요구하는데, 현재 3.1.3이 설치되어 있으면 충돌이 발생할 수 있다**고 설명한다.
즉, 가상환경은 **패키지 버전 충돌을 방지하기 위해 필요**하다. 

### venv 방식

```bash
python3 -m venv myenv
source myenv/bin/activate
```

### conda 방식

```bash
conda create -n myenv python=3.11
conda activate myenv
```

## 4. 장고 개발 환경 준비

### 윈도우

* `C:/venvs` 같은 디렉터리를 만든다.
* `venv` 모듈로 `mysite` 가상환경을 만든다.
* 가상환경 진입 후 장고를 설치한다. 

### 우분투

* `python3-venv` 모듈 설치
* `python3 -m venv mysite` 로 가상환경 생성
* `source .../bin/activate` 로 진입
* `deactivate` 로 종료 

### 장고 설치

가상환경 안에서 설치해야 하며, 자료 예시는 다음과 같다.

```bash
pip install django==3.1.3
```

또한 `pip` 자체도 최신 버전으로 업그레이드할 수 있다.

```bash
python -m pip install --upgrade pip
```

## 5. 장고 프로젝트 생성

장고 프로젝트를 만들기 전에는 **프로젝트 루트 디렉터리**를 먼저 준비해야 한다.
그 안에서 가상환경에 진입한 뒤 프로젝트를 생성한다. 

### 기본 생성 명령

```bash
django-admin startproject config .
```

여기서 마지막의 `.` 는
**현재 디렉터리를 프로젝트 디렉터리로 사용하겠다**는 의미다.

### 생성 후 주요 파일

* `manage.py`
* `config/`

  * `settings.py`
  * `urls.py`
  * `asgi.py`
  * `wsgi.py`
  * `__init__.py`

## 6. 개발 서버 실행

프로젝트 생성 후 개발 서버는 다음 명령으로 실행한다.

```bash
python3 manage.py runserver
```

브라우저에서 아래 주소로 접속하면 장고 기본 페이지를 확인할 수 있다.

```text
127.0.0.1:8000
```

개발 서버는 **현재 실행 중인 컴퓨터에서만 접속 가능**하며, 종료하려면 `Ctrl + C` 를 사용한다.

## 9. PyCharm 설정

PyCharm Community 버전을 설치한 뒤 `C:/projects/mysite` 프로젝트를 연다.
이후 해야 할 핵심 설정은 다음과 같다.

### 인터프리터 설정

* `File → Settings`
* `Project: mysite → Project Interpreter`
* 현재 Python 인터프리터를 **가상환경의 Python 경로**로 변경 

### settings.py 수정

* `LANGUAGE_CODE: en-us → ko-kr`
* `TIME_ZONE: UTC → Asia/Seoul`

이렇게 하면 개발 서버를 **한국어/한국 시간대 기준**으로 실행할 수 있다.

## 13. Celery 예제를 통한 비판적 사고

**비동기 처리와 백그라운드 작업의 필요성**을 설명한다.

### 문제 상황

회원가입 후 인증 이메일을 보내야 하고, 이메일 1건 전송에 5초가 걸린다고 가정한다.
만약 10,000건이 발생하면:

* `10000 × 5초 = 약 50000초`
* 약 **13시간 이상 소요**
* 요청 응답이 느려짐
* CPU 사용량 증가
* 사용자 대기 시간 증가
* 서버가 멈출 수도 있음 

### Celery 사용 시

* Django는 작업을 직접 처리하지 않고 **Queue에 넣음**
* `Django → Redis Queue → Celery Worker` 구조로 처리
* Worker 여러 개가 동시에 작업 가능
* 즉, **비동기 처리**, **작업 큐**, **백그라운드 작업**으로 문제를 완화할 수 있다.
