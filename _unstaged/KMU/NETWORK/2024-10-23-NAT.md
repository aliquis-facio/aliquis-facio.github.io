# NAT

NAT(Network Address Translation)는 내부 네트워크의 사설 IP를 외부 네트워크와 통신할 수 있도록 변환해주는 방식이다.

가상머신(VM) 환경에서 NAT를 사용하면 VM은 호스트 PC와는 다른 사설 네트워크 대역을 사용하고, 외부로 나갈 때는 호스트 PC를 통해 통신한다.

```text
VM 내부 네트워크 → Host NAT → 외부 네트워크
````

예를 들어 호스트 PC가 다음 IP를 사용한다고 하자.

```text
Host IP: 192.168.0.10
```

이때 NAT 네트워크는 호스트와 다른 대역을 사용하는 것이 좋다.

```text
Host Network: 192.168.0.0/24
NAT Network: 192.168.10.0/24
```

---

# NAT 네트워크 대역을 다르게 해야 하는 이유

NAT 네트워크는 호스트와 VM이 같은 IP 대역을 사용하면 문제가 발생할 수 있다.

예를 들어 호스트가 다음 네트워크에 있다고 하자.

```text
Host IP: 192.168.0.10
Host Network: 192.168.0.0/24
```

그런데 VM의 NAT 네트워크도 다음처럼 설정하면 충돌 가능성이 생긴다.

```text
VM NAT Network: 192.168.0.0/24
VM IP: 192.168.0.20
```

이 경우 호스트 입장에서는 `192.168.0.x` 대역이 실제 물리 네트워크인지, VM NAT 내부 네트워크인지 구분하기 어려워진다.

즉, 동일한 IP 대역이 두 곳에서 사용되기 때문에 라우팅이 꼬일 수 있다.

```text
192.168.0.0/24 → 실제 공유기 네트워크?
192.168.0.0/24 → VM NAT 네트워크?
```

네트워크 장치는 같은 IP 대역 안에서 어떤 장치로 데이터를 보내야 하는지 명확히 판단하기 어렵다.

따라서 NAT를 사용할 때는 호스트 네트워크와 VM NAT 네트워크의 대역을 다르게 설정해야 한다.

---

# 잘못된 NAT 설정 예시

```text
Host IP: 192.168.0.10
Host Network: 192.168.0.0/24

VM NAT Network: 192.168.0.0/24
VM IP: 192.168.0.100
```

이 설정은 권장되지 않는다.

호스트와 VM NAT 네트워크가 같은 대역을 사용하기 때문이다.

이 경우 다음과 같은 문제가 생길 수 있다.

* NAT 네트워크 충돌
* 라우팅 혼동
* VM에서 외부 접속 실패
* 호스트에서 VM으로 접근 실패
* 포트포워딩 동작 이상
* 브릿지 네트워크처럼 동작하는 것처럼 보임

같은 IP 대역을 사용한다면 NAT가 아니라 브릿지 네트워크를 사용하는 것이 개념적으로 더 적절하다.

---

# 올바른 NAT 설정 예시

호스트 PC가 `192.168.0.10`을 사용한다면 NAT 네트워크는 다른 대역으로 설정한다.

```text
Host IP: 192.168.0.10
Host Network: 192.168.0.0/24

VM NAT Network: 192.168.10.0/24
VM IP: 192.168.10.100
Gateway: 192.168.10.1
```

이렇게 설정하면 호스트의 실제 네트워크와 VM의 NAT 네트워크가 분리된다.

```text
192.168.0.0/24  → 실제 공유기/호스트 네트워크
192.168.10.0/24 → VM NAT 내부 네트워크
```

이 구조에서는 VM이 외부로 나갈 때 호스트의 NAT 기능을 통해 통신한다.

```text
VM(192.168.10.100)
        ↓
NAT Gateway(192.168.10.1)
        ↓
Host(192.168.0.10)
        ↓
Internet
```

---

# NAT와 Bridge의 차이

## NAT

NAT는 VM이 호스트 뒤쪽의 별도 사설 네트워크에 위치하는 방식이다.

```text
Internet
   |
Router
   |
Host PC
   |
NAT Network
   |
VM
```

예시:

```text
Host IP: 192.168.0.10
VM IP: 192.168.10.100
```

VM은 외부로 나갈 수 있지만, 외부에서 VM으로 직접 접근하기는 어렵다.
외부에서 VM으로 접속하려면 보통 **포트포워딩**이 필요하다.

---

## Bridge

Bridge는 VM이 호스트와 같은 물리 네트워크에 직접 연결된 것처럼 동작하는 방식이다.

```text
Internet
   |
Router
   |
Host PC
   |
VM
```

예시:

```text
Host IP: 192.168.0.10
VM IP: 192.168.0.20
```

이 경우 VM은 호스트와 같은 네트워크 대역을 사용한다.

따라서 같은 공유기에 연결된 다른 장치들도 VM에 직접 접근할 수 있다.

```text
Host Network: 192.168.0.0/24
Host IP: 192.168.0.10
VM IP: 192.168.0.20
```

즉, 같은 IP 대역을 쓰고 싶다면 NAT보다 Bridge가 더 적합하다.

---

# NAT를 쓸 때 대역을 다르게 써야 하는 이유

핵심은 라우팅 경로가 명확해야 한다는 것이다.

호스트와 NAT 네트워크가 같은 대역이면 호스트는 다음 두 경로를 혼동할 수 있다.

```text
192.168.0.0/24 → 실제 LAN
192.168.0.0/24 → VM NAT 내부망
```

반면 대역을 분리하면 경로가 명확해진다.

```text
192.168.0.0/24  → 실제 LAN
192.168.10.0/24 → VM NAT 내부망
```

그래서 NAT를 사용할 때는 다음처럼 설정하는 것이 좋다.

```text
Host Network: 192.168.0.0/24
NAT Network: 192.168.10.0/24
```

---

# 포트포워딩

NAT 환경에서는 VM이 외부로 나가는 것은 가능하지만, 외부에서 VM으로 직접 들어오는 것은 기본적으로 어렵다.

이유는 VM이 NAT 내부망에 있기 때문이다.

```text
외부 장치 → Host → NAT 내부 VM
```

외부 장치가 VM에 접근하려면 호스트가 특정 포트로 들어온 요청을 VM의 특정 포트로 전달해줘야 한다.
이것이 **포트포워딩**이다.

---

# 포트포워딩 예시

VM에서 웹 서버가 `80` 포트로 실행 중이라고 하자.

```text
VM IP: 192.168.10.100
VM Web Server Port: 80
```

호스트의 `8080` 포트로 들어온 요청을 VM의 `80` 포트로 전달하도록 설정할 수 있다.

```text
Host Port: 8080
Guest IP: 192.168.10.100
Guest Port: 80
```

그러면 외부에서는 다음 주소로 접근한다.

```text
http://호스트IP:8080
```

실제 내부 흐름은 다음과 같다.

```text
외부 클라이언트
      ↓
Host IP:8080
      ↓ 포트포워딩
VM IP:192.168.10.100:80
```

---

# VirtualBox 포트포워딩 예시

VirtualBox NAT 환경에서는 보통 다음과 같이 설정한다.

```text
Name: Web
Protocol: TCP
Host IP: 0.0.0.0 또는 비워둠
Host Port: 8080
Guest IP: 192.168.10.100
Guest Port: 80
```

접속 방법:

```text
http://127.0.0.1:8080
```

또는 같은 네트워크의 다른 장치에서 접근하려면:

```text
http://호스트PC_IP:8080
```

예시:

```text
http://192.168.0.10:8080
```

---

# SSH 포트포워딩 예시

VM에 SSH로 접속하고 싶다면 다음처럼 설정할 수 있다.

```text
Host Port: 2222
Guest IP: 192.168.10.100
Guest Port: 22
```

접속 명령:

```bash
ssh 사용자명@127.0.0.1 -p 2222
```

다른 장치에서 접근한다면:

```bash
ssh 사용자명@192.168.0.10 -p 2222
```

# NAT 설정 정리

호스트 IP가 다음과 같다면

```text
Host IP: 192.168.0.10
Host Network: 192.168.0.0/24
```

NAT 네트워크는 다음처럼 다른 대역으로 설정하는 것이 좋다.

```text
NAT Network: 192.168.10.0/24
Gateway: 192.168.10.1
VM IP: 192.168.10.x
```

같은 대역을 사용하면 NAT와 실제 LAN의 경로가 겹쳐 문제가 발생할 수 있다.

```text
잘못된 예:
Host Network: 192.168.0.0/24
NAT Network: 192.168.0.0/24
```

```text
올바른 예:
Host Network: 192.168.0.0/24
NAT Network: 192.168.10.0/24
```

---

# 핵심 정리

NAT는 VM을 호스트 뒤의 별도 사설 네트워크에 두고, 외부 통신은 호스트를 통해 나가게 하는 방식이다.

호스트와 VM NAT 네트워크가 같은 IP 대역을 사용하면 네트워크 경로가 겹쳐 라우팅 문제가 발생할 수 있다.

같은 IP 대역을 사용하려면 NAT가 아니라 Bridge를 사용하는 것이 적절하다.

NAT를 사용할 때는 호스트 네트워크와 VM NAT 네트워크를 다르게 설정해야 한다.

```text
Host: 192.168.0.0/24
NAT : 192.168.10.0/24
```

NAT 환경에서 외부에서 VM으로 접속하려면 포트포워딩이 필요하다.

```text
Host IP:8080 → VM IP:80
```

즉, NAT의 핵심은 다음과 같다.

```text
1. NAT는 호스트와 VM을 다른 네트워크 대역으로 분리한다.
2. VM은 호스트를 통해 외부로 나간다.
3. 외부에서 VM으로 들어오려면 포트포워딩이 필요하다.
4. 호스트와 NAT 네트워크의 IP 대역은 겹치지 않게 설정해야 한다.
```
