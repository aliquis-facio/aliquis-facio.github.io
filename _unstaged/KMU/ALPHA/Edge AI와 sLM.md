## 1. Edge AI란 무엇인가?

### 1.1 Edge AI란?

Edge AI는 데이터가 생성되는 현장(Edge), 즉 사용자의 기기(Device) 자체에서 AI 연산을 수행하는 기술을 말합니다.

클라우드(서버)로 데이터를 보내지 않고, 스마트폰, 노트북, 자동차, IoT 센서, CCTV 등 단말기 내부의 칩셋(NPU 등)을 이용해 AI 모델을 구동합니다. 이를 **On-Device AI(온디바이스 AI)** 라고도 부릅니다.

### 1.2 Edge AI 등장 배경

기존의 초거대 AI(ChatGPT 등)는 엄청난 연산량이 필요하여 거대한 데이터 센터(Cloud)에 의존해야 했습니다. 하지만 다음과 같은 문제가 발생했습니다.

- **지연 시간(Latency):** 서버에 갔다 오는 시간이 걸림.
- **보안(Privacy):** 민감한 내 데이터가 외부 서버로 전송됨.
- **비용(Cost):** 사용할 때마다 클라우드 서버 비용 발생.

이러한 문제를 해결하기 위해 **"내 기기에서 직접 돌리자"** 는 Edge AI가 부상하기 시작했습니다.

### 1.3 Cloud AI(LLM) vs Edge AI 비교

|구분|Cloud AI|Edge AI|
|---|---|---|
|두뇌 위치|Cloud Server|Smart Phone,PC,Server|
|인터넷 연결|필수|불필요(비행기 모드 작동 가능)|
|반응 속도|통신 상태에 따라 느려질 수 있음|즉시 반응|
|보안|데이터가 외부로 전송됨|데이터가 기기 밖을 나가지 않음|
|성능|석/박사급|특정 기능에 최적화|
|비용|서버 운영 및 API 비용|GPU 장비|

## 2. Edge AI와 LLM의 만남: sLM (Small LM)

"그럼 Edge AI에서는 우리가 배운 똑똑한 LLM을 못 쓰나요?"

아닙니다. 최근에는 LLM을 Edge 환경에서 돌리기 위해 모델의 크기를 줄이는 기술이 발전했습니다.

### 2.1 경량화 기술

거대한 LLM의 '지방'을 걷어내는 기술입니다.

- **가지치기:** 중요도가 낮은 파라미터(뉴런 연결)를 삭제합니다.
- **양자화:** 복잡한 소수점 계산을 단순한 정수 계산으로 바꿔 용량을 줄입니다.

### 2.2 sLM (Small Language Model)

이런 기술을 통해 파라미터 수를 줄여(약 20억~80억 개 수준), 노트북이나 스마트폰에서도 돌아가는 **'작지만 강한 모델'**들이 등장했습니다.

- _예시:_ Google의 **Gemma edge model**, Meta의 **Llama 3 (8B)**등.

---

### 2.3 우리 목표 시스템에서의 활용 방안

우리가 구축하려는 **'통제/실행/관리 권한이 부여된 챗봇'**에서 Edge AI 개념은 다음과 같이 활용될 수 있습니다.

#### 2.3.1 강력한 보안 (Security)

- **상황:** 사내 연봉 정보, 핵심 기술 문서 등 대외비 자료를 검색해야 할 때.
- **활용:** 외부 클라우드(ChatGPT 등)로 데이터를 보내지 않고, **사내 내부 서버(On-premise Edge)**에 설치된 sLM이 문서를 분석하고 답변합니다. 데이터가 회사 밖을 나가지 않으므로 보안 사고를 원천 차단합니다.

#### 2.3.2 실시간 제어 및 통제 (Real-time Control)

- **상황:** 챗봇에게 "지금 서버 온도가 너무 높으면 셧다운 시켜"라고 명령할 때.
- **활용:** 인터넷이 끊겨도 작동해야 하는 비상 시스템의 경우, Edge AI가 탑재되어 있다면 네트워크 장애와 상관없이 즉각적으로 위험을 감지하고 시스템을 제어할 수 있습니다.

#### 2.3.3 비용 효율화 (Cost Optimization)

- **활용:** 간단한 인사말, 단순 조회 업무는 가벼운 **Edge AI**가 처리하고, 복잡한 추론이나 창의적인 작업만 유료 **Cloud LLM**을 호출하는 **하이브리드(Hybrid) 방식**을 구축하여 운영 비용을 절감합니다.
