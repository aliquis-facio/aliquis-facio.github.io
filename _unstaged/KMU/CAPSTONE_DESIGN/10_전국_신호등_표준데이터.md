# 전국 신호등 표준데이터

위도, 경도

## API 내용 정리

올려주신 내용은 **공공데이터포털 전국신호등표준데이터 API**에 대한 명세로 보인다.
요청 주소는 `https://api.data.go.kr/openapi/tn_pubr_public_traffic_light_api`이며, 개발계정과 운영계정 모두 **자동승인**이고, 개발계정 기본 트래픽은 **1,000**이다. 

---

## 1. API 개요

이 API는 전국 단위의 **신호등 표준데이터**를 조회하는 Open API다.
즉, 특정 도시만이 아니라 여러 지자체의 신호등 정보를 표준 형식으로 조회할 수 있다.
응답에는 위치 정보뿐 아니라 신호등 종류, 점등 방식, 제어 방식, 보행자작동신호기 유무, 잔여시간표시기 유무, 시각장애인용 음향신호기 유무까지 포함된다. 

---

## 2. 요청 방식

### 요청 주소

```text
https://api.data.go.kr/openapi/tn_pubr_public_traffic_light_api
```



### 기본 요청 파라미터

* `pageNo` : 페이지 번호
* `numOfRows` : 한 페이지 결과 수, 최대 1000
* `type` : XML 또는 JSON 여부



### 조건 조회 파라미터

* `ctprvnNm` : 시도명
* `signguNm` : 시군구명
* `roadKnd` : 도로종류
* `roadRouteNo` : 도로노선번호
* `roadRouteNm` : 도로노선명
* `roadRouteDrc` : 도로노선방향
* `rdnmadr` : 소재지도로명주소
* `lnmadr` : 소재지지번주소
* `latitude` : 위도
* `longitude` : 경도



### 신호등 세부 속성 파라미터

* `sgngnrInstlMthd` : 신호기설치방식
* `roadType` : 도로형태
* `priorRoadYn` : 주도로여부
* `tfclghtManageNo` : 신호등관리번호
* `tfclghtSe` : 신호등구분
* `tfclghtColorKnd` : 신호등색종류
* `sgnaspMthd` : 신호등화방식
* `sgnaspOrdr` : 신호등화순서
* `sgnaspTime` : 신호등화시간
* `sotKnd` : 광원종류
* `signlCtrlMthd` : 신호제어방식
* `signlTimeMthdType` : 신호시간결정방식
* `opratnYn` : 점멸등운영여부
* `flashingLightOpenHhmm` : 점멸등운영시작시각
* `flashingLightCloseHhmm` : 점멸등운영종료시각
* `fnctngSgngnrYn` : 보행자작동신호기유무
* `remndrIdctYn` : 잔여시간표시기유무
* `sondSgngnrYn` : 시각장애인용음향신호기유무
* `drcbrdSn` : 도로안내표지일련번호
* `institutionNm` : 관리기관명
* `phoneNumber` : 관리기관전화번호
* `referenceDate` : 데이터기준일자
* `instt_code` : 제공기관코드



---

## 3. 응답 데이터

응답 결과도 요청 파라미터와 거의 같은 구조로 제공된다.
즉, 지역 정보, 주소 정보, 좌표, 신호등 설치 및 운영 방식, 부가 시설 여부, 관리기관 정보를 그대로 반환하는 형태다. 

### 주요 응답 항목

* 지역/주소: `ctprvnNm`, `signguNm`, `rdnmadr`, `lnmadr`
* 위치: `latitude`, `longitude`
* 관리 식별: `tfclghtManageNo`
* 신호등 특성: `tfclghtSe`, `tfclghtColorKnd`, `sgnaspMthd`, `sgnaspOrdr`, `sgnaspTime`
* 운영 정보: `signlCtrlMthd`, `signlTimeMthdType`, `opratnYn`
* 보행 편의 정보: `fnctngSgngnrYn`, `remndrIdctYn`, `sondSgngnrYn`
* 관리 정보: `institutionNm`, `phoneNumber`, `referenceDate`, `instt_code`

## 4. 이 API의 특징

### 전국 단위 데이터

서울시 전용이 아니라 전국 표준데이터이므로, 지역 확장성이 좋다. 

### 신호 운영 정보가 풍부함

단순 위치만 있는 것이 아니라, 신호등화순서, 신호등화시간, 점멸등 운영 시간, 제어 방식 같은 운영 정보까지 포함한다. 

### 보행약자 관련 정보 포함

* 보행자작동신호기유무
* 잔여시간표시기유무
* 시각장애인용음향신호기유무

이 세 항목이 포함되어 있어 보행약자 지원 서비스에 직접 활용하기 좋다. 

### 조건 검색이 다양함

지역, 주소, 도로 특성, 관리번호, 신호 운영 속성 등으로 세부 조회가 가능하다.

## 5. 에러 코드

이 API는 공공데이터포털 공통 오류 체계를 따른다. 주요 코드는 다음과 같다. 

* `00` : 정상
* `01` : 어플리케이션 에러
* `02` : 데이터베이스 에러
* `03` : 데이터없음 에러
* `04` : HTTP 에러
* `05` : 서비스 연결실패 에러
* `10` : 잘못된 요청 파라미터
* `11` : 필수 요청 파라미터 없음
* `12` : 해당 오픈API 서비스가 없거나 폐기됨
* `20` : 서비스 접근거부
* `21` : 일시적으로 사용할 수 없는 서비스키
* `22` : 서비스 요청 제한 횟수 초과
* `30` : 등록되지 않은 서비스키
* `31` : 기한만료된 서비스키
* `32` : 등록되지 않은 IP
* `33` : 서명되지 않은 호출
* `99` : 기타 에러

---

## 6. 활용 관점 정리

이 API는 다음 용도에 적합하다.

* 전국 단위 신호등 DB 구축
* 지역별 신호등 현황 조회
* 보행자 신호 관련 시설 분석
* 음향신호기, 잔여시간표시기, 보행자작동신호기 존재 여부 확인
* 횡단보도 데이터와 결합한 보행 안전 서비스 설계

특히 프로젝트 관점에서는 **횡단보도 API**와 함께 쓰면 좋다.
횡단보도 위치와 신호등 위치를 좌표 기반으로 매칭하면, 해당 횡단보도에

* 신호등이 있는지
* 잔여시간표시기가 있는지
* 시각장애인용 음향신호기가 있는지

를 함께 판단할 수 있기 때문이다. 

---

## 7. 한눈에 요약

공공데이터포털 전국신호등표준데이터 API는 전국 단위의 신호등 정보를 제공하는 Open API다. 위치와 주소뿐 아니라 신호등 구분, 색 종류, 신호등화방식, 신호제어방식, 점멸 운영 여부, 보행자작동신호기, 잔여시간표시기, 시각장애인용 음향신호기 유무까지 포함한다. 따라서 단순 위치 조회를 넘어 **보행 안전, 신호 운영, 교통약자 지원 분석**에 활용하기 좋은 데이터다. 

원하시면 다음에는 이 API를 **전국횡단보도표준데이터 API와 비교**해서 같이 정리해드리겠다.
