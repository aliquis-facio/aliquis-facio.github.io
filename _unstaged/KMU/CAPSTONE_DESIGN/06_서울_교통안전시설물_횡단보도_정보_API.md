# 서울 교통안전시설물 횡단보도 정보 정리

<font color="#ff0000">횡단보도를 하나의 시설물 데이터로 본다 → 행정/시설 관리 정보가 중심이다 → 사용 X</font>
좌표계: EPSG:5186

## 1. API 개요

이 API는 **서울특별시 교통안전시설물 중 횡단보도 정보**를 조회하기 위한 Open API이다.
서비스명은 **`trafficSafetyA004AInfo`** 이며, 서울 열린데이터 광장에서 발급받은 인증키를 사용해 호출한다. 샘플 URL도 함께 제공되어 있다. 

### 샘플 호출 형식

```text
http://openapi.seoul.go.kr:8088/(인증키)/xml/trafficSafetyA004AInfo/1/5/
```

즉, URL 구조는 대체로 다음과 같다.

```text
http://openapi.seoul.go.kr:8088/{KEY}/{TYPE}/{SERVICE}/{START_INDEX}/{END_INDEX}/
```

---

## 2. 요청 인자

이 API의 요청 인자는 다음과 같다. 

### 필수 요청값

* `KEY`
  인증키. 서울 열린데이터 광장에서 발급받은 키를 넣어야 한다.

* `TYPE`
  응답 형식. 지원 형식은 다음과 같다.

  * `xml`
  * `xmlf`
  * `xls`
  * `json`

* `SERVICE`
  서비스명. 이 API에서는 `trafficSafetyA004AInfo`를 사용한다.

* `START_INDEX`
  조회 시작 위치. 데이터 행의 시작 번호이다.

* `END_INDEX`
  조회 종료 위치. 데이터 행의 끝 번호이다.

### 선택 요청값

* `CRSWK_MNG_NO1`
  횡단보도관리번호. 특정 횡단보도를 지정해서 조회할 때 사용하는 값이다.

---

## 3. 응답 구조

응답에는 공통 메타 정보와 횡단보도 상세 정보가 함께 들어간다. 

### 공통 응답값

* `list_total_count`
  전체 데이터 건수

* `RESULT.CODE`
  요청 결과 코드

* `RESULT.MESSAGE`
  요청 결과 메시지

즉, 단순히 데이터만 주는 것이 아니라, 조회 성공 여부와 원인도 함께 반환하는 구조이다.

---

## 4. 주요 출력 항목

횡단보도 데이터 본문에는 다음과 같은 항목이 포함된다. 

### 기본 식별 정보

* `CRSWK_MNG_NO1` : 횡단보도관리번호
* `CRSWK_MNG_NO2` : 구횡단보도관리번호
* `NEW_RULE_ID` : 신규정규화ID
* `HSTRY_ID` : 이력ID

### 상태/분류 정보

* `STTS` : 상태
* `CRSWK_KND_CD` : 횡단보도종류코드
* `ROAD_SE` : 도로구분
* `EXPRS_SE` : 표출구분
* `JOB_SE` : 작업구분
* `CSTRN_FORM` : 공사형태

### 길이/구조 정보

* `WDTH_LEN` : 가로길이
* `VRTC_LEN` : 세로길이
* `ARW_INDCT_CNT` : 화살표시수량
* `ARW_INDCT_LEN` : 화살표시길이
* `EXPN_CAR_RD` : 고가 여부

### 위치/행정 정보

* `GU_CD` : 구코드
* `DONG_CD` : 동코드
* `LOTNO` : 지번
* `OLD_POLSTN_CD` : 구경찰서코드
* `NEW_POLSTN_CD` : 신경찰서코드
* `CMPTNC_BZENT` : 관할사업소
* `INTR_CD` : 교차로코드

### 설치/이력 정보

* `INSTL_YMD` : 설치일
* `RPLC_YMD` : 교체일
* `CSTRN_MNG_NO` : 공사관리번호

### 공간 정보

* `SPACE_DATA` : 공간데이터
* `XCRD` : X좌표
* `YCRD` : Y좌표

## 5. 이 API로 알 수 있는 것

이 API는 단순히 “횡단보도가 있다” 수준이 아니라, 다음 정보를 함께 제공한다. 

* 횡단보도의 고유 관리번호
* 횡단보도의 종류와 상태
* 횡단보도의 크기 정보
* 설치·교체 이력
* 행정구역 정보
* 좌표 기반 공간 정보

즉, **횡단보도 위치 확인**, **행정구역별 횡단보도 데이터 관리**, **특정 횡단보도 단건 조회**, **지도 시각화용 좌표 활용** 등에 사용할 수 있는 구조다.

## 6. 오류 코드와 메시지

이 API는 결과 코드와 메시지를 함께 반환하며, 주요 메시지는 다음과 같다. 

### 정상/데이터 없음

* `INFO-000` : 정상 처리되었습니다
* `INFO-200` : 해당하는 데이터가 없습니다

### 요청값 문제

* `ERROR-300` : 필수 값 누락
* `ERROR-301` : 파일 타입 값 누락 또는 유효하지 않음
* `ERROR-310` : 해당 서비스 없음
* `ERROR-331` : 요청시작위치 값 확인 필요
* `ERROR-332` : 요청종료위치 값 확인 필요
* `ERROR-333` : 요청 위치 값 타입 오류, 정수 입력 필요
* `ERROR-334` : 시작 위치가 종료 위치보다 큼

### 조회 범위 제한

* `ERROR-335` : 샘플데이터는 최대 5건까지만 조회 가능
* `ERROR-336` : 한 번에 최대 1000건까지만 조회 가능

### 서버 측 오류

* `ERROR-500` : 서버 오류
* `ERROR-600` : 데이터베이스 연결 오류
* `ERROR-601` : SQL 문장 오류

## 7. 구현할 때 주의할 점

이 명세를 실제 서비스에 붙일 때는 아래 사항을 신경 써야 한다.

### 7-1. 페이지 범위 제한

한 번에 최대 1000건까지만 가져올 수 있으므로, 데이터 전체 수집 시에는 반복 조회가 필요하다. `ERROR-336`이 이를 보여준다. 

### 7-2. 샘플 키와 실제 인증키 구분

샘플 URL은 `sample` 키 기준으로 설명되며, 샘플 조회는 1~5건 제한이 있다. 실제 서비스에서는 별도 인증키를 발급받아야 한다. 

### 7-3. 선택 파라미터 활용

`CRSWK_MNG_NO1`을 사용하면 특정 횡단보도만 조회할 수 있으므로, 전체 목록 조회와 단건 조회를 나눠 설계하기 좋다. 

### 7-4. 좌표계 확인 필요

명세에 `XCRD`, `YCRD`, `SPACE_DATA`가 있으므로, 실제 지도 서비스(T map, Kakao Map 등)와 붙일 때는 좌표계 변환 여부를 확인해야 한다. 텍스트 파일에는 좌표계 종류 자체는 직접 적혀 있지 않으므로, 이 부분은 실제 응답 예시나 원본 상세 문서에서 한 번 더 검증하는 것이 안전하다.

## 8. 한눈에 보는 요약

이 API는 **서울시 교통안전시설물 중 횡단보도 정보를 조회하는 Open API**이며,
`trafficSafetyA004AInfo`라는 서비스명으로 호출한다. 요청 시에는 인증키, 응답 형식, 조회 시작·종료 위치가 필수이고, 필요하면 횡단보도관리번호를 추가할 수 있다. 응답에는 횡단보도의 관리번호, 상태, 종류, 길이, 설치일, 행정구역, 좌표 정보 등이 포함되며, 결과 코드와 메시지를 통해 정상 처리 여부와 오류 원인도 확인할 수 있다. 특히 한 번에 최대 1000건까지만 조회 가능하므로 실제 서비스에서는 페이지 단위 반복 호출 구조로 사용하는 것이 적절하다.
