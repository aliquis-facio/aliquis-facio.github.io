# 서울시 음향신호기 관련 정보

좌표계: EPSG:5186

## 1. API 개요

이 API는 서울시 내 음향신호기 정보를 조회하기 위한 Open API다.
서비스명은 **`trafficSafetyA073PInfo`** 이며, 서울 열린데이터 광장에서 발급한 인증키를 사용해 호출한다. 샘플 URL은 다음 형식이다. 

```text id="d1"
http://openapi.seoul.go.kr:8088/(인증키)/xml/trafficSafetyA073PInfo/1/5/
```

즉 호출 구조는 아래와 같이 이해하면 된다.

```text id="d2"
http://openapi.seoul.go.kr:8088/{KEY}/{TYPE}/{SERVICE}/{START_INDEX}/{END_INDEX}/
```

---

## 2. 요청 인자

이 API의 요청값은 다음과 같다. 

### 필수 요청값

* `KEY` : 인증키
* `TYPE` : 요청 파일 형식
* `SERVICE` : 서비스명
* `START_INDEX` : 조회 시작 위치
* `END_INDEX` : 조회 종료 위치

### 선택 요청값

* `SUD_SGN_MNG_NO1` : 음향신호관리번호

즉, 기본적으로는 페이지 범위로 목록 조회를 하고, 필요하면 **음향신호관리번호**로 특정 데이터를 좁혀 조회할 수 있는 구조다. 

---

## 3. 지원 형식

`TYPE`에는 다음 형식을 사용할 수 있다. 

* `xml`
* `xmlf`
* `xls`
* `json`

---

## 4. 응답 구조

응답은 공통 정보와 본문 데이터로 나뉜다. 

### 공통 응답값

* `list_total_count` : 총 데이터 건수
* `RESULT.CODE` : 요청 결과 코드
* `RESULT.MESSAGE` : 요청 결과 메시지

즉, 조회 성공 여부와 실패 원인을 공통 필드에서 먼저 확인할 수 있다. 

---

## 5. 주요 출력값

이 API가 제공하는 핵심 필드는 다음과 같다. 

### 식별 및 관리 정보

* `SUD_SGN_MNG_NO1` : 음향신호관리번호
* `SUD_SGN_MNG_NO2` : 음향신호관리번호

* `XCRD` : X좌표
* `YCRD` : Y좌표

### 기타 정보

* `MKR` : 제조회사

## 6. 이 API로 알 수 있는 것

이 API는 단순히 음향신호기가 “있는지 없는지”만 보여주는 것이 아니라, 아래까지 확인할 수 있다. 

* 음향신호기의 고유 관리번호
* 어느 지주에 설치되어 있는지
* 방향 정보
* 설치일과 교체일
* 좌표와 위치 정보
* 제조사와 종류
* 상태와 작업 구분

즉, **시설 관리용 정보**와 **위치 정보**가 함께 들어 있는 API라고 볼 수 있다.

## 7. 오류 코드

이 API는 서울 열린데이터 광장 공통 오류 체계를 따른다. 주요 코드는 다음과 같다. 

### 정상/데이터 없음

* `INFO-000` : 정상 처리
* `INFO-200` : 해당 데이터 없음

### 요청 오류

* `ERROR-300` : 필수 값 누락
* `ERROR-301` : 파일 타입 누락 또는 오류
* `ERROR-310` : 서비스명 오류
* `ERROR-331` : 시작 위치 값 오류
* `ERROR-332` : 종료 위치 값 오류
* `ERROR-333` : 위치 값 타입 오류
* `ERROR-334` : 시작 위치가 종료 위치보다 큼

### 조회 제한

* `ERROR-335` : 샘플 데이터는 최대 5건까지
* `ERROR-336` : 한 번에 최대 1000건까지만 조회 가능

### 서버/DB 오류

* `ERROR-500` : 서버 오류
* `ERROR-600` : DB 연결 오류
* `ERROR-601` : SQL 오류

## 9. 프로젝트 활용 관점

이 API는 시각장애인 보행 보조 서비스 관점에서 꽤 유용하다. 이유는 다음과 같다. 

* 횡단보도 주변의 **음향신호기 존재 여부**를 확인할 수 있다.
* 좌표가 있으므로 **횡단보도 데이터와 위치 매칭**이 가능하다.
* 설치·교체·상태 정보가 있어 **운영 가능한 시설인지 판단하는 보조 자료**로 활용할 수 있다.
* 방향 정보가 있어 향후 **어느 방향 횡단보도에 연결된 장치인지 분석**하는 데 도움이 될 수 있다.

다만 이 자료만으로는 횡단보도와 음향신호기의 연결 관계가 직접 주어지지 않으므로, 실제 서비스에서는 **횡단보도 API와 좌표 기반으로 매칭하는 추가 로직**이 필요하다.
