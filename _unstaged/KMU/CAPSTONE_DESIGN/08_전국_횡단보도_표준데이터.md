## 전국 횡단보도 표준데이터 API

위도, 경도

## 1. API 개요

이 API는 전국 각 지자체가 제공하는 횡단보도 데이터를 표준 형식으로 묶어 제공한다.
즉, 서울처럼 특정 지역만 다루는 API가 아니라 **전국 단위**로 조회할 수 있다는 점이 핵심이다.
또한 단순 위치뿐 아니라 횡단보도의 구조, 신호기, 점자블록, 보도턱낮춤, 관리기관 정보까지 함께 포함한다. 

## 2. 요청 방식

기본적으로 페이지 기반 조회 구조다. 주요 요청 변수는 다음과 같다. 

### 공통 요청값

* `pageNo` : 페이지 번호
* `numOfRows` : 한 페이지 결과 수, 최대 1000
* `type` : XML 또는 JSON 형식 지정

### 조건 검색용 요청값

* `ctprvnNm` : 시도명
* `signguNm` : 시군구명
* `roadNm` : 도로명
* `rdnmadr` : 소재지도로명주소
* `lnmadr` : 소재지지번주소
* `crslkManageNo` : 횡단보도관리번호
* `crslkKnd` : 횡단보도종류

### 시설/속성 검색용 요청값

* `bcyclCrslkCmbnatYn` : 자전거횡단도겸용여부
* `highlandYn` : 고원식적용여부
* `latitude` : 위도
* `longitude` : 경도
* `cartrkCo` : 차로수
* `bt` : 횡단보도폭
* `et` : 횡단보도연장
* `tfclghtYn` : 보행자신호등유무
* `fnctngSgngnrYn` : 보행자작동신호기유무
* `sondSgngnrYn` : 음향신호기설치여부
* `greenSgngnrTime` : 녹색신호시간
* `redSgngnrTime` : 적색신호시간
* `tfcilndYn` : 교통섬유무
* `ftpthLowerYn` : 보도턱낮춤여부
* `brllBlckYn` : 점자블록유무
* `cnctrLghtFcltyYn` : 집중조명시설유무
* `institutionNm` : 관리기관명
* `phoneNumber` : 관리기관전화번호
* `referenceDate` : 데이터기준일자
* `instt_code` : 제공기관코드

즉, 이 API는 단순 전체 조회뿐 아니라 **지역, 주소, 관리번호, 편의시설 유무**까지 조건으로 걸 수 있는 구조다.

## 4. 이 API의 특징

### 전국 단위 데이터

서울시 API와 달리 특정 지자체 전용이 아니라 전국 단위 데이터다.
따라서 지역 확장성이 좋고, 특정 도시 외 지역까지 함께 다룰 수 있다. 

### 보행약자 관점 정보가 많음

음향신호기, 점자블록, 보도턱낮춤, 집중조명시설, 보행자작동신호기 같은 항목이 있어 시각장애인·교통약자 보행 지원 서비스에 직접적으로 활용하기 좋다. 

### 검색 조건이 풍부함

시도명, 시군구명, 도로명, 주소, 관리번호 등 다양한 조건으로 조회할 수 있어 원하는 구간만 필터링하기 쉽다. 

### 페이지 기반 대량 조회

`pageNo`, `numOfRows` 구조이며 한 페이지 최대 1000건까지 받을 수 있다. 전체 수집 시에는 반복 조회가 필요하다. 

## 5. 에러 코드

문서에 나온 주요 에러 코드는 다음과 같다. 

* `00` : 정상
* `01` : 어플리케이션 에러
* `02` : 데이터베이스 에러
* `03` : 데이터 없음
* `04` : HTTP 에러
* `05` : 서비스 연결실패
* `10` : 잘못된 요청 파라미터
* `11` : 필수 요청 파라미터 누락
* `12` : 존재하지 않거나 폐기된 오픈API 서비스
* `20` : 서비스 접근 거부
* `21` : 일시적으로 사용할 수 없는 서비스키
* `22` : 요청 제한 횟수 초과
* `30` : 등록되지 않은 서비스키
* `31` : 기한 만료된 서비스키
* `32` : 등록되지 않은 IP
* `33` : 서명되지 않은 호출
* `99` : 기타 에러

서울 열린데이터 광장 API보다 에러 체계가 더 세분화되어 있고, 공공데이터포털 공통 오류 규격을 따른다고 보면 된다. 

## 6. 활용 관점 정리

이 API는 다음 용도에 특히 적합하다.

* 전국 단위 횡단보도 DB 구축
* 시도/시군구별 횡단보도 현황 조회
* 음향신호기, 점자블록, 보도턱낮춤 여부 기반 필터링
* 보행약자 맞춤 경로 안내 보조 데이터
* 횡단보도 안전성·편의성 분석

특히 프로젝트 맥락에서는 **횡단보도 자체 존재 여부**뿐 아니라 **음향신호기 설치 여부**, **점자블록 유무**, **보도턱낮춤 여부**까지 함께 볼 수 있어서 활용성이 높다. 

## 7. 한눈에 요약

공공데이터포털 전국횡단보도표준데이터 API는 전국 단위 횡단보도 정보를 조회하는 표준 Open API로, 위치와 주소뿐 아니라 차로수, 횡단보도 폭·연장, 보행자신호등, 음향신호기, 점자블록, 보도턱낮춤, 관리기관 정보까지 제공한다. 페이지 기반 조회 방식이며 한 페이지 최대 1000건까지 가져올 수 있고, 조건 검색 항목이 많아 필요한 지역이나 조건만 골라 조회하기 좋다. 시각장애인 보행 보조 서비스 관점에서는 **음향신호기, 점자블록, 보도턱낮춤, 신호등 유무**를 함께 얻을 수 있다는 점이 가장 중요하다.
