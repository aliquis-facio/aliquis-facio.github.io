# L21 로컬 JSON 데이터 활용

## 1. Overview

이번 강의에서는 로컬 JSON 데이터를 웹 페이지에서 활용하는 방법을 학습한다.

주요 내용은 다음과 같다.

- 왜 데이터를 분리하는가?
- JSON 파일이란 무엇인가?
- `fetch`로 JSON 파일을 불러오는 방법
- 불러온 데이터를 화면에 출력하는 방법
- Live Server가 필요한 이유

---

## 2. 왜 데이터를 분리할까?

기존에는 데이터를 JavaScript 코드 안에 변수나 배열 형태로 직접 작성했다.

```js
const questions = [
  { q: "사자의 서식지는?", a: "사바나" },
  { q: "북극곰의 서식지는?", a: "북극" },
  { q: "판다의 서식지는?", a: "중국 산림" }
];
````

하지만 데이터가 많아질수록 코드가 복잡해지고 관리가 어려워진다.

따라서 데이터를 별도의 파일로 분리하면 다음과 같은 장점이 있다.

* JavaScript 코드는 로직 중심으로 정리할 수 있다.
* 데이터 수정이 쉬워진다.
* 코드와 데이터를 분리하여 유지보수가 편해진다.

예를 들어, JavaScript 파일에서는 데이터를 불러오는 로직만 작성하고, 실제 데이터는 `quiz.json` 같은 별도 파일에 저장할 수 있다.

```js
fetch('quiz.json')
  .then(res => res.json())
  .then(questions => {
    questions.forEach(...);
  });
```

---

## 3. JSON 파일이란?

JSON은 **JavaScript Object Notation**의 약자로, 데이터를 표현하기 위한 형식이다.

JavaScript 객체와 비슷하게 생겼지만 몇 가지 규칙이 있다.

### JSON의 주요 규칙

* 키는 반드시 큰따옴표 `"`로 감싸야 한다.
* 문자열 값도 반드시 큰따옴표 `"`를 사용해야 한다.
* 작은따옴표 `'`는 사용할 수 없다.
* 파일 확장자는 `.json`이다.

예시:

```json
[
  {
    "name": "사자",
    "habitat": "사바나",
    "endangered": false
  },
  {
    "name": "북극곰",
    "habitat": "북극",
    "endangered": true
  },
  {
    "name": "판다",
    "habitat": "중국 산림",
    "endangered": true
  }
]
```

### JSON에서 사용할 수 있는 자료형

| 자료형  | 예시              |
| ---- | --------------- |
| 문자열  | `"사자"`          |
| 숫자   | `2024`          |
| 참/거짓 | `true`, `false` |
| 배열   | `[ ... ]`       |
| 객체   | `{ ... }`       |
| null | `null`          |

주의할 점은 JSON에서는 큰따옴표만 사용할 수 있다는 것이다.

## 4. `fetch`로 JSON 불러오기

`fetch`는 파일이나 데이터를 가져올 때 사용하는 브라우저 내장 함수이다.

JSON 파일을 불러오는 기본 흐름은 다음과 같다.

```js
fetch('animals.json')
  .then(res => res.json())
  .then(data => {
    console.log(data);
  });
```

### 단계별 설명

| 단계                         | 설명                  |
| -------------------------- | ------------------- |
| `fetch(...)`               | 파일 요청 보내기           |
| `.then(res => res.json())` | 받은 응답을 JSON 형식으로 변환 |
| `.then(data => ...)`       | 변환된 데이터를 사용         |

`.then`이 두 번 사용되는 이유는 다음과 같다.

1. 첫 번째 `.then`에서는 파일 응답을 받는다.
2. 두 번째 `.then`에서는 JSON으로 변환된 데이터를 사용한다.

즉, `fetch`는 단순히 파일을 가져오는 역할이고, `res.json()`을 통해 실제 JavaScript에서 사용할 수 있는 데이터로 바꿔야 한다.

## 5. 화면에 데이터 출력하기

불러온 JSON 데이터는 DOM 조작을 통해 화면에 표시할 수 있다.

예를 들어, HTML에 다음과 같은 리스트 영역이 있다고 하자.

```html
<ul id="list"></ul>
```

JavaScript에서는 `fetch`로 JSON을 불러온 뒤, `forEach`로 배열을 반복하면서 요소를 생성할 수 있다.

```html
<script>
fetch('animals.json')
  .then(res => res.json())
  .then(data => {
    data.forEach(animal => {
      const li = document.createElement('li');
      li.textContent = `${animal.name} - ${animal.habitat}`;

      document.getElementById('list')
        .appendChild(li);
    });
  });
</script>
```

위 코드를 실행하면 화면에는 다음과 같이 출력된다.

```txt
사자 - 사바나
북극곰 - 북극
판다 - 중국 산림
```

핵심은 다음과 같다.

* `fetch`는 데이터를 가져오는 역할을 한다.
* 화면에 표시하는 작업은 DOM 조작으로 처리한다.
* 배열 데이터는 `forEach`로 반복하면서 요소를 추가할 수 있다.

## 6. Live Server가 필요한 이유

HTML 파일을 단순히 더블클릭해서 열면 주소가 `file://` 형식이 된다.

예시:

```txt
file:///C:/Users/.../index.html
```

이 경우 브라우저 보안 정책 때문에 로컬 JSON 파일을 `fetch`로 불러오는 것이 차단될 수 있다.

따라서 `fetch`를 정상적으로 사용하려면 `http://` 환경에서 실행해야 한다.

예시:

```txt
http://127.0.0.1:5500/index.html
```

이를 위해 VS Code의 Live Server 확장을 사용한다.

### Live Server 설치 및 실행 방법

1. VS Code 좌측 Extensions 탭을 연다.
2. `"Live Server"`를 검색한다.
3. 제작자가 Ritwick Dey인 확장을 설치한다.
4. HTML 파일을 우클릭한다.
5. `"Open with Live Server"`를 선택한다.

---

## 7. 핵심 정리

JSON 데이터를 활용하면 코드와 데이터를 분리할 수 있다.

정리하면 다음과 같다.

* 데이터가 많아지면 JavaScript 코드 안에 직접 작성하는 방식은 관리가 어렵다.
* JSON 파일을 사용하면 데이터를 별도 파일로 분리할 수 있다.
* JSON에서는 키와 문자열 값에 반드시 큰따옴표를 사용해야 한다.
* `fetch()`를 사용하면 로컬 JSON 파일을 불러올 수 있다.
* `res.json()`을 통해 응답을 JavaScript 데이터로 변환해야 한다.
* 화면 출력은 `createElement`, `textContent`, `appendChild` 같은 DOM 조작으로 처리한다.
* 로컬 JSON 파일을 `fetch`로 불러오려면 Live Server처럼 `http://` 환경이 필요하다.
