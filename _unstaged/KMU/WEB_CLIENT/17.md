# L17 사용자 중심 웹 설계

## 1. UX란 무엇인가

### UX와 UI의 차이

#### UX(User Experience)
UX는 사용자가 사이트를 사용하며 겪는 전체적인 경험을 의미한다.

- 사용자가 원하는 것을 쉽게 찾을 수 있는가
- 사이트 흐름이 자연스러운가
- 사용 후 만족감을 느끼는가

#### UI(User Interface)
UI는 사용자가 직접 보고 조작하는 화면 요소를 의미한다.

- 버튼
- 메뉴
- 색상
- 레이아웃
- 폰트
- 아이콘
- 이미지

> UI는 자동차의 인테리어이고, UX는 운전하는 경험 전체라고 볼 수 있다.  
> UI가 좋아도 UX가 나쁠 수 있고, 반대로 UI가 단순해도 UX가 좋을 수 있다.

---

## 2. 좋은 UX의 5가지 원칙

### 1. Usability · 사용 가능성
사용자가 헷갈리지 않고 목적을 달성할 수 있어야 한다.

### 2. Useful · 유용성
사용자가 원하는 가치를 제공해야 한다.

### 3. Desirable · 매력성
디자인이 매력적이고 사용자가 쓰고 싶게 만들어야 한다.

### 4. Findable · 발견 가능성
원하는 기능이나 정보를 빠르게 찾을 수 있어야 한다.

### 5. Accessible · 접근성
시각장애인, 색약자, 노인 등 다양한 사용자가 사용할 수 있어야 한다.

---

## 3. 타겟 사용자

사용자 중심 웹 설계는 “누가 이 사이트를 사용할 것인가?”를 정하는 것에서 시작한다.

### 모호한 타겟의 문제
예를 들어 “모든 사람을 위한 음악 사이트”는 너무 넓은 대상이기 때문에 디자인, 기능, 콘텐츠 방향이 흐려질 수 있다.

### 구체적인 타겟의 장점
예를 들어 “90년대를 기억하는 30대 직장인을 위한 옛날 가요 사이트”처럼 구체적으로 설정하면 디자인 톤, 콘텐츠, 기능이 명확해진다.

### 페르소나
페르소나는 가상의 사용자 한 명을 구체적으로 설정하는 것이다.

예시:

- 32세 직장인 김OO
- 출퇴근길에 옛날 노래 듣기를 좋아함
- 추억을 느낄 수 있는 콘텐츠를 원함

---

## 4. 사용자 여정(User Journey)

사용자 여정은 사용자가 사이트에서 어떤 흐름으로 움직이는지를 설계하는 것이다.

일반적인 흐름은 다음과 같다.

1. 진입  
   - 메인 페이지 도착

2. 탐색  
   - 사이트가 무엇인지 둘러봄

3. 행동  
   - 게임 플레이, 폼 작성 등 실제 기능 사용

4. 만족  
   - 정보를 얻거나 결과를 확인하고 사이트를 떠남

웹 프로젝트에서는 사용자가 `메인 → 팀원 소개 → 게임 → 폼` 페이지를 이동하면서 무엇을 보고, 무엇을 할지 미리 설계해야 한다.

---

## 5. 좋은 UI 사례

### 1. 직관적인 레이아웃

사용자는 화면을 보자마자 어디서 무엇을 해야 하는지 알 수 있어야 한다.

좋은 레이아웃은 다음과 같은 특징을 가진다.

- 중요한 버튼은 크게 표시
- 부가 정보는 작게 표시
- 핵심 행동이 눈에 잘 들어옴

나쁜 레이아웃은 모든 요소의 크기와 강조 정도가 비슷해서 무엇이 중요한지 알기 어렵다.

---

### 2. 일관된 디자인

모든 페이지가 같은 사이트처럼 보여야 한다.

일관성을 위해 통일해야 할 요소는 다음과 같다.

- 색상
- 폰트
- 버튼 스타일
- 레이아웃
- 간격

팀 프로젝트에서는 팀원별로 페이지를 나누어 만들다 보면 각 페이지의 디자인이 따로 놀기 쉽다.  
따라서 작업 전에 공통 디자인 규칙을 정해야 한다.

---

### 3. 시각적 위계

중요한 정보가 먼저 눈에 들어오도록 해야 한다.

시각적 위계를 만드는 요소는 다음과 같다.

- 글자 크기
- 글자 굵기
- 색상
- 여백
- 배치

좋은 화면은 제목, 부제목, 본문, 버튼, 부가 정보의 중요도가 명확히 구분된다.

---

### 4. 친절한 피드백

사용자가 어떤 행동을 했을 때 그 결과를 알 수 있어야 한다.

예시:

- 버튼을 누르면 색이 바뀜
- 제출 중에는 “처리 중...” 표시
- 입력 오류가 있으면 원인과 해결 방법 안내
- 제출 성공 시 성공 메시지 표시

좋은 에러 메시지 예시:

> 이메일 형식이 올바르지 않습니다.  
> 예: name@example.com

나쁜 에러 메시지 예시:

> Error 400  
> Invalid input

## 6. 디자인 영감 사이트

좋은 디자인을 많이 보는 것도 중요하다.

| 사이트 | 특징 |
|---|---|
| Awwwards | 전 세계 우수 웹사이트 모음 |
| Dribbble | UI/UX 디자인 작품과 트렌드 확인 |
| Behance | 디자이너 포트폴리오와 대형 프로젝트 참고 |
| Pinterest | 무드보드 제작에 유용 |
| Land-book | 랜딩 페이지 디자인 참고 |

단, 사이트 전체를 베끼기보다는 색감, 레이아웃, 버튼 스타일 등 부분적인 영감을 얻는 방식으로 활용해야 한다.

## 7. 우리 과제에 적용하기

이번 웹 프로젝트는 최소 4페이지 웹사이트를 만드는 것이다.

필수 페이지는 다음과 같다.

1. 메인 / 자유 페이지
2. 팀원 소개 페이지
3. 게임 페이지
4. 폼 페이지

계획서 발표 전 고민해야 할 내용은 다음과 같다.

- 우리 사이트의 타겟 사용자는 누구인가?
- 사용자가 우리 사이트에서 어떤 경험을 얻는가?
- 4개 페이지가 하나의 흐름으로 연결되는가?
- 디자인 컨셉을 어떻게 통일할 것인가?

## 8. 타겟 사용자 정하기

사이트 기획의 시작은 “누가 이 사이트에 들어올까?”를 정하는 것이다.

타겟 사용자가 구체적일수록 다음 요소가 명확해진다.

- 디자인 방향
- 콘텐츠 구성
- 기능 설계
- 사용자 흐름

예시 페르소나:

- 이름: 김지은
- 나이: 32세
- 직업: 마케터
- 상황: 출퇴근 시간이 길고, 지하철에서 추억거리를 찾고 싶어 함
- 관심사: 90년대 가요, 옛날 드라마
- 원하는 것: 30대 직장인이 공감할 수 있는 90년대 추억 콘텐츠

## 9. 와이어프레임

와이어프레임은 색상이나 이미지 없이 페이지 구조만 간단히 그린 스케치이다.

와이어프레임을 먼저 만들면 코딩 과정에서 구조를 잡기 쉽다.

### 와이어프레임 작성 도구

| 도구 | 특징 |
|---|---|
| 종이 + 펜 | 가장 빠르고 간단함 |
| Figma | 무료로 사용할 수 있는 전문 도구 |
| PowerPoint | 도형을 활용해 쉽게 제작 가능 |
| 아이패드 메모 | 손그림으로 빠르게 작성 가능 |

4페이지 웹사이트라면 각 페이지별 와이어프레임 4장을 준비하는 것이 좋다.

## 10. 디자인 컨셉

여러 명이 나누어 작업해도 하나의 사이트처럼 보이기 위해서는 디자인 규칙이 필요하다.

정해야 할 요소는 다음과 같다.

### 컬러 팔레트
보통 3~5개의 색을 정한다.

- 메인 색
- 서브 색
- 강조 색
- 배경 색
- 텍스트 색

### 폰트
제목과 본문 폰트를 구분할 수 있다.

주의할 점:

- 본문 폰트는 가독성이 좋아야 한다.
- 너무 화려한 폰트는 본문에 적합하지 않다.
- 폰트는 최대 2~3종 정도만 사용하는 것이 좋다.

### 분위기 키워드
사이트의 분위기를 나타내는 키워드를 3개 정도 정한다.

예시:

- 따뜻한, 친근한, 미니멀
- 레트로, 빈티지, 감성적
- 미래적, 깔끔, 신뢰감

---

## 11. 컨셉 통일성

좋은 프로젝트는 메인, 팀원 소개, 게임, 폼 페이지가 하나의 흐름으로 연결된다.

### 좋은 예시 1: 멸종위기 동물 알리미

- 메인: 멸종위기 동물 현황과 보호 필요성 소개
- 팀원: 각자 좋아하는 멸종위기 동물 소개
- 게임: 동물 이름 또는 서식지 매칭 게임
- 폼: 관심 동물을 입력하면 상세 정보와 보호 단체 안내

사용자가 동물을 알아가고, 보호 행동으로 이어지는 흐름이 생긴다.

---

### 좋은 예시 2: 90년대 추억 박물관

- 메인: 90년대 무드와 시대 타임라인
- 팀원: 90년대 명함 스타일 소개
- 게임: 90년대 유행어 또는 광고 음악 맞추기
- 폼: 출생연도를 입력하면 그 해 유행했던 것 표시

디자인과 콘텐츠가 모두 “90년대”라는 컨셉으로 연결된다.

---

### 좋은 예시 3: 이태원 동네 가이드

- 메인: 이태원 소개와 분위기 전달
- 팀원: 각자 추천하는 이태원 장소 소개
- 게임: 골목 사진을 보고 장소 맞추기
- 폼: 원하는 활동을 입력하면 맞춤 코스 추천

한 동네를 다양한 방식으로 알아가는 흐름이 형성된다.

---

## 12. 애매한 컨셉 예시

같은 카테고리로 묶이긴 했지만 페이지 간 흐름이 약한 경우이다.

### 음식 사이트
- 메인: 좋아하는 음식 소개
- 게임: 음식 이름 맞추기
- 폼: 좋아하는 음식 설문

문제점:

- 모두 음식과 관련은 있지만 단순 나열에 가깝다.
- 사용자가 사이트를 보고 얻는 경험이 명확하지 않다.

### 스포츠 사이트
- 메인: 다양한 스포츠 소개
- 게임: 종목 맞추기 퀴즈
- 폼: 좋아하는 스포츠 입력

문제점:

- “스포츠”라는 단어로만 묶여 있다.
- 사이트 전체의 메시지가 약하다.

---

## 13. 나쁜 컨셉 예시

주제와 게임, 폼이 전혀 관련 없는 경우이다.

### 동물원 사이트
- 메인: 동물원 소개
- 게임: 가위바위보
- 폼: BMI 계산기

문제점:

- 동물원 사이트에 가위바위보와 BMI 계산기가 왜 있는지 설명하기 어렵다.

### 여행 사이트
- 메인: 여행지 추천
- 게임: 테트리스
- 폼: 회원가입 폼

문제점:

- 게임과 폼이 사이트 주제와 연결되지 않는다.

### 카페 사이트
- 메인: 카페 메뉴
- 게임: 사다리타기
- 폼: MBTI 설문

문제점:

- 각 페이지가 서로 다른 사이트에서 가져온 것처럼 보인다.

> 발표 때 “왜 우리 사이트에 이 게임이 있나요?”라는 질문에 자연스럽게 답할 수 있어야 한다.

## 14. 계획서 발표에 꼭 들어가야 할 것

계획서 발표에는 다음 내용이 포함되어야 한다.

### 1. 팀 소개
- 팀명
- 팀원
- 역할 분담

### 2. 주제 & 타겟 사용자
- 어떤 사이트인지
- 누구를 위한 사이트인지
- 페르소나

### 3. 사용자에게 줄 경험
- 사용자가 사이트를 떠날 때 무엇을 얻는가

### 4. 핵심 기능
- 페이지별 기능
- 사용자가 각 페이지에서 할 수 있는 것

### 5. 와이어프레임
- 각 페이지 구조 스케치
- 최소 4장

### 6. 디자인 컨셉
- 컬러 팔레트
- 폰트
- 분위기 키워드

### 7. 통일성 설명
- 주제와 게임, 폼이 어떻게 연결되는지 설명

### 8. 일정 & 역할 분담표
- 누가
- 언제
- 무엇을 할 것인지

## 15. 핵심 정리

사용자 중심 웹 설계에서 중요한 것은 사용자의 입장에서 생각하는 것이다.

좋은 웹사이트는 다음 조건을 만족해야 한다.

- 타겟 사용자가 명확하다.
- 사용자의 이동 흐름이 자연스럽다.
- 화면 구조가 직관적이다.
- 디자인이 일관되어 있다.
- 중요한 정보가 먼저 보인다.
- 사용자의 행동에 적절한 피드백을 준다.
- 모든 페이지가 하나의 컨셉으로 연결된다.

결국 좋은 UX는 “사용자가 편하게, 자연스럽게, 만족스럽게 사용할 수 있는가”를 고민하는 것에서 시작한다.
