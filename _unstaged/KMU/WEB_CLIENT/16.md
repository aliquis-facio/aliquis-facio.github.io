# L16 JavaScript (4) 정리

## 1. Recap

지난 시간에는 JavaScript의 객체와 내장 객체를 다뤘다.

* JS에서는 거의 모든 것이 객체이다.
* 객체는 속성, 즉 데이터와 메서드, 즉 함수의 집합이다.
* 객체 생성 방법에는 객체 리터럴과 `new Object()`가 있다.

* 객체 순회에는 `for...in`, `Object.keys()`, `Object.values()`, `Object.entries()`를 사용할 수 있다.
* 객체 메서드 안에서는 `this`를 사용할 수 있다.
* 주요 내장 객체로는 `window`, `String`, `Array`, `Math`, `Date`, `Object` 등이 있다.

# 3. 사용자 정의 객체

## 3.1 class로 객체 만들기

객체 리터럴로 같은 구조의 객체를 여러 개 만들면 매번 같은 코드를 반복해야 한다.

이때 `class`를 사용하면 객체의 틀을 미리 만들고, `new`를 통해 같은 구조의 객체를 여러 개 생성할 수 있다.

```js
class Student {
  constructor(name, age, major) {
    this.name = name;
    this.age = age;
    this.major = major;
  }

  introduce() {
    console.log(`저는 ${this.name}입니다`);
  }

  birthday() {
    this.age += 1;
  }
}

const s1 = new Student('홍길동', 21, '컴공');
const s2 = new Student('김철수', 22, '전자');

s1.introduce();
s2.introduce();
```

## 3.2 class 구조

| 구성 요소         | 설명                         |
| ------------- | -------------------------- |
| `class 이름`    | 객체의 틀 이름, 관례적으로 대문자로 시작    |
| `constructor` | `new`로 객체를 만들 때 자동 실행되는 함수 |
| `this.속성`     | 새로 생성될 객체의 속성              |
| 메서드           | `function` 키워드 없이 작성       |
| `new Class()` | class를 기반으로 객체 생성          |

## 3.3 this의 의미

`class` 내부의 `this`는 `new`로 만들어질 객체를 가리킨다.

```js
const s1 = new Student('홍길동', 21, '컴공');
```

위 코드에서 `constructor` 내부의 `this`는 `s1`을 의미한다.

```js
const s2 = new Student('김철수', 22, '전자');
```

이 경우 `this`는 `s2`를 의미한다.

# 4. DOM이란?

## 4.1 DOM의 개념

DOM은 `Document Object Model`의 약자이다.

브라우저가 HTML 문서를 읽으면 HTML 요소들을 JavaScript가 다룰 수 있는 객체 형태로 변환한다. 이 객체 구조를 DOM이라고 한다.

JavaScript에서는 `document` 객체를 통해 DOM에 접근한다.

```html
<h1 id="title">안녕</h1>
<p class="msg">반갑습니다</p>
<button>클릭</button>
```

```js
const h1 = document.querySelector('#title');

console.log(h1.innerText);

h1.innerText = '반가워';
```

## 4.2 DOM 구조 예시

```text
document
  └─ html
      └─ body
          ├─ h1#title
          ├─ p.msg
          └─ button
```

## 4.3 DOM의 핵심 개념

* HTML의 모든 요소는 객체로 표현된다.
* 각 요소는 속성과 메서드를 가진다.
* JavaScript로 조작하면 화면이 즉시 변경된다.
* DOM은 트리 구조로 표현된다.

# 5. 요소 선택

## 5.1 getElementById

`getElementById()`는 ID로 요소 하나를 선택한다.

이때 `#`을 붙이지 않는다.

```js
const h1 = document.getElementById('title');
```

## 5.2 querySelector

`querySelector()`는 CSS 선택자를 사용하여 조건에 맞는 첫 번째 요소 하나를 선택한다.

```js
document.querySelector('#title');
document.querySelector('.msg');
document.querySelector('button');
document.querySelector('body > h1');
document.querySelector('p.msg');
```

## 5.3 getElementById와 querySelector 비교

| 메서드                | 설명                  |
| ------------------ | ------------------- |
| `getElementById()` | ID로 하나 찾기           |
| `querySelector()`  | CSS 선택자로 첫 번째 요소 찾기 |

## 5.4 CSS 선택자 표기

| 선택자      | 의미                   |
| -------- | -------------------- |
| `#title` | ID가 `title`인 요소      |
| `.msg`   | class가 `msg`인 요소     |
| `h1`     | `h1` 태그 요소           |
| `p.msg`  | class가 `msg`인 `p` 요소 |

실무에서는 한 가지 문법으로 다양한 선택이 가능한 `querySelector()`를 자주 사용한다.

# 6. CSS 선택자 복습

`querySelector()`와 `querySelectorAll()`은 CSS 선택자를 그대로 사용한다.

## 6.1 예제 구조

```html
<div>
  <h1>제목 A</h1>
  <p>문단 1</p>
  <section>
    <h1>제목 B (안쪽)</h1>
    <p>문단 2</p>
  </section>
  <p>문단 3</p>
</div>
```

## 6.2 주요 선택자

| 선택자        | 의미                | 결과         |
| ---------- | ----------------- | ---------- |
| `div h1`   | `div`의 모든 자손 `h1` | 제목 A, 제목 B |
| `div > h1` | `div`의 직접 자식 `h1` | 제목 A       |
| `h1 + p`   | `h1` 바로 다음 형제 `p` | 문단 1       |
| `h1 ~ p`   | `h1` 이후 모든 형제 `p` | 문단 1, 문단 3 |

# 7. 여러 요소 선택

## 7.1 querySelectorAll

`querySelectorAll()`은 조건에 맞는 여러 요소를 한 번에 가져온다.

```js
const items = document.querySelectorAll('.msg');

console.log(items.length);
console.log(items[0].innerText);
console.log(items[1].innerText);

items.forEach(function(el) {
  console.log(el.innerText);
});
```

## 7.2 여러 요소 선택 메서드 비교

| 메서드                        | 반환값              |
| -------------------------- | ---------------- |
| `querySelectorAll()`       | `NodeList`       |
| `getElementsByClassName()` | `HTMLCollection` |
| `getElementsByTagName()`   | `HTMLCollection` |

## 7.3 NodeList

`NodeList`는 배열과 비슷하지만 진짜 배열은 아니다.

가능한 기능은 다음과 같다.

* `.length`
* `.forEach()`
* 인덱스 접근

```js
items[0]
items.length
items.forEach(...)
```

하지만 `push()`, `pop()` 같은 배열 메서드는 사용할 수 없다.

진짜 배열로 바꾸려면 다음과 같이 작성한다.

```js
Array.from(nodeList)
```

# 8. 내용 변경

HTML 요소 안의 텍스트나 HTML을 읽고 변경할 때는 다음 속성을 사용한다.

* `innerText`
* `innerHTML`
* `textContent`

## 8.1 innerText

`innerText`는 사용자가 화면에서 보는 텍스트를 읽거나 변경한다.

```js
const p = document.querySelector('#msg');

console.log(p.innerText);
p.innerText = '바뀐 텍스트';
```

## 8.2 innerHTML

`innerHTML`은 HTML 태그를 포함하여 내용을 변경할 수 있다.

```js
p.innerHTML = '<b>굵은</b> 글씨';
```

위 코드는 `<b>` 태그가 실제 HTML로 해석되어 굵은 글씨가 표시된다.

## 8.3 textContent

`textContent`는 요소 안의 텍스트만 가져온다.

숨겨진 요소의 텍스트까지 가져올 수 있으며, 상대적으로 빠르다.

```js
console.log(p.textContent);
```

## 8.4 비교 정리

| 속성            | 특징           |
| ------------- | ------------ |
| `innerText`   | 사용자가 보는 텍스트  |
| `innerHTML`   | HTML 태그까지 처리 |
| `textContent` | 모든 텍스트, 더 빠름 |

## 8.5 innerHTML 사용 시 주의점

사용자 입력을 `innerHTML`에 그대로 넣으면 보안상 위험할 수 있다.

사용자 입력은 보통 `innerText`로 넣는 것이 안전하다.

# 9. 입력값 다루기

폼 요소의 값을 읽거나 변경할 때는 요소 종류에 따라 `value` 또는 `checked`를 사용한다.

## 9.1 value

`input`, `textarea`, `select`의 값은 `value`로 읽고 쓴다.

```js
const nameInput = document.querySelector('#name');

console.log(nameInput.value);
nameInput.value = '홍길동';
```

```js
const city = document.querySelector('#city');

console.log(city.value);
city.value = 'busan';
```

## 9.2 checked

`checkbox`와 `radio`는 선택 여부를 `checked`로 확인한다.

```js
const agree = document.querySelector('#agree');

console.log(agree.checked);
agree.checked = true;
```

## 9.3 폼 요소별 사용 속성

| 요소                 | 속성        |
| ------------------ | --------- |
| `text`, `password` | `value`   |
| `email`, `number`  | `value`   |
| `checkbox`         | `checked` |
| `radio`            | `checked` |
| `select`           | `value`   |
| `textarea`         | `value`   |

## 9.4 value 사용 시 주의점

`type="number"`인 input이라도 `value`는 문자열로 반환된다.

숫자로 사용하려면 `Number()`로 변환해야 한다.

```js
const age = Number(input.value);
```

## 9.5 innerText와 value 차이

| 대상                           | 사용하는 속성     |
| ---------------------------- | ----------- |
| `h1`, `p`, `div` 같은 일반 요소    | `innerText` |
| `input`, `textarea` 같은 입력 요소 | `value`     |

# 10. 스타일 변경

## 10.1 style 속성

JavaScript에서 요소의 CSS 스타일을 직접 변경하려면 `style` 속성을 사용한다.

```js
const title = document.querySelector('#title');

title.style.color = 'red';
title.style.fontSize = '32px';
title.style.backgroundColor = 'yellow';
title.style.padding = '20px';
title.style.border = '2px solid blue';
```

이 방식은 인라인 스타일로 적용된다.

## 10.2 CSS 속성명 변환

CSS에서 하이픈이 들어간 속성은 JavaScript에서 camelCase로 작성한다.

| CSS                | JavaScript        |
| ------------------ | ----------------- |
| `color`            | `color`           |
| `font-size`        | `fontSize`        |
| `background-color` | `backgroundColor` |
| `border-radius`    | `borderRadius`    |
| `text-align`       | `textAlign`       |
| `z-index`          | `zIndex`          |

## 10.3 단위 주의

크기 값을 지정할 때는 단위를 문자열로 포함해야 한다.

```js
title.style.fontSize = 32;      // 잘못된 예
title.style.fontSize = '32px';  // 올바른 예
```

# 11. 클래스 조작

## 11.1 classList

`style`을 직접 바꾸는 대신 클래스를 추가하거나 제거하는 방식이 더 깔끔하다.

이때 `classList`를 사용한다.

```js
const box = document.querySelector('#box');

box.classList.add('active');
box.classList.remove('active');
box.classList.toggle('hidden');
box.classList.contains('dark');
```

## 11.2 classList 주요 메서드

| 메서드          | 설명             |
| ------------ | -------------- |
| `add()`      | 클래스 추가         |
| `remove()`   | 클래스 제거         |
| `toggle()`   | 있으면 제거, 없으면 추가 |
| `contains()` | 클래스가 있는지 확인    |
| `replace()`  | 클래스 교체         |

## 11.3 style과 classList 비교

| 상황               | 권장 방식       |
| ---------------- | ----------- |
| 스타일 한두 개만 바꾸기    | `style`     |
| 여러 스타일을 한 번에 바꾸기 | `classList` |

실무에서는 `classList` 방식이 더 많이 사용된다.

특히 다크모드, 메뉴 열기와 닫기, 팝업 표시와 숨김 등에 `toggle()`이 자주 사용된다.

# 12. 요소 생성과 삭제

## 12.1 요소 생성

JavaScript로 HTML 요소를 새로 만들 수 있다.

```js
const newLi = document.createElement('li');

newLi.innerText = '새 항목';
newLi.className = 'item';
```

## 12.2 요소 추가

생성한 요소는 부모 요소에 추가해야 화면에 나타난다.

```js
const list = document.querySelector('#list');

list.appendChild(newLi);
```

## 12.3 요소 삭제

요소는 자기 자신을 삭제하거나 부모를 통해 삭제할 수 있다.

```js
newLi.remove();
```

```js
list.removeChild(newLi);
```

## 12.4 주요 메서드

| 메서드               | 설명          |
| ----------------- | ----------- |
| `createElement()` | 새 요소 생성     |
| `appendChild()`   | 맨 뒤에 추가     |
| `prepend()`       | 맨 앞에 추가     |
| `insertBefore()`  | 특정 위치 앞에 추가 |
| `remove()`        | 자기 자신 삭제    |
| `removeChild()`   | 자식 삭제       |

## 12.5 요소 생성 순서

1. `createElement()`로 요소 만들기
2. 속성이나 내용 설정하기
3. `appendChild()` 또는 `prepend()`로 화면에 추가하기

3번을 하지 않으면 화면에 표시되지 않는다.

# 13. 같은 요소는 DOM에 한 자리만 차지

하나의 요소를 두 곳에 추가하면 새로 복사되는 것이 아니라 기존 위치에서 새 위치로 이동한다.

```js
const list = document.querySelector('#list');

const newLi = document.createElement('li');
newLi.innerText = '새 항목';

list.appendChild(newLi);
list.prepend(newLi);
```

위 코드에서는 `newLi`가 뒤에 추가되었다가 다시 맨 앞으로 이동한다.

즉, 같은 요소가 두 개 보이는 것이 아니다.

두 곳에 모두 보이게 하려면 요소를 두 개 만들어야 한다.

```js
const li1 = document.createElement('li');
li1.innerText = '뒤에 추가';

const li2 = document.createElement('li');
li2.innerText = '앞에 추가';

list.appendChild(li1);
list.prepend(li2);
```

# 14. 부모, 자식, 형제 탐색

DOM 트리에서는 현재 요소를 기준으로 부모, 자식, 형제 요소를 찾을 수 있다.

```html
<ul id="list">
  <li>첫째</li>
  <li id="mid">둘째</li>
  <li>셋째</li>
</ul>
```

```js
const mid = document.querySelector('#mid');

console.log(mid.parentElement);
```

## 14.1 주요 탐색 속성

| 관계     | 속성                       |
| ------ | ------------------------ |
| 부모     | `parentElement`          |
| 자식들    | `children`               |
| 첫 자식   | `firstElementChild`      |
| 마지막 자식 | `lastElementChild`       |
| 이전 형제  | `previousElementSibling` |
| 다음 형제  | `nextElementSibling`     |

## 14.2 Node와 Element

`Node`는 텍스트, 공백까지 포함한다.

`Element`는 HTML 요소만 의미한다.

일반적인 DOM 조작에서는 `parentElement`, `children`, `nextElementSibling`처럼 Element 버전을 사용하는 것이 편하다.

# 15. JavaScript로 화면 다루는 흐름

DOM 조작의 기본 흐름은 3단계로 정리할 수 있다.

## 15.1 선택

HTML 요소를 가져온다.

```js
const title = document.querySelector('#title');
```

## 15.2 변경

요소의 내용, 속성, 스타일 등을 변경한다.

```js
title.innerText = '바뀐 제목';
```

## 15.3 반영

변경한 내용은 화면에 자동으로 반영된다.

추가 코드가 필요하지 않다.

## 15.4 DOM 조작 기본 패턴

```js
const btn = document.querySelector('#dark-btn');
const body = document.querySelector('body');

btn.addEventListener('click', function() {
  body.classList.toggle('dark');
});
```

이 흐름은 다음과 같이 이해할 수 있다.

1. `querySelector()`로 요소 선택
2. 속성, 내용, 스타일 변경
3. 화면에 자동 반영

# 16. 실습 1: 다크모드 토글

버튼을 클릭하면 페이지가 다크모드로 바뀌고, 다시 클릭하면 원래대로 돌아오는 기능이다.

```js
const btn = document.querySelector('#btn');
const body = document.querySelector('body');

btn.addEventListener('click', function() {
  body.classList.toggle('dark');

  if (body.classList.contains('dark')) {
    btn.innerText = '다크모드 끄기';
  } else {
    btn.innerText = '다크모드 켜기';
  }
});
```

## 핵심 개념

* `querySelector()`로 버튼과 body 선택
* `classList.toggle()`로 `dark` 클래스 추가 또는 제거
* `classList.contains()`로 현재 다크모드 상태 확인
* 상태에 따라 버튼 텍스트 변경

`addEventListener()`는 이벤트 처리 문법이며, 다음 강의에서 더 자세히 다룬다.

# 17. 실습 2: 회원가입 폼 입력값 확인

버튼을 클릭하면 입력된 모든 값을 가져와 `alert()`으로 표시하는 실습이다.

```js
const submitBtn = document.querySelector('#submit');

submitBtn.addEventListener('click', function() {
  const name = document.querySelector('#name').value;
  const email = document.querySelector('#email').value;
  const age = document.querySelector('#age').value;
  const city = document.querySelector('#city').value;
  const agree = document.querySelector('#agree').checked;

  const g = document.querySelector('input[name="g"]:checked');
  const gender = g ? g.value : '미선택';

  alert(
    `이름: ${name}\n` +
    `이메일: ${email}\n` +
    `나이: ${age}\n` +
    `성별: ${gender}\n` +
    `거주: ${city}\n` +
    `동의: ${agree}`
  );
});
```

## 핵심 개념

* 텍스트 입력값은 `.value`로 가져온다.
* 체크박스 선택 여부는 `.checked`로 가져온다.
* 라디오 버튼은 `input[name="g"]:checked` 선택자를 사용해 선택된 항목만 가져온다.
* 선택된 라디오 버튼이 없을 수 있으므로 삼항 연산자로 처리한다.

```js
const gender = g ? g.value : '미선택';
```

# 19. 시험 대비 요약

## class

`class`는 객체를 만들기 위한 틀이다.
`constructor`는 객체 생성 시 자동 실행된다.
`this`는 새로 생성되는 객체를 의미한다.

## DOM

DOM은 HTML 문서를 JavaScript가 조작할 수 있도록 객체 형태로 표현한 구조이다.
JavaScript는 `document` 객체를 통해 DOM에 접근한다.

## querySelector

CSS 선택자를 사용해 요소를 선택한다.
하나만 선택할 때는 `querySelector()`, 여러 개를 선택할 때는 `querySelectorAll()`을 사용한다.

## innerText / innerHTML / textContent

* `innerText`: 화면에 보이는 텍스트
* `innerHTML`: HTML 태그 포함
* `textContent`: 요소 안의 모든 텍스트

## value / checked

* 입력 요소의 값은 `value`
* 체크 여부는 `checked`

## style / classList

* 간단한 스타일 변경은 `style`
* 여러 스타일을 묶어 제어할 때는 `classList`

## DOM 조작 흐름

```text
요소 선택 → 내용/속성/스타일 변경 → 화면 자동 반영
```
