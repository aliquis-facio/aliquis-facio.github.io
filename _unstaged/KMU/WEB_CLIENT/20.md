# 2. 스크롤 이벤트

## 2.1 스크롤 이벤트 개념

스크롤 이벤트는 사용자가 페이지를 위아래 또는 좌우로 스크롤할 때 발생하는 이벤트이다.

주로 다음과 같은 기능에 사용된다.

* 스크롤 위치에 따라 헤더 모양 변경
* 무한 스크롤
* 위로 가기 버튼
* 특정 위치 도달 감지
* 섹션 이동 효과

스크롤 이벤트는 페이지에 스크롤할 공간이 있어야 발생한다.
즉, 페이지가 너무 짧으면 스크롤바가 생기지 않기 때문에 스크롤 이벤트도 발생하지 않는다.

테스트를 위해서는 다음과 같이 페이지 높이를 충분히 길게 만들 수 있다.

```css
body {
  height: 2000px;
}
```

## 2.2 스크롤 관련 속성 / 메서드

| 속성 / 메서드                     | 설명            |
| ---------------------------- | ------------- |
| `window.scrollY`             | 세로 스크롤 위치     |
| `window.scrollX`             | 가로 스크롤 위치     |
| `window.innerHeight`         | 브라우저 화면 높이    |
| `document.body.scrollHeight` | 전체 페이지 높이     |
| `scrollTo(x, y)`             | 특정 위치로 스크롤    |
| `scrollIntoView()`           | 특정 요소 위치로 스크롤 |

## 2.3 스크롤 위치 출력 예제

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
</head>
<body>
  <h1>스크롤 테스트</h1>

  <div style="height: 2000px; background: linear-gradient(#eef, #fee);">
    스크롤해보세요
  </div>

  <script src="script.js"></script>
</body>
</html>
```

```js
window.addEventListener("scroll", () => {
  console.log("스크롤 위치:", window.scrollY);
});
```

스크롤하면 `window.scrollY` 값이 0부터 점점 증가한다.

## 2.4 주의점

스크롤 이벤트는 짧은 시간에 수십~수백 번 발생할 수 있다.
따라서 스크롤 이벤트 안에서 너무 무거운 작업을 수행하면 성능 문제가 발생할 수 있다.

# 3. 스크롤 응용 — 헤더 작아지기

## 3.1 개념

페이지를 일정 거리 이상 스크롤하면 상단 헤더가 작아지는 패턴은 실제 웹사이트에서 자주 사용된다.

핵심은 다음 두 가지이다.

* `position: fixed`로 헤더를 화면 상단에 고정
* 스크롤 위치에 따라 클래스를 추가하거나 제거

## 3.2 예제 코드

```html
<header id="header">상단 헤더</header>

<div style="height: 2000px; padding: 80px 20px;">
  스크롤해서 100px 넘기면 헤더가 작아져요
</div>
```

```css
#header {
  position: fixed;
  top: 0;
  left: 0;
  right: 0;

  background: #1f3864;
  color: #fff;

  padding: 20px;
  font-size: 24px;
  transition: all 0.3s;
}

#header.shrink {
  padding: 4px;
  font-size: 14px;
}
```

```js
const header = document.getElementById("header");

window.addEventListener("scroll", () => {
  if (window.scrollY > 100) {
    header.classList.add("shrink");
  } else {
    header.classList.remove("shrink");
  }
});
```

## 3.3 핵심 정리

`position: fixed`가 없으면 스크롤할 때 헤더가 같이 올라가 사라진다.
`transition`을 사용하면 헤더 크기 변화가 부드럽게 보인다.

# 4. `scrollTo` / `scrollIntoView`

## 4.1 `scrollTo`

`scrollTo()`는 특정 좌표로 스크롤할 때 사용한다.
대표적으로 “맨 위로 가기” 버튼에 사용된다.

```js
window.scrollTo({
  top: 0,
  behavior: "smooth"
});
```

`behavior: "smooth"` 옵션을 사용하면 스크롤이 부드럽게 이동한다.

## 4.2 맨 위로 가기 버튼 예제

```html
<button id="topBtn" style="position: fixed; bottom: 20px; right: 20px;">
  ▲ 위로
</button>

<div style="height: 2000px;">긴 컨텐츠</div>

<script src="script.js"></script>
```

```js
const topBtn = document.getElementById("topBtn");

topBtn.addEventListener("click", () => {
  window.scrollTo({
    top: 0,
    behavior: "smooth"
  });
});
```

## 4.3 `scrollIntoView`

`scrollIntoView()`는 특정 요소가 보이는 위치로 스크롤할 때 사용한다.

```js
target.scrollIntoView({ behavior: "smooth" });
```

## 4.4 특정 섹션으로 이동 예제

```html
<a id="goto">3번 섹션으로 이동</a>

<section id="section1"><h2>1번</h2></section>
<section id="section2"><h2>2번</h2></section>
<section id="section3"><h2>3번 도착!</h2></section>
```

```css
#goto {
  display: inline-block;
  padding: 10px 20px;
  background: #2e74b5;
  color: #fff;
  cursor: pointer;
  border-radius: 6px;
}

section {
  height: 100vh;
  padding: 20px;
}
```

```js
const goto = document.getElementById("goto");

goto.addEventListener("click", () => {
  const target = document.getElementById("section3");
  target.scrollIntoView({ behavior: "smooth" });
});
```

## 4.5 `100vh`

`100vh`는 화면 높이의 100%를 의미한다.
섹션을 한 화면 크기로 만들 때 자주 사용된다.

# 5. 윈도우 / 문서 이벤트

## 5.1 개념

윈도우 / 문서 이벤트는 개별 요소가 아니라 페이지 전체 또는 브라우저 창 단위에서 발생하는 이벤트이다.

## 5.2 주요 이벤트

| 이벤트                  | 발생 시점               | 주 사용처      |
| -------------------- | ------------------- | ---------- |
| `load`               | 이미지, CSS까지 모두 로드 완료 | 초기화        |
| `DOMContentLoaded`   | HTML 파싱 완료          | 빠른 초기화     |
| `resize`             | 창 크기 변경             | 반응형 처리     |
| `beforeunload`       | 페이지를 떠나기 직전         | 저장 안내      |
| `visibilitychange`   | 탭 전환 / 최소화          | 일시정지, 알림   |
| `online` / `offline` | 인터넷 연결 / 끊김         | 네트워크 상태 표시 |

## 5.3 `load`와 `DOMContentLoaded`

```js
window.addEventListener("load", () => {
  console.log("모두 로드됨");
});

document.addEventListener("DOMContentLoaded", () => {
  console.log("HTML 파싱 완료");
});
```

`DOMContentLoaded`는 이미지 로딩을 기다리지 않기 때문에 `load`보다 빠르게 실행된다.

## 5.4 창 크기 변경 감지

```js
window.addEventListener("resize", () => {
  console.log(window.innerWidth, window.innerHeight);
});
```

## 5.5 페이지 떠나기 경고

```js
window.addEventListener("beforeunload", (e) => {
  e.preventDefault();
  e.returnValue = "";
});
```

새로고침하거나 탭을 닫기 전에 경고창을 띄울 수 있다.

## 5.6 탭 전환 감지

```js
document.addEventListener("visibilitychange", () => {
  if (document.hidden) {
    console.log("다른 탭으로 갔어요");
  } else {
    console.log("돌아왔어요!");
  }
});
```

게임이나 영상 재생 페이지에서 탭이 바뀌면 일시정지하는 기능에 사용할 수 있다.

## 5.7 네트워크 상태 감지

```js
window.addEventListener("online", () => alert("인터넷 연결됨"));
window.addEventListener("offline", () => alert("인터넷 끊김!"));
```

# 6. 터치 이벤트

## 6.1 개념

터치 이벤트는 사용자가 손가락으로 화면을 터치할 때 발생한다.
스마트폰이나 태블릿에서 주로 사용된다.

PC에서는 일반적으로 발생하지 않지만, 개발자 도구의 모바일 모드에서 테스트할 수 있다.

## 6.2 주요 터치 이벤트

| 이벤트           | 발생 시점           |
| ------------- | --------------- |
| `touchstart`  | 손가락이 화면에 닿음     |
| `touchmove`   | 손가락이 화면 위에서 움직임 |
| `touchend`    | 손가락이 화면에서 떨어짐   |
| `touchcancel` | 터치가 취소됨         |

## 6.3 주요 속성

| 속성                 | 설명                 |
| ------------------ | ------------------ |
| `e.touches`        | 현재 화면 위에 있는 모든 손가락 |
| `e.changedTouches` | 방금 변화한 손가락         |
| `touch.clientX`    | 터치한 손가락의 X 좌표      |
| `touch.clientY`    | 터치한 손가락의 Y 좌표      |

터치는 여러 손가락을 동시에 사용할 수 있기 때문에 `e.touches`를 배열처럼 다룬다.

## 6.4 터치 시작 위치 예제

```html
<div id="box" style="height: 200px; background: #eef;"></div>
```

```js
const box = document.getElementById("box");

box.addEventListener("touchstart", (e) => {
  const touch = e.touches[0];
  console.log("터치 시작:", touch.clientX, touch.clientY);
});
```

## 6.5 손가락 움직임 추적

```js
box.addEventListener("touchmove", (e) => {
  e.preventDefault();

  const touch = e.touches[0];
  console.log("이동 중:", touch.clientX, touch.clientY);
});
```

`e.preventDefault()`를 사용하면 터치 중 페이지 스크롤을 막을 수 있다.

## 6.6 스와이프 방향 감지

```js
let startX = 0;

box.addEventListener("touchstart", (e) => {
  startX = e.touches[0].clientX;
});

box.addEventListener("touchend", (e) => {
  const endX = e.changedTouches[0].clientX;
  const diff = endX - startX;

  if (diff > 50) alert("오른쪽 스와이프!");
  if (diff < -50) alert("왼쪽 스와이프!");
});
```

## 6.7 손가락 개수 확인

```js
box.addEventListener("touchstart", (e) => {
  console.log("손가락 개수:", e.touches.length);

  if (e.touches.length === 2) {
    console.log("핀치 시작?");
  }
});
```

# 7. 폼 이벤트와 `preventDefault`

## 7.1 폼 이벤트 개념

폼 이벤트는 사용자가 입력창, 선택창, 버튼 등을 조작할 때 발생한다.

## 7.2 주요 폼 이벤트

| 이벤트      | 발생 시점               |
| -------- | ------------------- |
| `submit` | 폼 제출                |
| `input`  | 값이 바뀔 때마다 실시간 발생    |
| `change` | 값 변경 후 포커스가 빠질 때 발생 |
| `focus`  | 포커스가 들어옴            |
| `blur`   | 포커스가 빠짐             |

## 7.3 `input`과 `change` 차이

`input`은 사용자가 타이핑할 때마다 즉시 발생한다.
`change`는 값이 바뀐 뒤 포커스가 빠질 때 한 번 발생한다.

## 7.4 `preventDefault`

`preventDefault()`는 이벤트의 기본 동작을 막는다.

대표적인 예시는 다음과 같다.

* 폼 제출 막기
* 링크 이동 막기
* 터치 스크롤 막기

## 7.5 폼 제출 검증 예제

```html
<form id="loginForm">
  <input id="user" placeholder="아이디">
  <input id="pw" type="password" placeholder="비밀번호">
  <button type="submit">로그인</button>
</form>
```

```js
const form = document.getElementById("loginForm");

form.addEventListener("submit", (e) => {
  const user = document.getElementById("user").value;
  const pw = document.getElementById("pw").value;

  if (user === "" || pw === "") {
    e.preventDefault();
    alert("모두 입력해주세요");
  }
});
```

## 7.6 실시간 글자 수 표시

```html
<textarea id="intro"></textarea>
<span id="counter">0</span>
```

```js
const intro = document.getElementById("intro");
const counter = document.getElementById("counter");

intro.addEventListener("input", (e) => {
  counter.textContent = e.target.value.length;
});
```

## 7.7 포커스 안내

```html
<input id="email" placeholder="이메일">
<div id="hint" style="display: none;">example@email.com 형식</div>
```

```js
const email = document.getElementById("email");
const hint = document.getElementById("hint");

email.addEventListener("focus", () => {
  hint.style.display = "block";
});

email.addEventListener("blur", () => {
  hint.style.display = "none";
});
```

# 8. 이벤트 버블링과 `stopPropagation`

## 8.1 이벤트 버블링

이벤트 버블링은 자식 요소에서 발생한 이벤트가 부모 요소로 전달되는 현상이다.

예를 들어 버튼이 `inner` 안에 있고, `inner`가 다시 `outer` 안에 있다면 버튼 클릭 시 이벤트 흐름은 다음과 같다.

```text
button → inner → outer
```

## 8.2 버블링 예제

```html
<div id="outer">
  outer
  <div id="inner">
    inner
    <button id="btn">클릭</button>
  </div>
</div>
```

```js
const outer = document.getElementById("outer");
const inner = document.getElementById("inner");
const btn = document.getElementById("btn");

outer.addEventListener("click", () => console.log("outer"));
inner.addEventListener("click", () => console.log("inner"));
btn.addEventListener("click", () => console.log("btn"));
```

버튼을 클릭하면 다음 순서로 출력된다.

```text
btn
inner
outer
```

## 8.3 `stopPropagation`

`stopPropagation()`은 이벤트가 부모 요소로 전달되는 것을 막는다.

```js
btn.addEventListener("click", (e) => {
  e.stopPropagation();
  console.log("btn");
});
```

이 경우 버튼 클릭 시 `btn`만 출력되고, `inner`, `outer`는 실행되지 않는다.

## 8.4 활용 예시

이벤트 버블링과 `stopPropagation()`은 다음 기능에서 자주 사용된다.

* 모달 바깥 클릭 시 닫기
* 드롭다운 메뉴 닫기
* 카드 내부 버튼 클릭 시 카드 클릭 이벤트 막기
* 메뉴 영역 안쪽 클릭은 유지하고 바깥 클릭은 닫기

# 9. 요소 보이기 / 숨기기와 `classList`

## 9.1 개념

요소 보이기 / 숨기기는 메뉴 토글, 모달, 탭 전환, 다크모드 등에 자주 사용된다.

구현 방식은 크게 두 가지이다.

1. `style.display` 직접 변경
2. `classList`로 클래스 토글

일반적으로 `classList` 방식이 더 권장된다.

---

## 9.2 `style.display` 직접 변경

```html
<button id="toggleBtn">메뉴 보기</button>
<div id="menu" style="display: none;">숨겨진 메뉴</div>
```

```js
const btn = document.getElementById("toggleBtn");
const menu = document.getElementById("menu");

btn.addEventListener("click", () => {
  if (menu.style.display === "none") {
    menu.style.display = "block";
  } else {
    menu.style.display = "none";
  }
});
```

## 9.3 `display` 값

| 값        | 효과                  |
| -------- | ------------------- |
| `none`   | 완전히 숨김. 공간도 차지하지 않음 |
| `block`  | 한 줄을 차지하며 표시        |
| `flex`   | flex 컨테이너로 표시       |
| `inline` | 한 줄 안에서 표시          |

---

## 9.4 `classList` 토글 방식

```html
<button id="menuBtn">메뉴</button>
<nav id="sideMenu">메뉴 내용</nav>
```

```css
#sideMenu {
  display: none;
}

#sideMenu.open {
  display: block;
}

.dark-mode {
  background: #222;
  color: #eee;
}
```

```js
const menuBtn = document.getElementById("menuBtn");
const sideMenu = document.getElementById("sideMenu");

menuBtn.addEventListener("click", () => {
  sideMenu.classList.toggle("open");
});

document.body.classList.toggle("dark-mode");
```

## 9.5 `classList` 주요 메서드

| 메서드              | 동작                              |
| ---------------- | ------------------------------- |
| `.add("x")`      | 클래스 `x` 추가                      |
| `.remove("x")`   | 클래스 `x` 제거                      |
| `.toggle("x")`   | 있으면 제거, 없으면 추가                  |
| `.contains("x")` | 클래스 존재 여부를 `true` / `false`로 반환 |

## 9.6 핵심 정리

`classList.toggle()`을 사용하면 보이기 / 숨기기, 활성 / 비활성, 다크모드 전환 등을 간단하게 구현할 수 있다.

스타일은 CSS에 두고, 동작은 JS에 두는 방식이 더 깔끔하다.

# 10. `alert`, `confirm`, `prompt`

## 10.1 개념

`alert`, `confirm`, `prompt`는 브라우저에서 기본으로 제공하는 팝업 함수이다.

간단한 상호작용에는 유용하지만, 디자인을 세밀하게 조정하기 어렵기 때문에 실제 서비스에서는 직접 만든 모달을 사용하는 경우가 많다.

## 10.2 주요 함수

| 함수          | 반환값              | 용도         |
| ----------- | ---------------- | ---------- |
| `alert()`   | 없음               | 알림 표시      |
| `confirm()` | `true` / `false` | 예 / 아니오 확인 |
| `prompt()`  | 입력값 / `null`     | 짧은 입력 받기   |

## 10.3 `alert`

```js
alert("저장되었습니다");
```

확인 버튼 하나만 있는 알림창을 띄운다.

## 10.4 `confirm`

```js
const ok = confirm("정말 삭제하시겠습니까?");

if (ok) {
  console.log("삭제 진행");
} else {
  console.log("취소됨");
}
```

사용자가 확인을 누르면 `true`, 취소를 누르면 `false`가 반환된다.

## 10.5 `prompt`

```js
const name = prompt("이름을 입력하세요");

if (name) {
  alert("안녕하세요 " + name + "님!");
} else {
  alert("이름을 입력해주세요");
}
```

입력값이 있으면 문자열을 반환하고, 취소를 누르면 `null`을 반환한다.

## 10.6 버튼과 함께 사용

```html
<button id="deleteBtn">삭제</button>
```

```js
const deleteBtn = document.getElementById("deleteBtn");

deleteBtn.addEventListener("click", () => {
  const ok = confirm("정말 삭제?");

  if (ok) {
    alert("삭제 완료!");
  }
});
```

# 11. 모달 만들기

## 11.1 모달 개념

모달은 화면 가운데 뜨고, 뒤쪽 영역을 어둡게 가리는 팝업 창이다.

모달 구현에는 다음 개념이 함께 사용된다.

* `position: fixed`
* `z-index`
* `classList`
* 이벤트 버블링
* `stopPropagation`

## 11.2 HTML 구조

```html
<button id="openBtn">모달 열기</button>

<div id="modal" class="modal-overlay">
  <div id="modalBox" class="modal-box">
    <h3>알림</h3>
    <p>정말 삭제하시겠습니까?</p>
    <button id="closeBtn">닫기</button>
  </div>
</div>
```

## 11.3 CSS

```css
.modal-overlay {
  display: none;

  position: fixed;
  top: 0;
  left: 0;
  right: 0;
  bottom: 0;

  background: rgba(0, 0, 0, 0.5);
  z-index: 1000;

  justify-content: center;
  align-items: center;
}

.modal-overlay.show {
  display: flex;
}

.modal-box {
  background: #fff;
  padding: 30px;
  border-radius: 8px;
}
```

## 11.4 JavaScript

```js
const openBtn = document.getElementById("openBtn");
const closeBtn = document.getElementById("closeBtn");
const modal = document.getElementById("modal");
const modalBox = document.getElementById("modalBox");

openBtn.addEventListener("click", () => {
  modal.classList.add("show");
});

closeBtn.addEventListener("click", () => {
  modal.classList.remove("show");
});

modal.addEventListener("click", () => {
  modal.classList.remove("show");
});

modalBox.addEventListener("click", (e) => {
  e.stopPropagation();
});
```

## 11.5 모달 구현 핵심

| 코드                  | 역할                   |
| ------------------- | -------------------- |
| `position: fixed`   | 화면에 고정               |
| `z-index: 1000`     | 다른 요소 위에 표시          |
| `display: none`     | 기본 상태에서 숨김           |
| `.show`             | 모달을 보이게 함            |
| `stopPropagation()` | 모달 안쪽 클릭 시 닫히지 않도록 함 |

## 11.6 ESC 키로 모달 닫기

```js
document.addEventListener("keydown", (e) => {
  if (e.key === "Escape") {
    modal.classList.remove("show");
  }
});
```

# 12. 타이머 — `setTimeout` / `setInterval`

## 12.1 개념

타이머 함수는 일정 시간이 지난 뒤 코드를 실행하거나, 일정 시간마다 코드를 반복 실행할 때 사용한다.

대표적인 사용 예시는 다음과 같다.

* 자동 알림
* 카운트다운
* 슬라이드쇼
* 실시간 시계
* 자동 저장
* 반복 애니메이션

## 12.2 주요 함수

| 함수                    | 동작                  |
| --------------------- | ------------------- |
| `setTimeout(fn, ms)`  | `ms` 후 `fn`을 한 번 실행 |
| `setInterval(fn, ms)` | `ms`마다 `fn`을 반복 실행  |
| `clearTimeout(id)`    | `setTimeout` 취소     |
| `clearInterval(id)`   | `setInterval` 중지    |

`1000ms`는 1초이다.

## 12.3 3초 후 알림

```js
setTimeout(() => {
  alert("3초 지났어요!");
}, 3000);
```

## 12.4 1초마다 반복 실행

```html
<div id="count">0</div>
```

```js
const count = document.getElementById("count");

let n = 0;

const timerId = setInterval(() => {
  n++;
  count.textContent = n;
}, 1000);

setTimeout(() => {
  clearInterval(timerId);
}, 5000);
```

## 12.5 카운트다운

```html
<div id="display">10</div>
```

```js
const display = document.getElementById("display");

let count = 10;

const id = setInterval(() => {
  display.textContent = count;
  count--;

  if (count < 0) {
    clearInterval(id);
    alert("종료!");
  }
}, 1000);
```

## 12.6 주의점

`setInterval()`을 사용한 뒤에는 필요할 때 반드시 `clearInterval()`로 반복을 멈춰야 한다.
그렇지 않으면 계속 실행되어 성능 문제나 의도하지 않은 동작이 발생할 수 있다.

# 13. `Date` 객체

## 13.1 개념

`Date` 객체는 JavaScript에서 날짜와 시간을 다룰 때 사용하는 내장 객체이다.

다음과 같은 작업에 사용된다.

* 현재 시각 가져오기
* 날짜 출력
* 시간 차이 계산
* D-Day 계산
* 실시간 시계 만들기

## 13.2 `Date` 객체 생성

```js
const now = new Date();
```

## 13.3 주요 메서드

| 메서드                    | 반환값                |
| ---------------------- | ------------------ |
| `getFullYear()`        | 연도                 |
| `getMonth()`           | 월. 0부터 시작          |
| `getDate()`            | 일                  |
| `getDay()`             | 요일. 0은 일요일, 6은 토요일 |
| `getHours()`           | 시                  |
| `getMinutes()`         | 분                  |
| `getSeconds()`         | 초                  |
| `getTime()`            | 1970년부터 현재까지의 밀리초  |
| `toLocaleDateString()` | 지역 형식의 날짜 문자열      |
| `toLocaleTimeString()` | 지역 형식의 시간 문자열      |

## 13.4 `getMonth()` 주의점

`getMonth()`는 0부터 시작한다.

```text
1월 = 0
2월 = 1
...
12월 = 11
```

따라서 실제 월을 표시하려면 `getMonth() + 1`을 사용해야 한다.

## 13.5 현재 시각 출력

```js
const now = new Date();

console.log(now);
console.log(now.getFullYear());
console.log(now.getMonth() + 1);
console.log(now.toLocaleString());
```

## 13.6 실시간 시계

```html
<div id="clock"></div>
```

```js
const clock = document.getElementById("clock");

setInterval(() => {
  const now = new Date();
  clock.textContent = now.toLocaleTimeString();
}, 1000);
```

## 13.7 특정 날짜 만들기

```js
const xmas = new Date(2026, 11, 25);

console.log(xmas.toLocaleDateString());
```

월은 0부터 시작하므로 12월을 나타내려면 `11`을 사용한다.

## 13.8 D-Day 계산

```js
const today = new Date();
const exam = new Date(2026, 5, 5);

const diffMs = exam - today;
const diffDays = Math.floor(diffMs / (1000 * 60 * 60 * 24));

console.log("기말고사까지 D-" + diffDays);
```

날짜 객체끼리 빼면 밀리초 단위 차이가 계산된다.
이를 하루 단위로 바꾸려면 다음 값으로 나누면 된다.

```js
1000 * 60 * 60 * 24
```

# 14. 페이지 이동 — `location` / `history`

## 14.1 개념

페이지 이동은 멀티 페이지 웹사이트에서 핵심적인 기능이다.

페이지 이동 방식은 크게 다음 두 가지가 있다.

* HTML의 `<a>` 태그 사용
* JavaScript의 `location`, `history` 객체 사용

---

## 14.2 HTML `<a>` 태그

### 같은 폴더의 페이지로 이동

```html
<a href="about.html">소개</a>
```

### 새 탭으로 열기

```html
<a href="https://kookmin.ac.kr" target="_blank">학교</a>
```

### 이미지를 링크로 사용

```html
<a href="index.html">
  <img src="./images/logo.png">
</a>
```

### 같은 페이지의 특정 위치로 이동

```html
<a href="#section3">3번 섹션</a>

<section id="section3">
  ...
</section>
```

### 이메일 / 전화 링크

```html
<a href="mailto:abc@test.com">메일</a>
<a href="tel:01012345678">전화</a>
```

---

## 14.3 JavaScript `location` 객체

`location` 객체는 현재 URL 정보를 읽거나, 다른 페이지로 이동할 때 사용한다.

### 현재 URL 정보 읽기

```js
console.log(location.href);
console.log(location.host);
console.log(location.pathname);
console.log(location.search);
console.log(location.hash);
```

| 속성                  | 설명     |
| ------------------- | ------ |
| `location.href`     | 전체 URL |
| `location.host`     | 호스트    |
| `location.pathname` | 경로     |
| `location.search`   | 쿼리스트링  |
| `location.hash`     | 해시값    |

### 페이지 이동

```js
location.href = "main.html";
```

### 새로고침

```js
location.reload();
```

### 조건부 이동 예제

```html
<button id="loginBtn">로그인</button>
```

```js
const loginBtn = document.getElementById("loginBtn");

loginBtn.addEventListener("click", () => {
  const ok = true;

  if (ok) {
    location.href = "main.html";
  } else {
    alert("로그인 실패");
  }
});
```

`<a href>`는 단순 이동에 적합하고, `location.href`는 조건 검사 후 이동할 때 적합하다.

---

## 14.4 JavaScript `history` 객체

`history` 객체는 브라우저의 방문 기록을 이용해 앞뒤로 이동할 때 사용한다.

```js
history.back();
history.forward();
history.go(-1);
history.go(1);
```

## 14.5 `history` 주요 메서드

| 메서드 / 속성            | 설명                        |
| ------------------- | ------------------------- |
| `history.back()`    | 이전 페이지로 이동                |
| `history.forward()` | 다음 페이지로 이동                |
| `history.go(n)`     | `n`만큼 이동. 음수는 뒤로, 양수는 앞으로 |
| `history.length`    | 방문 기록 개수                  |

예시는 다음과 같다.

```js
history.back();
history.go(-1);

history.forward();
history.go(1);

history.go(2);
```

# 15. 핵심 요약

## 15.1 이벤트 처리 핵심

이번 강의의 핵심은 사용자의 행동과 브라우저 환경 변화에 반응하여 페이지를 동적으로 바꾸는 것이다.

주요 이벤트는 다음과 같다.

* `scroll`
* `resize`
* `DOMContentLoaded`
* `touchstart`
* `touchmove`
* `touchend`
* `submit`
* `input`
* `focus`
* `blur`
* `click`
* `keydown`

## 15.2 자주 쓰이는 메서드

| 메서드                  | 용도            |
| -------------------- | ------------- |
| `addEventListener()` | 이벤트 등록        |
| `preventDefault()`   | 기본 동작 막기      |
| `stopPropagation()`  | 이벤트 버블링 막기    |
| `classList.add()`    | 클래스 추가        |
| `classList.remove()` | 클래스 제거        |
| `classList.toggle()` | 클래스 토글        |
| `scrollTo()`         | 특정 좌표로 스크롤    |
| `scrollIntoView()`   | 특정 요소로 스크롤    |
| `setTimeout()`       | 일정 시간 후 1회 실행 |
| `setInterval()`      | 일정 시간마다 반복 실행 |

## 15.3 구현 관점에서 중요한 점

* 스크롤 이벤트는 매우 자주 발생하므로 무거운 작업을 피해야 한다.
* 폼 제출을 막으려면 `submit` 이벤트에서 `preventDefault()`를 사용한다.
* 모달 바깥 클릭 닫기는 이벤트 버블링을 활용한다.
* 모달 내부 클릭이 바깥으로 전달되지 않게 하려면 `stopPropagation()`을 사용한다.
* 스타일 변경은 직접 `style`을 바꾸는 것보다 `classList`로 CSS 클래스를 토글하는 방식이 좋다.
* `setInterval()`은 필요할 때 반드시 `clearInterval()`로 정리한다.
* `Date.getMonth()`는 0부터 시작하므로 실제 월을 표시할 때 `+1`이 필요하다.
* 단순 페이지 이동은 `<a href>`, 조건부 이동은 `location.href`를 사용하면 좋다.
