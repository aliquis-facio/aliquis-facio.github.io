이번 강의에서는 JavaScript에서 웹페이지를 동적으로 제어하기 위한 **이벤트 처리**의 기본 개념을 학습한다.

주요 내용은 다음과 같다.

- JS / CSS 파일 분리와 폴더 구조

## 2. JS / CSS 파일 분리와 폴더 구조

HTML 코드가 길어지면 모든 코드를 HTML 파일 안에 작성하지 않고, CSS와 JavaScript를 외부 파일로 분리하는 것이 좋다.

### 파일 분리의 장점

- 코드 유지보수가 쉬워진다.
- 협업할 때 역할 분담이 명확해진다.
- 여러 HTML 페이지에서 같은 CSS / JS 파일을 재사용할 수 있다.
- HTML은 구조, CSS는 디자인, JS는 동작을 담당하도록 역할을 나눌 수 있다.

### 기본 연결 방식

```html
<link rel="stylesheet" href="style.css">
<script src="script.js"></script>
````

CSS 파일은 일반적으로 `<head>` 안에 연결한다.

JavaScript 파일은 보통 `</body>` 직전에 연결한다.
그 이유는 HTML 요소가 생성되기 전에 JS가 실행되면 `getElementById()` 등이 `null`을 반환할 수 있기 때문이다.

### defer 속성

`script`를 `<head>` 안에 두고 싶다면 `defer` 속성을 사용할 수 있다.

```html
<script src="script.js" defer></script>
```

`defer`를 사용하면 HTML 문서를 모두 읽은 뒤 JS가 실행된다.

### 권장 폴더 구조

```text
my-project/
├─ index.html
├─ style.css
├─ script.js
└─ images/
   └─ logo.png
```

### 상대 경로

* `./` : 현재 폴더
* `../` : 상위 폴더

## 3. 이벤트란?

이벤트는 사용자가 웹페이지에서 발생시키는 동작을 의미한다.

예를 들어 다음과 같은 동작들이 이벤트에 해당한다.

* 버튼 클릭
* 입력창에 글자 입력
* 페이지 스크롤
* 마우스 올리기
* 키보드 누르기

이벤트가 발생하면 브라우저는 이를 감지하고, 미리 등록된 JavaScript 함수를 실행한다.

### 이벤트 처리 흐름

```text
① 사용자 동작 발생
        ↓
② 브라우저가 이벤트 감지
        ↓
③ 등록된 함수 실행
```

### 이벤트 예시

| 사용자 동작     | 실행되는 기능  |
| ---------- | -------- |
| 로그인 버튼 클릭  | 로그인 처리   |
| 검색창 입력     | 자동완성 표시  |
| 메뉴에 마우스 올림 | 서브메뉴 펼침  |
| 페이지 스크롤    | 헤더 색상 변경 |

이벤트는 정적인 HTML 페이지를 동적으로 움직이는 웹페이지로 만드는 핵심 개념이다.

---

## 4. 이벤트 등록 방법

이벤트를 등록하는 방식은 크게 두 가지가 있다.

1. HTML 속성 방식
2. `addEventListener()` 방식

---

### 4.1 HTML 속성 방식

HTML 태그 안에 직접 이벤트 속성을 작성하는 방식이다.

```html
<button onclick="sayHi()">인사</button>
```

```js
function sayHi() {
  alert("안녕!");
}
```

### 특징

* 코드가 짧고 직관적이다.
* 간단한 예제에서는 사용하기 쉽다.
* 하지만 HTML과 JS가 섞이기 때문에 실제 프로젝트에서는 권장되지 않는다.

---

### 4.2 addEventListener 방식

JavaScript 파일에서 이벤트를 따로 등록하는 방식이다.

```html
<button id="btn">인사</button>
```

```js
const btn = document.getElementById("btn");

btn.addEventListener("click", () => {
  alert("안녕!");
});
```

### 특징

* HTML과 JavaScript를 분리할 수 있다.
* 하나의 요소에 여러 이벤트를 등록할 수 있다.
* 실제 프로젝트에서는 이 방식을 사용하는 것이 권장된다.

### 비교

| 구분    | HTML 속성 방식   | addEventListener           |
| ----- | ------------ | -------------------------- |
| 장점    | 코드가 짧고 직관적   | 이벤트 여러 개 등록 가능, JS/HTML 분리 |
| 단점    | HTML과 JS가 섞임 | 코드가 조금 길어짐                 |
| 추천 상황 | 아주 간단한 동작    | 실제 프로젝트                    |

---

## 5. 이벤트 객체

이벤트 객체는 이벤트가 발생했을 때 브라우저가 자동으로 전달해주는 객체이다.

이벤트 함수의 첫 번째 매개변수로 받을 수 있으며, 보통 `e` 또는 `event`라는 이름을 사용한다.

```js
btn.addEventListener("click", (e) => {
  console.log(e.target);
  console.log(e.type);
});
```

### 주요 속성

| 속성               | 설명          |
| ---------------- | ----------- |
| `e.target`       | 이벤트가 발생한 요소 |
| `e.target.value` | 입력 요소의 현재 값 |
| `e.type`         | 이벤트 종류      |
| `e.key`          | 눌린 키 값      |

---

### 5.1 클릭 이벤트 객체 예시

```html
<button id="btn">클릭</button>
```

```js
const btn = document.getElementById("btn");

btn.addEventListener("click", (e) => {
  console.log(e.target);
  console.log(e.type);
});
```

`e.target`은 클릭된 요소를 의미하고, `e.type`은 이벤트 종류인 `"click"`을 의미한다.

---

### 5.2 입력값 가져오기

입력 이벤트에서는 `e.target.value`를 통해 현재 입력값을 가져올 수 있다.

```html
<input id="myInput" placeholder="타이핑">
```

```js
const input = document.getElementById("myInput");

input.addEventListener("input", (e) => {
  console.log(e.target.value);
});
```

또는 요소에 직접 접근해서 값을 가져올 수도 있다.

```js
const userName = document.getElementById("userName");
console.log(userName.value);
```

---

### 5.3 키보드 이벤트 객체

```js
document.addEventListener("keydown", (e) => {
  if (e.key === "Escape") {
    alert("ESC 키를 눌렀어요!");
  }
});
```

`e.key`를 사용하면 사용자가 어떤 키를 눌렀는지 확인할 수 있다.

---

## 6. 마우스 이벤트

마우스 이벤트는 사용자의 마우스 동작에 따라 발생하는 이벤트이다.

### 주요 마우스 이벤트

| 이벤트           | 발생 시점             |
| ------------- | ----------------- |
| `click`       | 요소를 클릭했을 때        |
| `dblclick`    | 요소를 더블클릭했을 때      |
| `mousedown`   | 마우스 버튼을 눌렀을 때     |
| `mouseup`     | 마우스 버튼을 뗐을 때      |
| `mousemove`   | 마우스가 움직일 때        |
| `mouseover`   | 요소 위로 마우스가 들어왔을 때 |
| `mouseout`    | 요소 밖으로 마우스가 나갔을 때 |
| `mouseenter`  | 요소 위로 마우스가 들어왔을 때 |
| `mouseleave`  | 요소 밖으로 마우스가 나갔을 때 |
| `contextmenu` | 마우스 우클릭을 했을 때     |

---

### 6.1 클릭 이벤트

```html
<button id="btn">클릭</button>
```

```js
const btn = document.getElementById("btn");

btn.addEventListener("click", () => {
  console.log("클릭됨");
});
```

---

### 6.2 더블클릭 이벤트

```html
<img id="img" src="cat.jpg" width="200">
```

```js
const img = document.getElementById("img");

img.addEventListener("dblclick", () => {
  alert("더블클릭!");
});
```

---

### 6.3 마우스 좌표 추적

```html
<div id="box" style="height:200px;background:#eef"></div>
```

```js
const box = document.getElementById("box");

box.addEventListener("mousemove", (e) => {
  console.log(e.clientX, e.clientY);
});
```

`e.clientX`, `e.clientY`는 브라우저 화면 기준 마우스 좌표를 의미한다.

---

### 6.4 마우스 올림 / 내림

```html
<div id="menu">메뉴</div>
```

```js
const menu = document.getElementById("menu");

menu.addEventListener("mouseenter", () => {
  console.log("열림");
});

menu.addEventListener("mouseleave", () => {
  console.log("닫힘");
});
```

드롭다운 메뉴처럼 마우스를 올렸을 때 열리고, 벗어났을 때 닫히는 기능에 사용할 수 있다.

---

### 6.5 우클릭 메뉴 막기

```js
document.addEventListener("contextmenu", (e) => {
  e.preventDefault();
  alert("우클릭 금지!");
});
```

`e.preventDefault()`는 브라우저의 기본 동작을 막는다.

---

## 7. mouseover와 mouseenter의 차이

`mouseover`와 `mouseenter`는 둘 다 마우스가 요소 위로 들어올 때 발생한다.

하지만 자식 요소를 포함하느냐에 차이가 있다.

---

### 7.1 mouseover / mouseout

`mouseover`와 `mouseout`은 자식 요소에 마우스가 들어가도 부모 요소에서 이벤트가 다시 발생한다.

즉, 부모 안에 자식 요소가 있을 때 마우스가 다음과 같이 이동하면 이벤트가 여러 번 발생할 수 있다.

```text
외부 → 부모 → 자식 → 부모 → 외부
```

이 경우 자식 요소로 이동할 때도 부모 기준으로 `mouseout`, `mouseover`가 다시 발생할 수 있다.

---

### 7.2 mouseenter / mouseleave

`mouseenter`와 `mouseleave`는 자식 요소를 무시한다.

즉, 부모 영역에 처음 들어올 때 한 번, 부모 영역을 완전히 벗어날 때 한 번만 발생한다.

드롭다운 메뉴처럼 깜빡임 없이 안정적인 동작이 필요한 경우에는 `mouseenter`와 `mouseleave`를 사용하는 것이 좋다.

---

### 7.3 비교 예시

```html
<div id="parent" style="padding:30px;background:#2e74b5;">
  <div id="child" style="padding:24px;background:#f0c040;">
    child
  </div>
</div>
```

```js
const parent = document.getElementById("parent");

parent.addEventListener("mouseover", () => {
  console.log("over!");
});

parent.addEventListener("mouseout", () => {
  console.log("out!");
});

parent.addEventListener("mouseenter", () => {
  console.log("enter!");
});

parent.addEventListener("mouseleave", () => {
  console.log("leave!");
});
```

### 출력 비교

| 이벤트                | 외부 → 부모 | 부모 → 자식    | 자식 → 부모    | 부모 → 외부 |
| ------------------ | ------- | ---------- | ---------- | ------- |
| `mouseover/out`    | over    | out → over | out → over | out     |
| `mouseenter/leave` | enter   | 발생 안 함     | 발생 안 함     | leave   |

### 핵심 정리

드롭다운 메뉴를 `mouseover/out`으로 만들면 자식 항목 위로 이동할 때마다 메뉴가 깜빡일 수 있다.

따라서 메뉴 구현에는 일반적으로 `mouseenter/leave`가 더 적합하다.

## 8. 키보드 이벤트

키보드 이벤트는 사용자의 키보드 입력에 반응하는 이벤트이다.

주로 다음과 같은 기능에 사용된다.

* 검색창에서 Enter 키 처리
* ESC 키로 모달 닫기
* 단축키 구현
* 게임 캐릭터 이동
* 방향키 조작

---

### 8.1 주요 키보드 이벤트

| 이벤트       | 발생 시점     |
| --------- | --------- |
| `keydown` | 키를 누르는 순간 |
| `keyup`   | 키를 떼는 순간  |

---

### 8.2 주요 속성

| 속성           | 설명                  |
| ------------ | ------------------- |
| `e.key`      | 입력된 키 값             |
| `e.code`     | 물리적인 키 위치           |
| `e.ctrlKey`  | Ctrl 키가 함께 눌렸는지 여부  |
| `e.shiftKey` | Shift 키가 함께 눌렸는지 여부 |
| `e.altKey`   | Alt 키가 함께 눌렸는지 여부   |

---

### 8.3 Enter 키로 검색 실행

```html
<input id="search" placeholder="검색어 입력">
```

```js
const search = document.getElementById("search");

search.addEventListener("keydown", (e) => {
  if (e.key === "Enter") {
    alert("검색: " + search.value);
  }
});
```

---

### 8.4 ESC 키 처리

```js
document.addEventListener("keydown", (e) => {
  if (e.key === "Escape") {
    alert("ESC 눌렀어요!");
  }
});
```

---

### 8.5 Ctrl + S 단축키 처리

```js
document.addEventListener("keydown", (e) => {
  if (e.ctrlKey && e.key === "s") {
    e.preventDefault();
    alert("문서 저장!");
  }
});
```

`Ctrl + S`는 브라우저의 기본 저장 단축키이므로, 직접 저장 기능을 만들고 싶다면 `e.preventDefault()`로 기본 동작을 막아야 한다.

---

### 8.6 화살표 키 처리

```js
document.addEventListener("keydown", (e) => {
  if (e.key === "ArrowLeft") {
    console.log("왼쪽");
  }

  if (e.key === "ArrowRight") {
    console.log("오른쪽");
  }
});
```

게임이나 슬라이더, 키보드 네비게이션 기능에 사용할 수 있다.

---

## 9. 자주 사용하는 e.key 값

`e.key`는 키보드 이벤트에서 어떤 키가 눌렸는지 확인할 때 사용한다.

정확한 값을 모를 때는 다음처럼 콘솔에 출력해서 확인할 수 있다.

```js
document.addEventListener("keydown", (e) => {
  console.log(e.key);
});
```

---

### 9.1 특수 키

| 키         | `e.key` 값     |
| --------- | ------------- |
| Enter     | `"Enter"`     |
| ESC       | `"Escape"`    |
| Tab       | `"Tab"`       |
| Shift     | `"Shift"`     |
| Ctrl      | `"Control"`   |
| Alt       | `"Alt"`       |
| Backspace | `"Backspace"` |
| Delete    | `"Delete"`    |
| Caps Lock | `"CapsLock"`  |

특수 키는 앞 글자가 대문자인 경우가 많으므로 대소문자를 주의해야 한다.

---

### 9.2 방향키

| 키     | `e.key` 값      |
| ----- | -------------- |
| ← 왼쪽  | `"ArrowLeft"`  |
| → 오른쪽 | `"ArrowRight"` |
| ↑ 위   | `"ArrowUp"`    |
| ↓ 아래  | `"ArrowDown"`  |

---

### 9.3 문자 / 숫자 키

| 키         | `e.key` 값 |
| --------- | --------- |
| 알파벳 a     | `"a"`     |
| Shift + a | `"A"`     |
| 숫자 1      | `"1"`     |
| 스페이스바     | `" "`     |

스페이스바는 `"Space"`가 아니라 공백 문자 `" "`로 비교해야 한다.

```js
if (e.key === " ") {
  console.log("스페이스바");
}
```

---

### 9.4 기능 키 / 페이지 키

| 키         | `e.key` 값        |
| --------- | ---------------- |
| F1 ~ F12  | `"F1"` ~ `"F12"` |
| Home      | `"Home"`         |
| End       | `"End"`          |
| Page Up   | `"PageUp"`       |
| Page Down | `"PageDown"`     |
| Insert    | `"Insert"`       |

---

## 10. 핵심 정리

### 이벤트 처리 기본 구조

```js
요소.addEventListener("이벤트이름", () => {
  실행할 코드
});
```

예시:

```js
const btn = document.getElementById("btn");

btn.addEventListener("click", () => {
  alert("버튼 클릭");
});
```

---

### 이벤트 객체 사용 구조

```js
요소.addEventListener("이벤트이름", (e) => {
  console.log(e.target);
});
```

이벤트 객체를 사용하면 이벤트가 발생한 요소, 입력값, 키보드 입력값 등을 확인할 수 있다.

---

### 마우스 이벤트 핵심

* 클릭: `click`
* 더블클릭: `dblclick`
* 마우스 이동: `mousemove`
* 마우스 올림: `mouseenter`
* 마우스 벗어남: `mouseleave`
* 우클릭: `contextmenu`

메뉴 구현에는 `mouseover/out`보다 `mouseenter/leave`가 적합하다.

---

### 키보드 이벤트 핵심

* 키를 누르는 순간: `keydown`
* 키를 떼는 순간: `keyup`
* 눌린 키 확인: `e.key`
* Ctrl / Shift / Alt 조합 확인:

  * `e.ctrlKey`
  * `e.shiftKey`
  * `e.altKey`
