# L14 JavaScript (2) 정리

## 1. 변수 선언: `var`, `let`, `const`

JavaScript에서 변수를 선언할 때는 `var`, `let`, `const`를 사용할 수 있다.

| 구분 | var | let | const |
|---|---|---|---|
| 재선언 | 가능 | 불가 | 불가 |
| 재할당 | 가능 | 가능 | 불가 |

### 핵심 정리

- `var`
  - 예전 방식
  - 재선언이 가능해서 버그를 유발할 수 있음
  - 현재는 사용을 지양함

- `let`
  - 재할당 가능
  - 재선언 불가
  - 값이 바뀌어야 하는 변수에 사용

- `const`
  - 재할당 불가
  - 재선언 불가
  - 기본적으로 가장 권장되는 방식

```js
const name = '홍길동';
name = '김철수'; // 오류
````

### const 배열/객체 주의점

`const`로 배열이나 객체를 선언해도 내부 값은 변경할 수 있다.

```js
const arr = [1, 2, 3];

arr.push(4);     // 가능
arr = [5, 6];    // 오류
```

즉, `const`는 변수 자체의 재할당을 막는 것이지, 배열이나 객체 내부 변경까지 막는 것은 아니다.

## 2. 데이터 타입

JavaScript의 주요 데이터 타입은 다음과 같다.

| 타입        | 설명          | 예시                   |
| --------- | ----------- | -------------------- |
| string    | 문자열         | `'hello'`, `"world"` |
| number    | 숫자          | `42`, `3.14`         |
| boolean   | 참/거짓        | `true`, `false`      |
| null      | 의도적으로 비어 있음 | `null`               |
| undefined | 값이 아직 없음    | `undefined`          |

### typeof 사용

```js
typeof 'hello';    // "string"
typeof 42;         // "number"
typeof true;       // "boolean"
typeof undefined;  // "undefined"
typeof null;       // "object"
```

`typeof null`의 결과가 `"object"`로 나오는 것은 JavaScript의 역사적 버그이다.

### null과 undefined 차이

| 값           | 의미                  |
| ----------- | ------------------- |
| `undefined` | 값을 아직 넣지 않은 상태      |
| `null`      | 의도적으로 비어 있음을 표시한 상태 |

```js
let x;
console.log(x); // undefined

let y = null;
console.log(y); // null
```

## 3. 타입 변환

`prompt()`로 입력받은 값은 항상 문자열이다.
따라서 숫자 계산을 하려면 `Number()`를 사용해 숫자로 변환해야 한다.

```js
const age = Number(prompt('나이를 입력하세요'));
```

### 문자열과 숫자의 연산

```js
'3' + '4'; // "34"
3 + 4;     // 7
```

`+` 연산자는 문자열 연결에도 사용되기 때문에 주의해야 한다.

```js
'3' * 2; // 6
'3' - 2; // 1
'3' / 2; // 1.5
'3' + 2; // "32"
```

`*`, `-`, `/`는 문자열을 자동으로 숫자로 변환하지만, `+`는 문자열 연결로 처리될 수 있다.

### 주요 변환 함수

```js
Number('42');      // 42
Number('3.14');    // 3.14
Number('hello');   // NaN

String(42);        // "42"

Boolean(0);        // false
Boolean('');       // false
Boolean('hello');  // true
Boolean(1);        // true
```

### NaN

`NaN`은 `Not a Number`의 줄임말이다.
숫자로 변환할 수 없는 값이 계산에 사용될 때 발생한다.

```js
Number('hello');   // NaN
isNaN('hello');    // true
isNaN(42);         // false
```

### Falsy 값

Boolean으로 변환했을 때 `false`가 되는 값은 다음과 같다.

```js
0
''
null
undefined
NaN
false
```

## 4. 연산자

## 4-1. 산술 연산자

| 연산자  | 의미   | 예시       |
| ---- | ---- | -------- |
| `+`  | 더하기  | `3 + 2`  |
| `-`  | 빼기   | `3 - 2`  |
| `*`  | 곱하기  | `3 * 2`  |
| `/`  | 나누기  | `3 / 2`  |
| `%`  | 나머지  | `5 % 2`  |
| `**` | 거듭제곱 | `2 ** 3` |
| `++` | 1 증가 | `x++`    |
| `--` | 1 감소 | `x--`    |

```js
5 % 2;   // 1
2 ** 3;  // 8
```

### 전위 증가와 후위 증가

```js
let a = 5;

console.log(a++); // 5
console.log(++a); // 7
```

* `a++`: 현재 값을 먼저 사용하고 나중에 증가
* `++a`: 먼저 증가시키고 값을 사용

## 4-2. 할당 연산자

| 연산자  | 의미      |
| ---- | ------- |
| `=`  | 대입      |
| `+=` | 더한 후 대입 |
| `-=` | 뺀 후 대입  |
| `*=` | 곱한 후 대입 |
| `/=` | 나눈 후 대입 |

```js
x += 1;
// x = x + 1 과 같음
```

## 4-3. 비교 연산자

| 연산자   | 의미               |
| ----- | ---------------- |
| `===` | 값과 타입이 모두 같은지 비교 |
| `!==` | 값 또는 타입이 다른지 비교  |
| `==`  | 값만 비교            |
| `!=`  | 값만 다름 비교         |
| `>`   | 초과               |
| `<`   | 미만               |
| `>=`  | 이상               |
| `<=`  | 이하               |

### `===` 사용 권장

```js
3 === '3'; // false
3 == '3';  // true
```

`==`는 자동 타입 변환이 일어나므로 예상하지 못한 결과가 나올 수 있다.
따라서 일반적으로 `===` 사용이 권장된다.

## 4-4. 논리 연산자

| 연산자  | 의미  |   |    |
| ---- | --- | - | -- |
| `&&` | AND |   |    |
| `    |     | ` | OR |
| `!`  | NOT |   |    |

```js
true && true;   // true
true || false;  // true
!true;          // false
```

## 5. 조건문

조건문은 특정 조건에 따라 다른 코드를 실행할 때 사용한다.

## 5-1. if / else if / else

```js
if (score >= 90) {
  result = 'A';
} else if (score >= 80) {
  result = 'B';
} else if (score >= 70) {
  result = 'C';
} else {
  result = 'F';
}
```

### 사용 상황

* 범위 비교
* 복잡한 조건 처리
* 여러 조건 조합

## 5-2. switch

`switch`는 하나의 값을 여러 경우와 비교할 때 사용한다.

```js
switch (day) {
  case 1:
    console.log('월요일');
    break;
  case 2:
    console.log('화요일');
    break;
  default:
    console.log('기타');
}
```

### break 주의

`switch`에서 `break`를 생략하면 아래 case까지 계속 실행된다.
이를 `fall-through`라고 한다.

## 5-3. 삼항 연산자

간단한 `if / else`는 삼항 연산자로 줄일 수 있다.

```js
const result = score >= 60 ? '합격' : '불합격';
```

형식은 다음과 같다.

```js
조건 ? 참일 때 값 : 거짓일 때 값
```

## 6. Math 객체

`Math` 객체는 JavaScript에서 제공하는 내장 수학 기능 모음이다.
따로 import하지 않아도 바로 사용할 수 있다.

## 6-1. 반올림 관련 함수

```js
Math.floor(4.9); // 4, 내림
Math.ceil(4.1);  // 5, 올림
Math.round(4.5); // 5, 반올림
```

## 6-2. 최댓값 / 최솟값

```js
Math.max(1, 5, 3); // 5
Math.min(1, 5, 3); // 1
```

## 6-3. 절댓값 / 거듭제곱 / 제곱근

```js
Math.abs(-7);    // 7
Math.pow(2, 3);  // 8
Math.sqrt(16);   // 4
```

## 6-4. 원주율

```js
Math.PI;
// 3.141592653589793
```

## 6-5. 랜덤 숫자

```js
Math.random();
```

`Math.random()`은 `0 이상 1 미만`의 랜덤 실수를 반환한다.
즉, 1은 절대 나오지 않는다.

### 1부터 10까지 랜덤 정수

```js
Math.floor(Math.random() * 10) + 1;
```

### min 이상 max 이하 랜덤 정수 공식

```js
Math.floor(Math.random() * (max - min + 1)) + min;
```

예를 들어 1부터 100까지 랜덤 정수는 다음과 같다.

```js
Math.floor(Math.random() * 100) + 1;
```

## 7. 함수 심화

함수는 특정 작업을 수행하는 코드 묶음이다.

## 7-1. 매개변수

매개변수는 함수에 값을 전달할 때 사용한다.

```js
function greet(name) {
  alert('안녕하세요, ' + name);
}

greet('홍길동');
```

## 7-2. return

`return`은 함수의 결과값을 밖으로 돌려준다.

```js
function add(a, b) {
  return a + b;
}

const result = add(3, 4); // 7
```

`return`이 없으면 함수는 `undefined`를 반환한다.

```js
function sayHi() {
  alert('Hi');
}

console.log(sayHi()); // undefined
```

## 7-3. 기본값 매개변수

매개변수에 기본값을 지정할 수 있다.

```js
function greet(name = '손님') {
  alert('안녕하세요, ' + name);
}

greet();       // 안녕하세요, 손님
greet('철수'); // 안녕하세요, 철수
```

## 7-4. 함수 표현식

함수를 변수에 저장할 수 있다.

```js
const add = function(a, b) {
  return a + b;
};
```

## 7-5. 화살표 함수

함수를 더 짧게 표현할 수 있다.

```js
const add = (a, b) => a + b;
```

### 같은 기능의 함수 표현 비교

```js
function add(a, b) {
  return a + b;
}

const add = function(a, b) {
  return a + b;
};

const add = (a, b) => a + b;
```

## 8. 배열 기초

배열은 여러 값을 하나의 변수에 저장할 때 사용한다.

```js
const fruits = ['apple', 'banana', 'orange'];
```

## 8-1. 인덱스 접근

배열의 인덱스는 0부터 시작한다.

```js
fruits[0]; // 'apple'
fruits[1]; // 'banana'
fruits[2]; // 'orange'
```

## 8-2. 배열 길이

```js
fruits.length; // 3
```

## 8-3. 배열 값 변경

```js
fruits[1] = 'grape';
```

## 8-4. 배열 추가 / 제거

| 메서드         | 의미        |
| ----------- | --------- |
| `push()`    | 배열 끝에 추가  |
| `pop()`     | 배열 끝에서 제거 |
| `unshift()` | 배열 앞에 추가  |
| `shift()`   | 배열 앞에서 제거 |

```js
fruits.push('mango');     // 끝에 추가
fruits.pop();             // 끝에서 제거
fruits.unshift('kiwi');   // 앞에 추가
fruits.shift();           // 앞에서 제거
```

## 8-5. 포함 여부 확인

```js
fruits.includes('apple'); // true
```

## 9. 반복문

반복문은 같은 작업을 여러 번 실행할 때 사용한다.

## 9-1. for 문

```js
for (let i = 0; i < 5; i++) {
  console.log(i);
}
```

출력 결과:

```js
0
1
2
3
4
```

## 9-2. 배열 순회

```js
const fruits = ['apple', 'banana', 'orange'];

for (let i = 0; i < fruits.length; i++) {
  console.log(fruits[i]);
}
```

## 9-3. for...of 문

배열의 값을 더 간단히 순회할 수 있다.

```js
for (const fruit of fruits) {
  console.log(fruit);
}
```

## 9-4. while 문

```js
let count = 0;

while (count < 5) {
  console.log(count);
  count++;
}
```

## 9-5. break

`break`는 반복문을 중간에 종료한다.

```js
for (let i = 0; i < 10; i++) {
  if (i === 5) break;
  console.log(i);
}
```

출력 결과:

```js
0
1
2
3
4
```

## 9-6. continue

`continue`는 현재 반복을 건너뛰고 다음 반복으로 넘어간다.

```js
for (let i = 0; i < 5; i++) {
  if (i === 2) continue;
  console.log(i);
}
```

출력 결과:

```js
0
1
3
4
```

## 10. 실습 1: 계산기

숫자 두 개와 연산자를 입력받아 계산 결과를 출력하는 프로그램이다.

```js
function calculate(a, b, op) {
  if (op === '+') return a + b;
  if (op === '-') return a - b;
  if (op === '*') return a * b;
  if (op === '/') {
    if (b === 0) return '0으로 나눌 수 없습니다';
    return a / b;
  }
  return '알 수 없는 연산자';
}

const a = Number(prompt('첫 번째 숫자'));
const b = Number(prompt('두 번째 숫자'));
const op = prompt('연산자 (+, -, *, /)');

const result = calculate(a, b, op);
alert(`${a} ${op} ${b} = ${result}`);
```

### 이 코드에서 사용된 개념

* 함수
* 매개변수
* `return`
* `Number()` 타입 변환
* 조건문
* 0으로 나누기 예외 처리
* 템플릿 리터럴

### 추가 연습

* `%` 나머지 연산 추가하기
* 숫자가 아닌 입력이 들어왔을 때 처리하기

## 11. 실습 2: 업다운 게임

컴퓨터가 1부터 100 사이의 랜덤 숫자를 정하고, 사용자가 숫자를 맞히는 게임이다.

```js
const answer = Math.floor(Math.random() * 100) + 1;
let tries = 0;

while (true) {
  const guess = Number(prompt('1~100 사이 숫자를 맞혀보세요!'));
  tries++;

  if (isNaN(guess)) {
    alert('숫자를 입력해주세요');
    continue;
  }

  if (guess === answer) {
    alert(`정답! ${tries}번 만에 맞혔습니다!`);
    break;
  } else if (guess < answer) {
    alert('UP! 더 큰 숫자입니다');
  } else {
    alert('DOWN! 더 작은 숫자입니다');
  }
}
```

### 이 코드에서 사용된 개념

* `Math.random()`
* `Math.floor()`
* 랜덤 정수 생성
* `while (true)`
* `break`
* `continue`
* `tries++`
* `isNaN()`
* 조건문

### 코드 흐름

1. 컴퓨터가 1~100 사이의 정답 숫자를 생성한다.
2. 사용자가 숫자를 입력한다.
3. 입력값이 숫자가 아니면 다시 입력하게 한다.
4. 입력값이 정답보다 작으면 `UP`을 출력한다.
5. 입력값이 정답보다 크면 `DOWN`을 출력한다.
6. 정답을 맞히면 시도 횟수를 출력하고 반복을 종료한다.

### 추가 연습

* 최대 시도 횟수 제한 추가하기
* 현재 가능한 숫자 범위 표시하기
* 정답을 맞힌 뒤 다시 시작 기능 추가하기

# 핵심 암기 포인트

* 기본은 `const`
