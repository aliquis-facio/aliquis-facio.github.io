# 2. 데이터 수집 방식 관련 질문

	- 장소명 + 지역명
3. 릴스 하나에 여러 장소가 있을 때, 저장 단위는 무엇인가?
	- 장소 1개 = 개별 place 1개
4. 사용자가 링크를 넣었을 때 바로 저장할 것인가, 후보 검토 화면을 먼저 보여줄 것인가?
	- 장소명이 명확하지 않아 처리하기 어려울 때에만 후보 검토 화면을 보여준다
5. 인스타그램 API를 직접 사용할 계획인가, 아니면 사용자가 제공한 링크/텍스트/이미지를 기반으로 파싱할 계획인가?
	1. 사용자가 인스타 링크 입력  
	2. 링크에서 접근 가능한 공개 메타데이터/oEmbed/페이지 텍스트 추출 시도  
	3. 실패 시 사용자가 본문 텍스트 직접 붙여넣기  
	4. 캡처 이미지 업로드 + OCR
6. 릴스나 게시글 본문에 장소명이 명확하지 않은 경우는 어떻게 처리할 것인가?
	   * 댓글까지 분석
	   * 영상 내 자막 OCR
	   * 해시태그 분석
	   * 위치 태그 분석

	- 처리
8. 장소가 실제 상호명이 아니라 “성수 신상 카페”, “홍대 분위기 좋은 술집”처럼 표현될 때 어떻게 후보를 찾을 것인가?

```json
{
  "name": "장소명",
  "category": "카페",
  "address": {
	  "도로명 주소": "도로명 주소",
	  "지번 주소": "지번 주소"
  },
```

```text
PlaceReviewSummary
- 리뷰 분석 결과

VisitLog
- 방문일, 만족도, 메모

RecommendationLog
- 언제 어떤 조건으로 추천됐는지
```

# 8. MVP 범위 관련 질문

지금 아이디어는 기능이 꽤 크다. 처음부터 전부 만들기보다는 MVP를 정해야 한다.

1. 1차 MVP에서 꼭 필요한 기능은 무엇인가?

```text
인스타 링크 입력
→ 장소명 후보 추출
→ 네이버/카카오 지도 검색
→ 장소 정보 저장
→ 유형별 리스트/지도 보기
```

2. 구글맵까지 1차에 꼭 필요한가?
	- 국내 장소 중심이니 네이버/카카오 우선으로
3. 리뷰 분석과 광고 검증은 MVP에 포함할 것인가, 아니면 2차 기능으로 둘 것인가?
	- 2차 기능
4. 모바일 앱으로 만들 것인가, 웹 서비스로 만들 것인가?
	- 우선 웹으로
5. 사용자가 가장 먼저 체감할 수 있는 기능 하나만 고른다면 무엇인가?
	- 정리 자동화

MVP:  
- 리뷰 분석 없음  
- 리뷰 수 / 평점 / 지도 링크만 저장 가능하면 저장  
- 검증도 대신 "정보 완성도" 점수 제공  
  
2차:  
- 최신 리뷰 30~50개 분석  
- 장점/단점/주의 키워드 추출  
- 긍정/부정 절사평균  
  
3차:  
- 최신 500개까지 확장  
- 시계열 평판 변화  
- 광고성/체험단 패턴 탐지
