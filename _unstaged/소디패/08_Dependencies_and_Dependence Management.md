# DDM(Dependencies & Dependence Management)
## 1. 핵심 개념

- **의존성 증가 → 재사용성/유연성/유지보수성 감소**. 목표는 **상호 의존(결합)을 통제**하는 것. OO 자체가 의존성 관리 도구/기법의 집합이라고 볼 수 있다.
- **나쁜 구조의 징후 4가지**: 경직성(rigidity), 취약성(fragiIity), 낮은 재사용성, 높은 점성(viscosity).

## 2. 의존 관계 종류

- **상속(Inheritance)**: 특수화/일반화, 코드 재사용 목적이지만 **매우 강한 의존**. 상속 인터페이스는 언어 차원 지원이 제한적이라 문서화가 필수.
- **사용 관계(Dependency/use)**: 다른 객체의 서비스를 **런타임에 사용**해 책임을 수행.
- **연관/집합/합성(Association/Aggregation/Composition)**
    - 연관: 소유자 없음, 생명주기 독립. 예: 의사–환자.
    - 집합: 소유 관계 있으나 생명주기 독립(부서–직원).
    - 합성: 전체 소멸 시 부분도 소멸(집–방).

## 3. UML 표기

- 상속: 실선+삼각형, 인터페이스 구현: 점선+삼각형, 연관: 실선, 집합: 빈 다이아, 합성: 채운 다이아, 사용 의존: 점선+빈 화살촉.

## 4. 관리 원칙

1. **서비스 형태를 인터페이스로 정의**: 사용 관계의 결합을 낮추는 기본기. 제공자는 정적으로 몰라도 됨(대체 가능). 많은 패턴의 뼈대.
2. **상속을 인터페이스 기반 사용으로 대체**: 상속의 강결합을 피하고 합성/주입으로 유연성 확보.
3. **계약에 의한 설계(Design by Contract)**: 사전/사후조건, 불변식으로 역할·책임 경계 명시.
4. **합성 우선 원칙**: **폴리모orphism이 꼭 필요하지 않다면 합성 우선**.
