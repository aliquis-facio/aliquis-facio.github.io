## 3. 생성형 AI와 피싱 / WormGPT / FraudGPT
### 1) 생성형 AI 기반 피싱 이메일

- 공격자가 생성형 AI(일반 모델 또는 악성 LLM)를 이용해:
    - 문법·어색한 표현 없는 **완벽한 자연어 피싱 메일** 생성.
    - 피해자 이름, 직책, 회사 등 **개인화(Spear Phishing)** 도 자동으로 반영.
    - 수십·수백 개의 다양한 템플릿을 대량 생성 가능.
- 결과:
    - 예전 피싱 특유의 **조잡한 번역체**가 사라져서, 사람·필터 둘 다 구분이 더 어려워짐.
- 방어 측면 키워드
    - **MFA, 메일 인증(DMARC/SPF/DKIM), 사용자 교육, AI 기반 메일 필터** 등.

### 2) WormGPT / FraudGPT

- **WormGPT**
    - 다크웹·해킹 포럼에서 유통된, **“제한 없는 ChatGPT”를 표방한 악성 LLM**.
    - BEC(기업 이메일 사기)용 피싱 메일, 악성 코드 설명·보조 등 **사이버 범죄 지원 목적**으로 사용됨.
- **FraudGPT**
    - 마찬가지로 다크웹에서 광고된 **금융 사기·피싱 특화 LLM**.
    - 피싱 메일, 악성 코드, 가짜 웹사이트 페이지 등 사이버 범죄를 돕는 용도로 소개됨.
- 키 메시지:
    - “정상 LLM vs 악성 LLM(또는 jailbreak된 LLM)” → **AI의 듀얼 유즈(dual-use) 문제**.
    - 방어도 AI 기반으로 진화하지만, **공격자도 AI를 쓰는 시대**.

## 7. SOC / MTTD / MTTR + AI 활용
### 1) SOC (Security Operations Center, 보안 관제 센터)

- **역할**
    - 조직(또는 금융권 전체)의 로그·이벤트를 수집·모니터링.
    - 침해 시도 탐지, 경보 분석, 사고 대응 지휘.
- 구성 요소
    - SIEM/로그 수집, SOAR(자동 대응), 위협 인텔리전스, 분석가(관제 요원) 등.

### 2) MTTD / MTTR

- **MTTD (Mean Time To Detect)**
    - “공격이 발생해서 **탐지하기까지 걸린 평균 시간**”.
- **MTTR (Mean Time To Recovery)**
    - “사고 발생 후, **정상 상태로 복구하는 데 걸린 평균 시간**”.
- 보안 성숙도 지표:
    - **MTTD·MTTR를 얼마나 줄이느냐**가 SOC·보안 조직의 핵심 목표.

### 3) AI 활용 (암호화 트래픽 식별, 스크립트 대응 AI)

- **암호화 패킷 식별 AI**
    - 패킷 내용은 TLS 등으로 암호화되어도,
        - 패킷 길이, 방향, 타이밍, SNI, 메타데이터 패턴 등으로
        - “정상/이상 트래픽”을 구분하는 AI 모델.
    - 예: 이상한 패턴의 TLS 세션(짧은 세션 다량, DGA 도메인 등)을 탐지.
- **스크립트 대응 AI**
    - SOC에서 발생한 경보를 분석하고,
        - 방화벽/EDR/IPS 등에 적용할 **대응 스크립트(규칙)** 를 자동으로 제안·생성하는 AI.
    - SOAR와 결합해서,
        - “경보 분석 → 조치 스크립트 생성 → 검토 후 자동 실행”까지 일부 자동화.

## 9. 지도학습 vs 비지도학습 – 이상 탐지
### 1) 지도 학습 기반 모델

- 공격·정상 이벤트에 **라벨이 붙은 데이터**로 학습.
- 장점
    - 알려진 공격 유형에 대해 **정확도가 높음**.
- 한계
    - 보지 못한 **신규 공격(Zero-day, 새로운 APT TTP)**에는 취약.
    - 레이블링 비용이 크고 최신 공격을 계속 학습시켜야 함.

### 2) 비지도 학습 기반 모델 (이상 탐지)

- 대부분 **정상 데이터만으로 모델을 학습**.
    - 정상 패턴을 학습한 뒤, 거기서 벗어나는 것을 “이상”으로 탐지.
- 장점
    - **새로운 공격**도 “정상과 다르면” 어느 정도 탐지 가능.
- 한계
    - 비정상 = 공격이 아닐 수도 → **오탐(False Positive)이 많이 발생**.
    - 실제 운영 시, SOC 인력 피로도 증가 → 튜닝·검증 필수.
