다음은 제공된 강의 자료 PDF와 녹음 기반 AI 추출 텍스트를 종합해 재구성한 강의 정리본이다. 강의명과 주차 정보는 PDF 표지에서 확인했고, 전체 흐름은 PDF의 목차형 전개를 기준으로, 교수자의 구두 설명은 추출 텍스트로 보강했다.

# 강의 제목

**웹클라이언트컴퓨팅 L02 – 인터넷과 웹 / 웹 시스템**

# 1. 강의 개요

이번 강의는 크게 두 부분으로 구성된다. 첫째, **인터넷이 어떻게 탄생했고 어떤 구조로 동작하는가**를 다룬다. 둘째, **웹 시스템이 무엇이며, 웹 브라우저·HTML·URL·HTTP·클라이언트-서버 구조가 어떻게 연결되는가**를 설명한다. PDF는 통신의 역사 → 데이터 통신 → 프로토콜 → 라우터와 인터넷 → 아르파넷 → IP/DNS → 웹의 시작 → 웹 브라우저 → HTML/URL/HTTP → 클라이언트-서버 → HTML 태그 순으로 전개된다. 교수자는 이 흐름에 맞춰 개념을 쉬운 비유와 사례로 풀어 설명했고, 특히 “왜 이런 개념이 필요해졌는가”를 반복해서 강조했다.

# 2. 핵심 개념 정리

## 통신

- 정의: 의사, 감정, 정보를 한쪽에서 다른 쪽으로 전달하는 것.
    
- 왜 중요한지: 인터넷도 결국 통신의 한 형태이므로, 인터넷을 이해하려면 먼저 통신의 기본 개념을 알아야 한다.
    
- 교수자 설명 요약: 사람 사이의 대화는 가장 기본적인 통신이지만 근거리 통신에 한정된다. 멀리 전달하려면 우편, 더 빠른 원거리 전달을 위해 봉화 같은 방식이 등장했다.
    
- PDF 자료 핵심 내용: 대화(근거리), 우편(원거리지만 느림), 봉화(원거리 긴급 전달) 순으로 설명한다.
    
- 이해를 돕는 예시: 가까운 사람과 말로 이야기하는 것은 대화, 먼 곳에 문서를 보내는 것은 우편에 해당한다.
    
- 주의할 점 / 헷갈리기 쉬운 점: 인터넷은 갑자기 등장한 것이 아니라, 통신 방식이 점차 발전한 결과라는 점을 놓치기 쉽다.
    

## 데이터 통신

- 정의: 컴퓨터와 같은 기기들이 데이터를 주고받는 것.
    
- 왜 중요한지: 인터넷은 본질적으로 컴퓨터 간 데이터 통신 네트워크다.
    
- 교수자 설명 요약: 예전에는 테이프 드라이브나 플로피 디스크를 직접 옮기는 방식으로 데이터를 전달했다. 이후 전선을 통해 직접 데이터를 보내는 방식이 등장하면서 본격적인 데이터 통신이 시작되었다.
    
- PDF 자료 핵심 내용: 데이터 통신의 정의, 저장매체 기반 전달, 전선을 통한 최초의 데이터 통신을 제시한다.
    
- 이해를 돕는 예시: USB를 직접 가져다 주는 방식은 네트워크 이전의 데이터 전달 방식과 비슷하다.
    
- 주의할 점 / 헷갈리기 쉬운 점: 저장매체로 옮기는 것은 ‘데이터 전달’이지만, 네트워크 기반 실시간 통신과는 다르다.
    

## 프로토콜

- 정의: 통신할 때 지켜야 하는 약속 또는 규약.
    
- 왜 중요한지: 누가 먼저 보내는지, 어느 속도로 보내는지 등이 정해지지 않으면 통신이 성립하지 않는다.
    
- 교수자 설명 요약: 사람의 대화도 말하는 사람과 듣는 사람이 정해져야 하듯, 컴퓨터 통신도 규칙이 필요하다.
    
- PDF 자료 핵심 내용: 데이터 통신 시 누가 먼저 보내고 얼마나 빠르게 보낼지 정하는 통신 약속이 필요하다고 설명한다.
    
- 이해를 돕는 예시: 둘 다 동시에 말하면 대화가 꼬이듯, 컴퓨터도 규칙 없이 데이터를 보내면 충돌이나 오류가 난다.
    
- 주의할 점 / 헷갈리기 쉬운 점: 프로토콜은 단순히 “약속”이 아니라 실제 통신 절차와 형식을 정하는 기술 규약이다.
    

## 라우터와 네트워크

- 정의: 라우터는 여러 기기를 효율적으로 연결하고 데이터가 갈 방향을 정해 주는 장치이며, 네트워크는 연결된 기기들의 집합이다.
    
- 왜 중요한지: 컴퓨터 수가 많아질수록 모든 기기를 일대일로 연결하는 것은 비효율적이기 때문이다.
    
- 교수자 설명 요약: 두 대는 직접 연결 가능하지만, 기기가 많아지면 선의 수가 폭발적으로 증가한다. 라우터를 쓰면 각 기기를 라우터에만 연결해도 상호 통신이 가능해진다.
    
- PDF 자료 핵심 내용: 두 대 연결, 여러 대 직접 연결, 라우터를 이용한 네트워크를 비교해 보여 준다.
    
- 이해를 돕는 예시: 모든 사람끼리 직접 전화선을 까는 대신, 교환국 하나를 통해 연결하는 방식으로 이해할 수 있다.
    
- 주의할 점 / 헷갈리기 쉬운 점: 라우터는 단순 분배기가 아니라, 목적지에 맞춰 경로를 정하는 역할을 한다.
    

## 인터넷

- 정의: 네트워크와 네트워크를 연결한, 즉 **네트워크의 네트워크**.
    
- 왜 중요한지: 오늘날 전 세계 컴퓨터 통신의 기본 인프라다.
    
- 교수자 설명 요약: 한 개의 네트워크 안에서 끝나는 것이 아니라, 여러 네트워크가 서로 연결된 거대한 구조가 인터넷이다.
    
- PDF 자료 핵심 내용: inter-net이라는 말 그대로 수많은 네트워크를 연결한 구조라고 설명한다.
    
- 이해를 돕는 예시: 한 건물의 내부망이 여러 건물, 여러 기관의 망과 이어져 거대한 전체망이 되는 구조다.
    
- 주의할 점 / 헷갈리기 쉬운 점: 인터넷과 웹은 같은 말이 아니다. 인터넷은 기반 네트워크, 웹은 그 위에서 동작하는 서비스다.
    

## 아르파넷(ARPAnet)

- 정의: 현대 인터넷의 시초가 된 초기 분산형 네트워크.
    
- 왜 중요한지: 인터넷의 역사적 출발점이기 때문이다.
    
- 교수자 설명 요약: 냉전 시기 미국이 일부 통신 시설이 파괴되어도 전체 통신이 마비되지 않도록 분산형 구조를 연구했고, 그 결과 아르파넷이 만들어졌다.
    
- PDF 자료 핵심 내용: 1960년대 ARPA가 전쟁 대비 차원에서 패킷 교환 기반 분산 네트워크를 연구했고, 1969년 4개 대학을 연결한 아르파넷이 탄생했다고 설명한다.
    
- 이해를 돕는 예시: 한 지점이 고장 나도 전체 시스템이 멈추지 않도록 우회 가능한 도로망을 만드는 것과 비슷하다.
    
- 주의할 점 / 헷갈리기 쉬운 점: 인터넷은 처음부터 대중용 서비스가 아니라 국방·연구 목적에서 출발했다.
    

## IP 주소 / 도메인 이름 / DNS 서버

- 정의:
    
    - IP 주소: 인터넷에 연결된 각 컴퓨터를 식별하는 주소
        
    - 도메인 이름: 사람이 기억하기 쉬운 문자 기반 주소
        
    - DNS 서버: 도메인 이름을 IP 주소로 변환하는 서버
        
- 왜 중요한지: 인터넷 상에서 정확한 목적지를 찾아가려면 주소 체계가 필요하다.
    
- 교수자 설명 요약: 숫자로 된 IP 주소를 모두 기억하기 어렵기 때문에 google.com 같은 도메인을 사용하고, DNS가 이를 숫자 주소로 바꿔 준다.
    
- PDF 자료 핵심 내용: IP 주소, 도메인 이름, DNS 서버의 역할을 순차적으로 설명한다.
    
- 이해를 돕는 예시: 도메인은 사람용 별칭, IP는 실제 주소, DNS는 전화번호부 역할을 한다.
    
- 주의할 점 / 헷갈리기 쉬운 점: 사용자가 입력하는 것은 보통 도메인이며, 실제 통신은 IP 기반으로 이루어진다.
    

## 문자 기반 인터페이스

- 정의: 키보드 입력을 중심으로 컴퓨터에 명령을 내리는 방식의 인터페이스.
    
- 왜 중요한지: 초기 웹 시스템과 현대 프롬프트 인터페이스를 비교하는 기준이 된다.
    
- 교수자 설명 요약: 하이텔, 천리안 같은 초기 서비스는 키보드만으로 조작했다. 교수자는 ChatGPT의 프롬프트 방식도 넓게 보면 문자 기반 인터페이스로 볼 수 있다고 설명했다.
    
- PDF 자료 핵심 내용: 웹의 시작은 문자 기반 인터페이스였고 초창기 인터넷은 국방/연구용이었다고 제시한다.
    
- 이해를 돕는 예시: 메뉴 키를 눌러 이동하는 천리안과 자연어를 입력하는 ChatGPT는 방식은 다르지만 둘 다 텍스트 입력 기반이다.
    
- 주의할 점 / 헷갈리기 쉬운 점: “프롬프트 방식 = 옛날 문자 인터페이스와 완전히 동일”은 아니다. 입력 형식과 처리 방식에는 차이가 있다.
    

## 웹 브라우저와 모자이크

- 정의: 웹 서버의 문서, 이미지, 동영상 등을 화면에 보여 주고 하이퍼링크를 따라 이동할 수 있게 해 주는 응용 프로그램.
    
- 왜 중요한지: 웹을 대중화한 핵심 도구다.
    
- 교수자 설명 요약: 초기 문자 중심 환경과 달리, 모자이크는 문자와 그림을 한 화면에서 보여 주고 링크를 클릭해 이동할 수 있게 만들었다.
    
- PDF 자료 핵심 내용: 1993년 일리노이대의 마크 앤드리슨과 에릭 비나가 모자이크를 개발했고, 이것이 이후 브라우저의 원형이 되었다고 설명한다.
    
- 이해를 돕는 예시: 오늘날 크롬, 엣지, 파이어폭스가 모두 이 계보에 있다.
    
- 주의할 점 / 헷갈리기 쉬운 점: 웹 브라우저는 단순 뷰어가 아니라 하이퍼링크 기반 탐색을 가능하게 한 도구다.
    

## HTML

- 정의: 하이퍼텍스트를 만들기 위한 마크업 언어(HyperText Markup Language).
    
- 왜 중요한지: 웹페이지의 기본 뼈대를 구성한다.
    
- 교수자 설명 요약: 프로그램 개발에 C, Java, Python이 필요하듯, 웹페이지를 만들려면 HTML이 필요하다. HTML은 태그를 이용해 문서 구조와 의미를 표현한다.
    
- PDF 자료 핵심 내용: HTML의 정의, 웹페이지와의 관계, 마크업 언어의 개념, `<b>`, `<i>` 같은 예시를 제시한다.
    
- 이해를 돕는 예시: `<h1>`은 제목, `<p>`는 문단, `<img>`는 이미지, `<a>`는 링크를 표현한다.
    
- 주의할 점 / 헷갈리기 쉬운 점: HTML은 프로그래밍 언어라기보다 구조를 표시하는 마크업 언어에 가깝다.
    

## URL / HTTP / HTTPS / WWW

- 정의:
    
    - URL: 자원의 위치를 나타내는 주소 체계
        
    - HTTP: 하이퍼텍스트 전송 프로토콜
        
    - HTTPS: HTTP의 암호화 버전
        
    - WWW: 월드와이드웹 서비스임을 나타내는 표현
        
- 왜 중요한지: 사용자가 웹 자원에 접근하는 가장 기본적인 주소 및 통신 체계다.
    
- 교수자 설명 요약: `https://www.naver.com` 같은 주소는 프로토콜 + 웹 서비스 표기 + 도메인 이름으로 이해할 수 있다.
    
- PDF 자료 핵심 내용: HTTP는 하이퍼텍스트를 요청하는 프로토콜, HTTPS는 보안 버전, WWW는 world wide web을 뜻한다고 설명한다.
    
- 이해를 돕는 예시: 온라인 결제나 로그인 정보처럼 민감한 데이터는 HTTPS를 사용해야 한다.
    
- 주의할 점 / 헷갈리기 쉬운 점: URL은 웹 주소만이 아니라 더 넓은 자원 위치 체계로 볼 수 있다. 다만 이번 강의에서는 웹 주소 맥락이 중심이다.
    

## 클라이언트-서버 구조

- 정의: 서비스를 요청하는 쪽(클라이언트)과 요청에 응답하는 쪽(서버)으로 나뉜 구조.
    
- 왜 중요한지: 웹 시스템의 기본 작동 원리이기 때문이다.
    
- 교수자 설명 요약: 브라우저가 서버에 HTTP 요청을 보내고, 서버는 HTML 등을 응답한다. 서버는 항상 대기하고 있어야 하며, 이런 상시 동작 프로그램을 데몬이라고 부른다.
    
- PDF 자료 핵심 내용: 클라이언트, HTTP, 데몬, HTTPD의 의미를 설명한다.
    
- 이해를 돕는 예시: 사용자가 브라우저에서 주소를 입력하면 서버가 페이지를 보내 주는 구조다.
    
- 주의할 점 / 헷갈리기 쉬운 점: 브라우저는 클라이언트이고, 웹페이지 그 자체와는 다르다.
    

# 3. 세부 내용 정리

## 3-1. 통신에서 인터넷으로

강의는 “통신이란 무엇인가”에서 시작한다. 먼저 사람 사이의 대화, 우편, 봉화처럼 점차 거리와 속도의 문제를 해결해 온 통신의 역사를 설명한다. 이후 전기의 발명으로 유선 전화, 무선 통신 시스템이 등장했고, 여기서 컴퓨터 간 **데이터 통신** 개념으로 넘어간다. 교수자는 이 과정에서 “인터넷도 결국 더 효율적인 통신 방식의 발전 결과”라는 점을 자연스럽게 연결했다.

## 3-2. 저장매체 전달에서 네트워크 통신으로

초기에는 테이프 드라이브, 플로피 디스크 같은 저장매체로 데이터를 옮겼다. 그러나 이 방식은 느리고 비효율적이다. 그래서 전선을 통해 직접 데이터를 보내는 방식이 나타났고, 이것이 본격적인 데이터 통신의 출발점이 된다. 교수자는 플로피 디스크를 예로 들며 요즘은 거의 쓰지 않는 저장매체라는 점도 덧붙였다.

## 3-3. 프로토콜의 필요성

데이터를 주고받으려면 단순 연결만으로는 부족하다. 누가 먼저 보내는지, 어느 속도로 보내는지 같은 규칙이 필요하다. 이 규칙이 **프로토콜**이다. 교수자는 사람의 대화에 비유하여 “각자 동시에 말하거나 듣기만 하면 통신이 성립하지 않는다”고 설명했다.

## 3-4. 두 대 연결에서 라우터 기반 네트워크로

두 대의 컴퓨터는 1대1 연결이 가능하지만, 여러 대가 되면 필요한 선의 수가 급증한다. 이를 해결하기 위해 **라우터**를 중심으로 각 기기를 연결하는 방식이 등장한다. 라우터는 목적지에 맞게 데이터를 전달해 주므로 전체 연결 비용과 복잡도를 크게 줄인다. 이 단계에서 인터넷은 하나의 네트워크가 아니라 **네트워크들의 연결**이라는 개념으로 확장된다.

## 3-5. 아르파넷과 인터넷의 역사

강의는 이어서 인터넷의 역사적 배경으로 **아르파넷**을 소개한다. 냉전 시기 전쟁 대비용 분산형 네트워크 연구가 인터넷의 출발점이었다. 핵심은 일부가 파괴되어도 전체가 멈추지 않는 구조다. 교수자는 “국방 목적에서 출발했지만 현대 인터넷의 시초가 되었다”는 점을 짚었다.

## 3-6. IP, 도메인, DNS

인터넷에 연결된 수많은 컴퓨터를 구별하려면 주소가 필요하고, 이것이 **IP 주소**다. 그러나 숫자 주소는 사람이 기억하기 어렵기 때문에 **도메인 이름**을 사용한다. 이때 **DNS 서버**가 도메인을 IP로 바꿔 준다. 교수자는 “도메인을 입력하면 서버가 IP 주소로 변환한다”는 동작 흐름을 반복해 설명했다.

## 3-7. 웹의 시작: 문자 기반 인터페이스

이제 인터넷 위에서 동작하는 **웹 시스템**으로 넘어간다. 웹의 초기 형태는 하이텔, 천리안 같은 문자 기반 인터페이스였다. 마우스 중심 GUI 이전에는 키보드만으로 메뉴를 이동하고 명령을 내렸다. 교수자는 이 맥락에서 ChatGPT의 프롬프트도 넓게 보면 문자 기반 인터페이스로 볼 수 있는지 질문을 던졌고, 결론적으로는 “입력 방식이 텍스트라는 점에서 문자 기반 인터페이스로 볼 수 있다”고 정리했다.

## 3-8. 모자이크와 웹 브라우저의 등장

1993년 모자이크 브라우저는 문자만이 아니라 그림과 링크를 한 화면에 보여 주고, 클릭을 통한 이동을 가능하게 했다. 이것이 웹 브라우저의 핵심 전환점이다. 교수자는 하이퍼링크를 “지금도 그대로 유지되고 있는 웹의 핵심 컨셉”으로 설명했다.

## 3-9. HTML, URL, HTTP, HTTPS

웹페이지는 HTML로 작성된다. 사용자는 URL을 통해 자원을 요청하고, HTTP/HTTPS 프로토콜을 통해 서버와 통신한다. 교수자는 특히 HTTPS의 보안 의미를 설명하며, 오늘날 결제·비밀번호·개인정보 전송 때문에 보안이 필수라고 강조했다.

## 3-10. 클라이언트-서버 구조와 HTML 태그 기초

브라우저는 클라이언트, 서버는 요청을 처리하는 쪽이다. 서버는 늘 대기해야 하므로 데몬 개념이 등장한다. 이후 HTML의 탄생 배경, 태그의 의미, 기본 태그 예시로 강의가 마무리된다. 교수자는 다음 주부터는 본격적으로 HTML 코딩을 배우게 된다고 예고하면서, `<h1>`, `<p>`, `<b>`, `<i>` 등 대표 태그와 문서 구조를 간단히 소개했다.

# 4. 교수자가 강조한 내용

- **인터넷은 네트워크의 네트워크**라는 정의
    
- **프로토콜은 통신 규약**이며, 통신 성립의 필수 조건이라는 점
    
- **IP 주소, 도메인 이름, DNS 서버의 관계**
    
- **웹과 인터넷은 같은 개념이 아니라는 점**
    
- **웹 브라우저의 핵심은 하이퍼링크 기반 탐색**
    
- **HTML은 웹 개발의 뼈대**라는 점
    
- **HTTP와 HTTPS의 차이**, 특히 HTTPS의 보안 필요성
    
- 다음 시간부터는 개념 설명보다 **HTML 실습·코딩 중심 수업**으로 넘어간다는 점
    

# 5. 자료에 없지만 녹음에서 보강된 설명

- 플로피 디스크나 테이프 드라이브는 교수자도 직접 써 본 적이 거의 없다고 하며, 요즘 학생들이 체감하기 어려운 옛 저장매체라고 설명했다.
    
- 라우터 설명에서 “J에게 파일을 보내고 싶다고 라우터에 말하면 방향을 바꿔 전달한다”는 식으로 매우 직관적인 비유를 사용했다.
    
- ChatGPT 프롬프트를 문자 기반 인터페이스로 볼 수 있는지 질문을 던져, 과거 텍스트 UI와 현재 AI 인터페이스를 연결해 사고하게 했다.
    
- HTML 설명에서 단순 정의로 끝내지 않고, `<b>`, `<i>`, `<h1>`, `<p>` 등을 실제 입력 시 화면이 어떻게 달라지는지 예고했다.
    
- 이번 시간은 역사와 구조 이해가 목적이고, 다음부터는 실제로 HTML을 직접 작성하면서 배운다고 안내했다.

# 8. 한눈에 보는 요약

이번 강의는 인터넷과 웹의 기본 구조를 이해하는 입문 강의였다. 통신은 대화, 우편, 봉화 같은 방식에서 출발해 전기와 데이터 통신으로 발전했다. 컴퓨터 간 통신에는 프로토콜이 필요하며, 기기가 많아질수록 라우터 중심 네트워크가 효율적이다. 이렇게 여러 네트워크가 연결된 것이 인터넷이다. 인터넷의 역사적 출발점은 분산형 구조를 목표로 한 아르파넷이다. 인터넷에서 각 컴퓨터는 IP 주소로 식별되며, 사람이 쓰기 쉽게 도메인 이름과 DNS가 사용된다. 웹은 인터넷 위에서 동작하는 서비스이며, 초기에는 문자 기반 인터페이스였다. 모자이크 브라우저의 등장은 하이퍼링크와 멀티미디어 표현을 가능하게 하며 웹의 대중화를 이끌었다. 웹페이지는 HTML로 작성되고, URL과 HTTP/HTTPS를 통해 클라이언트와 서버가 통신한다. 다음 수업부터는 이런 개념을 바탕으로 HTML 코딩을 본격적으로 배우게 된다.

# 9. 확인이 필요한 불확실한 부분

- **녹음 원본(m4a) 자체를 직접 분석한 결과는 확인하지 못했고**, 제공된 **AI 추출 텍스트와 PDF를 중심으로 재구성**했다. 따라서 음성 억양, 판서, 현장 반응 등은 직접 반영되지 않았을 수 있다.
    
- 추출 텍스트에는 오인식이 다수 있었다. 예를 들어 `인천 네트워킹 프로토콜`은 문맥상 **Inter-networking Protocol**, `DMS`는 **DNS**, `HTP/HPS/HDP/HTTP D` 등은 각각 **HTTP/HTTPS/HTTPD**로 보정하는 것이 타당하다. 이 보정은 PDF의 표기와 문맥을 기준으로 했다.
    
- 추출 텍스트에서 외부 영상 설명 중 `ISP`가 `ISU`, `IST` 등으로 잘못 인식된 부분이 있다. 문맥상 인터넷 서비스 제공자 **ISP**를 의미하는 것으로 판단된다.
    
- PDF에는 **패킷 교환 방식**이 아르파넷 배경 설명에 등장하지만, 교수자의 본 강의 구두 설명에서는 이 부분이 아주 자세히 전개되지는 않았다. 따라서 시험 범위에서는 “분산형 네트워크의 필요성” 수준으로 이해하는 것이 우선일 가능성이 높다.
    
- PDF 21페이지의 도메인 규칙 예시는 “일반적 경향”을 보여 주는 수준이며, 교수자도 실제로는 반드시 그 규칙만 따르는 것은 아니라고 설명했다. 따라서 엄격한 규칙 암기보다는 **도메인 네임이 IP 대신 쓰이는 문자 기반 주소 체계**라는 점이 핵심이다.
