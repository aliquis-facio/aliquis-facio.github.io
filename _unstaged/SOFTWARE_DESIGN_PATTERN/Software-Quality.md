# 소프트웨어 품질
## 1) 품질의 정의

- **Crosby**: 요구사항 준수, **Deming**: 고객만족, **Feigenbaum**: 전사적·동적 품질, **Ishikawa**: 제품·서비스·정보·프로세스·사람·시스템, **Juran**: 사용 적합성·계획–통제–개선, **Shewhart**: 객관·주관 양면.
    
- **정리**: 명세 적합성=Verification, 고객 기대 충족=Validation.
    

## 2) 관리 사이클 & 측정

- **Shewhart/PDCA**: 계획→실행→점검→개선, 카이젠의 핵심 도구.
    
- **측정의 역할**: “측정→이해→통제→개선”의 출발점, 측정=사물/사건에 숫자 부여.
    

## 3) 품질관리(QC)

- 프로세스 기반 통제(입력→프로세스→출력), 전통적 QC ↔ **통계적 QC** 대비.
    

## 4) 알고리즘 관점의 품질

- **Correctness**, **Efficiency(O(f(n))—시간/공간)**, **Maintainability(확장성)**, **Understandability(구조)**.
    

## 5) 소프트웨어 품질 모델(계보)

- **McCall → Boehm → Dromey(1995) → ISO/IEC 9126 → Dromey(1998) → ISO/IEC 25010(SQuaRE)**.
    
- 최신 **ISO/IEC 25010 품질 특성 모델** 제시.
    
- 참고 연대: McCall(1977), Boehm(1978), ISO 9126(1991/2001–2003), ISO 25010(2011), Dromey(1995).
    

## 6) 조직 역량

- **CMMI**, **Six Sigma** 등 조직 차원의 성숙·개선 프레임워크.
